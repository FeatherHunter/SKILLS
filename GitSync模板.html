prompt.sh 
智能分析 Git 变更并生成有意义的 commit：

工作目录：/mnt/d/2Study/StudyNotes

流程：
1. cd /mnt/d/2Study/StudyNotes
2. 确保 git remote 使用 HTTPS + token
3. git pull --rebase origin master
4. git diff --name-only 列出所有变更文件
5. 如果没有变更文件，输出「无变更」并结束
6. 实时分析文件关联性，按「功能模块」对变更文件分组

   **提交类型（中文带 emoji）**：
   - 📚 文档：文档更新
   - ✨ 功能：新增功能
   - 🐛 修复：Bug 修复
   - ♻️ 重构：代码重构
   - 🚀 优化：性能/体验提升
   - 🔧 杂务：其他小改动

   **Commit 模板**：
   标题：<emoji> <类型>: <简洁描述>
   详情：
   • 📝 概述：<一句话描述改动核心>
   • 📂 文件：<涉及文件>
   • ✏️ 详情：<每个文件的具体改动>
   • 🎯 影响：<功能影响、用户体验变化>
   • ✅ 验证：<如何验证改动有效>
   • 🔗 关联：<关联的任务/文档，可选>
   • 💡 备注：<其他补充说明，可选>

7. 对每个功能分组分别执行 git add && git commit
8. 所有 commit 完成后统一 git push
9. 输出每个 commit 的标题（或说明无变更）

注意：
- 不预设任何分组策略，由 AI 实时判断
- 详情描述要具体、清晰、能看出核心改动

prompt.sh 
智能分析 Git 变更并生成有意义的 commit：

工作目录：/mnt/d/2Study/StudyNotes/.db

流程：
1. cd /mnt/d/2Study/StudyNotes/.db
2. 确保 git remote 使用 HTTPS + token
3. git pull --rebase origin master
4. git diff --name-only 列出所有变更文件
5. 如果没有变更文件，输出「无变更」并结束
6. 动态发现目录结构：执行 ls -la 和 find
7. 对每个变更的 .db 文件执行 dump
8. 读取 .sql 文件的 git diff，分析数据库内容变化
9. 根据 .sql diff 中出现的 media 路径，动态关联 media 文件
10. git add 所有相关文件（包括 .db、.sql、关联的 media）

   **提交类型（中文带 emoji）**：
   - 📊 数据：数据库记录变化（可含 media）
   - 🖼️ 素材：仅 media 文件变化
   - 📚 文档：文档更新
   - ✨ 功能：新增功能
   - 🐛 修复：Bug 修复
   - 🚀 优化：性能/体验提升

   **Commit 模板**：
   标题：<emoji> <类型>: <简洁描述>
   详情：
   • 📝 概述：<一句话描述改动核心>
   • 📂 文件：<涉及文件（含 .db、.sql、关联的 media）>
   • ✏️ 详情：<每个文件的具体改动，描述要具体、要有血有肉>
   • 🎯 影响：<功能影响、用户体验变化>
   • ✅ 验证：<如何验证改动有效>

11. 对每个功能分组分别执行 git add && git commit
12. 所有 commit 完成后统一 git push
13. 输出每个 commit 的标题（或说明无变更）

注意：
- 不预设任何目录结构，由 AI 动态发现
- .db 文件必须先 dump 到 .sql，再分析 diff
- 详情描述要具体、要有血有肉
