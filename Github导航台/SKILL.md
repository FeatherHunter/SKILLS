# Github导航台 · 技能文档

## 触发词

- `更新文档导航`
- `导航台`
- `docs 导航`

---

## 核心功能

### 1. 扫描 docs/ 目录

返回：
- 所有直接子目录名称 + 每个子目录的 HTML 文件数量
- 根目录下的独立 HTML 文件

### 2. 生成入口页面

- 输出到 `docs/index.html`
- 用户访问 GitHub Pages 时看到的就是这个页面

### 3. Git 联动

- 生成后对比 git 有无变化
- 有变化 → commit + push
- 无变化 → 报告「已是最新」

---

## HTML 生成规则（必须遵守）

### 规则1：目录层级浏览

显示 docs/ 的所有直接子目录，每个子目录显示文件数量。

### 规则2：文件列表

点击任意子目录 → 显示该子目录下的所有 HTML 文件。每个文件可点击打开。

### 规则3：默认状态

页面加载时显示全部子目录汇总视图，用户点击后进入筛选视图。

### 规则4：纯静态

无需后端，纯 HTML + CSS + JS，双击可打开，部署 GitHub Pages 可访问。

### 规则5：必须使用tast-skill 和 ui-ux-pro-max-skill 这两个技能

### 规则6：必须使用superpowers 技能 写代码

### 规则7：顶级视觉风格，按照该规则实现HTML

野兽派粗野主义
> 一张前卫的野兽派风格网站首页，UI/UX设计。刻意粗糙的HTML原生感布局，巨大的等宽字体标题以不对称角度打破网格。采用混凝土灰与未加工的深炭色为主调，点缀以刺眼的警告红和故意错位的边框线。包含鼠标悬停时的毛刺故障效果和原始的下划线链接。整体氛围反叛、激进、反设计、不容妥协。8K，UI设计，Brutalist website，Awwwards实验风格。

## 工作流程

```
触发「更新文档导航」
    ↓
扫描 docs/ 目录结构
    ↓
生成或更新 index.html（有变化才写文件）
    ↓
有变化 → commit + push
无变化 → 报告「已是最新」
    ↓
报告结果
```

---

## 错误处理

| 情况 | 处理 |
|------|------|
| docs/ 为空 | 正常生成，带空状态提示 |
| push 失败 | 报错，本地文件保留 |
| 无写入权限 | 报错并告知 |
