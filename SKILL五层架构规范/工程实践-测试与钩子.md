## 2. 处理一个 skill 时的思考清单（AGENT PROMPT）

**当你在 SKILLS/ 下处理任何 skill 时，按以下顺序自检**：

```
□ 该 skill 是否有 tests/ 目录？
    ├─ ✗ 无 → 建议添加最小可行测试套件（参考下方模板）
    └─ ✓ 有 → 跑 `cd <skill> && python -m pytest tests/ -v` 验证通过

□ 修改后是否同步更新了 <skill>.html 镜像？
    （按各 skill SKILL.md L27 "HTML 同步规范" 要求）

□ 是否触及了硬规则 / 唤醒词 / CLI？
    ├─ 是 → 同步 SKILL.md + <skill>.html + features/
    └─ 否 → 仅代码改动，pre-commit hook 自动验证

□ 是否添加了新 skill 或新功能模块？
    ├─ 是 → 在该 skill 的 SKILL.md 第一段明确"管什么、不管什么"
    └─ 否 → 跳过

□ commit 前确认：
    ├─ ☐ 无 `git commit --no-verify`（硬规则禁止）
    ├─ ☐ 无 git 库直调（gitpython / simple-git）
    └─ ☐ pre-commit hook 已跑（49 用例通过）
```

## 4. 添加 tests/ 的最小可行模板

参考 `居家管家/tests/` 结构（4 文件 ≈ 30 分钟搭好）：

```
<skill>/tests/
├── conftest.py        # 添加 scripts/ 到 sys.path + 共享 fixture
├── test_validators.py # 硬规则单元测试（不依赖 DB）
├── test_render.py     # 模板生成测试（如有 HTML 模板）
└── test_payloads.py   # payload 函数结构测试（需 DB）
```

### pytest.ini（最小）

```ini
[pytest]
testpaths = tests
python_files = test_*.py
addopts = -v --tb=short
```

### 安装 pytest

```bash
pip install pytest
```

### 验证

```bash
cd <skill> && python -m pytest tests/ -v
```

## 7. 已知边界（用户决策）

- ✗ **不做 web 端 CI**（GitHub Actions / GitLab CI）
- ✗ **不做 web merge 防护**（branch protection / required check）
- ✗ **不做 SDK 防护**（依赖 SKILL.md 硬规则 + AI 自约束）
- ✓ **本地 hook + SKILL.md 硬规则**是当前完整防护

接受的风险：
- Web 端绕过（用户决策忽略）
- `--no-verify`（依赖 AI 自约束 + commit message HOTFIX 约定）
- Git 库直调（依赖 AI 自约束）
