# 01 · 第一性原理

> 1 段话 + 1 张图。读完后你应该能向任何人 30 秒内讲清"Skill 是什么"。

## Skill 是什么

**Skill = 领域 + 工具 + 文档 + 规则**。一个 Skill 定义一整个领域(如"居家管家"= 家庭物品领域),包含该领域的所有:

- **实体** — 物品 / 分类 / 位置 / 标签
- **操作** — 增删改查 / 统计 / 联动
- **规则** — 硬规则(系统强制)+ 软规则(AI 自觉)
- **接口** — CLI / JSON / 错误码
- **文档** — SKILL.md / HTML 镜像 / references

> 没有 Skill 的 Agent = 空有厨艺没有工具的厨师。

## 为什么分层

不分层的 Skill 短期能跑,分层的 Skill 长期能活。"层"的作用只有一个:**让你写代码时知道往哪放**。

## 5 层结构图

```
              ⑤ 文档层  ←── AI / 人读
                ↑↓
              ④ 接口层  ←── CLI / JSON
                ↑↓
              ③ 规则层  ←── validators.py + references/
                ↑↓
              ② 操作层  ←── *_ops.py (本地) + integrations/ (对外)
                ↑↓
              ① 数据层  ←── db.py + SQLite
```

## 5 层职责

| 层 | 职责 | 落点 |
|---|---|---|
| ① 数据层 | 表 / 索引 / 约束 / 事务 / 迁移 / 备份 | `db.py` |
| ② 操作层(本地) | 原子化 CLI 子命令 | `*_ops.py` |
| ② 操作层(对外) | 外部服务调用 | `integrations/` |
| ③ 规则层 | 硬规则系统校验 + 软规则 AI 自觉 | `validators.py` + `references/` |
| ④ 接口层 | argparse + `{status, data, message}` | CLI 入口 |
| ⑤ 文档层 | SKILL.md + HTML + references/ | `.md` + `.html` |

## 规模伸缩

不是所有 Skill 都需要完整 5 层。**按规模伸缩**,避免过度工程。

| 规模 | 标准 | 方案 |
|---|---|---|
| 微型 | ≤ 1 个 .py,≤ 200 行 | **3 层合并**(文档 / 接口 / 操作+数据 写一起) |
| 中型 | 2-5 个 .py | **5 层完整(推荐)** |
| 大型 | 5+ 个 .py 或多人协作 | 5 层 + 对外操作独立子模块 |

**判断标准**:1 周内能写完用微型;1 个月内能写完用中型;多人协作 / 长期维护用大型。
