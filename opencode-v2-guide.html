一、V1 vs V2 核心差异

1. 三大破坏性变更

其余功能保持兼容——V1 的配置文件会被 V2 在内存中自动翻译，无需强制迁移。

维度V1V2
可执行文件opencodeopencode2 并存，V1 仍可用
插件 API旧 API新 Effect 插件 API 必须重写
服务端 API旧契约新契约 必须迁移
TUI 配置tui.json(c) 分层多文件单一全局 ~/.config/opencode/cli.json，首次启动自动迁移

兼容策略： V1 的 opencode.json(c) / .opencode/ 配置会被 V2 在内存中翻译，不强制迁移。如果你只想"先试试"，直接装 opencode2 就能跑。

新 V2 权限写法示例（合并到一个数组里）：

json 
{
  "permissions": [
    { "action": "shell",     "resource": "git push *", "effect": "ask"   },
    { "action": "edit",      "resource": "*",          "effect": "allow" },
    { "action": "websearch", "resource": "*",          "effect": "deny"  }
  ]
}

三、V2 优化 SKILL 可利用的特性

description 控制曝光

frontmatter 写清 description，没描述的 skill 不会被广告给模型，等于隐身。

autoinvoke 屏蔽

metadata.opencode/autoinvoke: false 从模型可选列表移除，但 ID 仍可显式激活。

slash 控制

slash: false 不出现在 / 命令面板，但模型仍可见。

HTTP 远程目录

配置 skills: ["https://..."] 提供 index.json，version 字段触发缓存刷新。

优先级覆盖

项目级 .opencode/skills > ~/.config/opencode/skills > 全局 compat，后注册同名 skill 覆盖前者。

skill 权限门

{ action: "skill", resource: "internal-*", effect: "deny" }，按 ID 精细控制谁能加载。

自动挂载为 /<id>

CLI 中 /<id> 直接激活——调试 skill 极方便。

目录式布局

.opencode/skills/<id>/SKILL.md + scripts/ + references/，路径相对 SKILL.md。

支持文件不自动加载

skill 必须显式指示 agent 去 read references/*.md，否则它们只是装饰。

优化 SKILL 实战清单

  1. ID 命名：kebab-case 1–64 字符，与目录名一致；ID 来自路径，不是 frontmatter name
  2. description 是关键：用一句话写清"何时调用"，否则模型永远不会用
  3. 保护内部 skill：用 action: skill + deny 防止主代理误触发
  4. 调试优先级冲突：用 /skills 列表查重复 ID

四、V2 代码开发可利用的特性

1. 内置代理（按 Tab 切换）

AgentMode用途
buildprimary默认编码代理，工具默认允许
planprimary只读计划代理，编辑仅允许 plan 文件
generalsubagent多步研究/通用任务，不能再启子代理
exploresubagent只读探索（read/glob/grep/webfetch/websearch）

TUI 切换方式：Tab / Shift+Tab 循环，或 /agents 选择。compaction/title/summary 是隐藏的系统维护代理。

2. 自定义代理（项目级 Markdown）

放 .opencode/agents/<name>.md，frontmatter 支持 description / mode / model / system / permissions / steps / color / hidden。

markdown 
---
description: 审查改动，不修改文件
mode: subagent
model: anthropic/claude-sonnet-4-5#high
color: warning
steps: 8
permissions:
  - { action: edit,  resource: "*", effect: deny }
  - { action: shell, resource: "*", effect: deny }
---
按严重性列出 bug、安全、回归、缺测，含 file:line 引用。

子代理调用方式：

  • 工具调用 subagent（foreground / background）
  • 对话中 @explore 找认证错误处理位置 触发 @ 提及
  • 父代理通过 subagent 权限决定可启哪些子代理

3. 自定义斜杠命令（极强大）

放 .opencode/commands/<name>.md：

markdown 
---
description: 审查指定代码
agent: plan
model: anthropic/claude-sonnet-4-5#high
---
审查 $ARGUMENTS：先报 bug，再列缺测。

$ARGUMENTS

全量参数。

$1 / $2

位置参数，引号合并空格。

Shell 插值

`` !`git diff` `` 在 prompt 提交前替换为命令输出，绕开工具权限，慎用。

嵌套目录

.opencode/commands/team/review.md → /team/review。

4. 权限系统（必须掌握）

ordered 数组，最后匹配规则胜出：

json 
{
  "permissions": [
    { "action": "subagent", "resource": "*",       "effect": "deny"  },
    { "action": "subagent", "resource": "explore", "effect": "allow" },
    { "action": "shell",    "resource": "git *",   "effect": "ask"   }
  ]
}

必看的动作名变化：

V1V2
bashshell
tasksubagent
write / patchedit（合并）
read / webfetch / skill …同名保留

5. 其他开发利器

Snapshots / Undo

snapshots: false 关闭。试错回滚。

LSP + Formatters

配置 formatter / lsp。实时类型检查 + 自动格式化。

AGENTS.md

项目级系统指令，instructions，支持全局+项目多级。

References

按需挂载上下文目录，避免污染主 prompt。

MCP servers

mcp.servers 接 Playwright 等外部工具。

Compaction

compaction.keep.tokens / buffer 长会话自动压缩。

Attachments

会话中上传文件。

Session warming

保持模型连接热度。

Session sharing

share: auto/manual/disabled 分享会话链接。

五、立即可用的命令

你已经会 /sessions 和 /rename 了，下面这些大概率没用过：

命令作用
opencode2 service status查看后台服务健康
opencode2 service restart重启后台服务
opencode2 api get /api/health验证服务健康
opencode2 --standalone不用共享服务，隔离调试

opencode2 api get /api/agent/v2-agent-list列出所有 agent

TUI 中 Tab / Shift+Tab切换主代理

六、最小行动清单

优化 SKILL：

  1. 每个 SKILL.md 必写 description
  2. 内部 skill 加 autoinvoke: false + skill 权限 deny

代码工作流：

  1. plan 用于设计，explore 用于调研，build 用于实现——三者切换比单代理高效得多
  2. 所有项目起手写 ordered permissions，把 git push / webfetch / shell 默认 ask

效率提升：

  1. 高频工作流做成 .opencode/commands/*.md，用 $ARGUMENTS + shell 插值串起来
  2. 开 snapshots 默认值；用 /undo 快速回滚；LSP/Formatter 接进 opencode.jsonc

调试顺序： 配置不生效时，依次查 ~/.config/opencode/cli.json → .opencode/ 优先级 → description 是否写了 → autoinvoke 是否被关 → agent 的 skill 权限 → 用 opencode2 api 列已发现项。
