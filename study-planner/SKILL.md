# Study Planner

你是最强大脑，以最强大脑执行该任务。

制定高效学习计划，数据驱动，简洁执行。

工作目录（绝对路径）：
- Windows: `D:\2Study\StudyNotes`
- WSL: `/mnt/d/2Study/StudyNotes`

## 安装
触发时检查**当前AI智能体**的 skills 目录：
- 不存在 `skills/study-planner/SKILL.md` → 创建SKILL文件(桥接器文件)，SKILL文件内容指向本文件
- 桥接器只符合SKILL规范且包含SKILL触发条件，和SKILL本体文件路径

## 双目录保存规则

### 规则

1. **主存储**：`study-planner/plans/` 是唯一真实数据源，所有操作以此为准
2. **备份存储**：`个人/` 目录下的副本与主存储内容完全一致，不做差异化修改
3. **写入时机**：生成或优化任何计划时，**同时**写入主存储和备份路径
4. **周一日期计算**：周计划以 ISO 周为准，周一为该周第一天
   - 例如：2026年第21周（周一4/20）→ 周计划备份到 `个人/20260420/{YYYYWW}_周计划.md`

### 写入逻辑

```
生成计划 → 写入 plans/ → 判断类型 → 写入个人目录备份路径
```

- 生成百日计划 → 写入 `plans/...` → 额外写入 `个人/{YYYYMMDD}_百日计划.md`
- 生成月计划 → 写入 `plans/...` → 额外写入 `个人/{YYYYMM}/{YYYYMM}_月计划.md`
- 生成周计划 → 写入 `plans/...` → 额外写入 `个人/{周一}/`
- 生成今日计划 → 写入 `plans/...` → 额外写入 `个人/{YYYYMMDD}/{YYYYMMDD}_今日计划.md`

## 触发方式
### 方式一：cron 触发（每日 08:00）
- 检测 cron 任务，不存在则创建：`0 8 * * *` 执行 `skill study-planner`
- 按序检查：
  1. 无长期计划 → 科学分析JD文件 + learning-system进度 → 生成长期计划
  2. 无当月计划 → 从长期计划分解 → 生成当月计划
  3. 有当月计划 → **优化当月计划**（详见「月计划优化」章节）
  4. 无本周计划 → 从当月计划分解 + learning-system上周完成度 → 生成本周计划
  5. 有本周计划 → **优化本周计划**（详见「周计划优化」章节）
  6. 生成今日计划（详见「今日计划生成」章节）

### 方式二：手动触发
- "制定计划"、"生成百日计划"、"优化计划"
- 按用户需求生成对应粒度的计划

## 计划生成

### 长期大计划
- 用户指定周期（半年/100天）
- 科学分析 JD 文件提取技能需求（读取 `jd/` 下所有 md 文件，统计词频）
- 结合 learning-system 进度 + 用户采访结果生成计划
- 保存为 `plans/{YYYYMMDD}_百日计划.md`

### 强制前置：理解用户真实情况（长期计划必做）
生成长期计划前，**必须通过对话采访用户**，收集以下核心信息：

| 问题 | 目的 |
|------|------|
| 每天真实可用的学习时间（工作日/周末） | 避免按"4h/天"虚估，导致计划崩塌 |
| 最近的真实项目经历（技术细节、用户规模、个人贡献） | 面试被问项目时的底牌 |
| 最大的技术短板是什么 | 计划优先补的缺口 |
| 有没有面试经验 | 影响"面试冲刺"阶段的起点 |

采访示例：
> "每天真实能投入多少小时？工作日和周末分开算"
> "最近参与的真实项目，能讲讲技术细节吗？用什么协议/架构/规模多大"
> "你觉得面试时最容易被问住的是哪块"

**原则：不采访就不生成计划**。对话记录可存档保存到 `个人/` 目录。

## 月计划优化

**触发**：cron 08:00 检测到已有当月计划

**优化逻辑**：
1. 读取百日计划当前阶段目标，对照当月计划完成度
2. 读取过去一个月的 daily-schedule 作息报告，看实际投入了多少时间
3. 读取 learning-system 月进度，看完成了多少
4. 判断：当月目标是否合理？进度超前/滞后多少？
5. 如果滞后 > 20% → 调整下月目标，或顺延未完成的任务
6. 如果超前 → 评估是否可以加速或补充额外目标
7. 月计划优化后同步更新到百日计划的阶段进度

**输出**：生成调整说明（为什么改、改了什么），让用户知道变化原因

## 周计划优化

**触发**：cron 08:00 检测到已有本周计划

**优化逻辑**：
1. 读取当月计划本周任务，对照完成度
2. 读取 daily-schedule 本周每日作息报告，看今天之前实际投入了多少小时
3. 读取 learning-system 本周进度，看昨天完成了什么
4. 判断：本周目标是否还能达成？
5. 如果进度滞后 → 调整本周剩余任务，优先核心内容（可删减次要任务）
6. 如果进度超前 → 评估是否补充额外内容，或留出复习时间
7. 周计划优化后同步更新到当月计划的周进度

## 今日计划生成

### 数据源使用规则
1. daily-schedule 过去2周每周总结 → 了解用户实际能投入多少时间、哪天容易摆烂
2. daily-schedule 上次周总结到昨天的每日作息报告 → 了解近期状态趋势（越来越累？还是稳定？）
3. learning-system 昨日完成度 → 昨天学了啥、完成了多少
4. learning-system 整体进度 → 当前在哪个知识点、在整个学习路径的什么位置
5. 周几/节假日 → 调整任务量（周末可以多一点）

### 强制约束

1. **报告必须严格符合「今日计划格式模板」**，结构缺失视为不合格
2. 以最强大脑执行：不受限于任何预设规则，以最聪明的方式安排任务
3. 计划生成后，以最强大脑视角审视并修正一遍。
4. **作息时间表必须安排：**
   - **修行*：元母意程，至少 20 分钟
   - **锻炼**：任何形式，至少 20 分钟
5. 多用emoji表情

### 今日计划格式模板

```markdown

# {YYYY-MM-DD} 今日学习计划

> 生成时间：{HH:MM}
> 数据来源：周计划({YYYYWW}) + learning-system进度 + daily-schedule作息报告({MMDD})

---

> - 不套用"标准作息"，基于真实数据推断
```

## JD 处理

**触发场景**：用户说"JD截图已准备好了"

### 处理流程

1. 用户发送 JD 截图
2. 分析截图提取信息（公司、职位、薪资、技能要求等）
3. 保存截图至 `jd/images/{YYYYMMDD}_{公司简称}.jpg`
4. 生成 md 记录关键信息
5. 保存到 `jd/{YYYYMMDD}_{公司简称}.md`

### JD目录

```
jd/
├── {YYYYMMDD}_{公司简称}.md    # JD记录
└── images/                    # 截图
```

**生成计划时**：科学分析 `jd/` 目录下所有 md 文件
