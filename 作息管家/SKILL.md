## 功能入口

> 触发条件：用户提到"作息"、"作息记录"、"同步作息"、"总结作息"等

### 1. 同步作息表 / 更新作息表

**触发词**：
- "同步作息"
- "更新作息表"
- "同步作息表"
- "把今天的作息同步一下"

**执行流程**：
```
1. 调用 prepare-messages 获取消息
2. AI 分析消息 → 生成细粒度记录
3. AI 补全空白（category='未知'）
4. 调用 add_record_full() 逐条写入
5. 保证首尾相接
```

**连续性要求**：
- 记录必须 00:00-23:59 全覆盖
- 每条记录的 time_start = 上一条 time_end
- 空白时间段必须填充 `category=未知`

**粒度要求**：
- 按活动切换点细分
- 宁可多条记录，不要合并

> 详细规范见: `references/同步流程.md`

---

### 2. 总结某日作息

**触发词**：
- "总结作息"
- "生成摘要"
- "作息摘要"
- "今天作息怎么样"

**执行条件**：
- ⚠️ 该日期的 `schedule_records` 必须已填满 **24小时**
- 未满24小时的日期**不允许**生成 summary

**执行流程**：
```
1. 查询该日期的 schedule_records
2. 验证是否覆盖 00:00-23:59
3. 按 category 分组合并 duration_minutes
4. 调用 add_summary_full() 写入摘要
```

**输出格式**：
```
📊 2026-05-22 作息摘要
==================================================
  😴 睡眠: 7h
  💼 工作: 4h
  📚 学习: 0h
  🏋️ 运动: 0h
  🚴 通勤: 36m
  🍽️ 餐饮: 1h
  🎮 娱乐: 1h
  ❓ 未知: 1h

  总计: 24h ✓
```

> 详细规范见: `references/数据库结构.md`

### 4. 查询计划作息（新增）

**触发词**：
- "查计划"
- "今天计划是什么"
- "这周的计划"
- "查询计划作息"

**调用 CLI**：
```bash
python3 scripts/schedule_cli.py query-plans <日期1,日期2,...>
# 示例：查询单天
python3 scripts/schedule_cli.py query-plans 2026-05-22
# 示例：查询多天
python3 scripts/schedule_cli.py query-plans 2026-05-20,2026-05-21,2026-05-22
```

**输出示例**：
```
============================================================
📅 2026-05-22 计划作息
============================================================
  00:00 - 01:00  睡觉
  01:00 - 02:00  睡觉
  ...
  08:00 - 09:00  30min通勤(骑车)+30min工作
  09:00 - 10:00  40min工作(写代码)+20min会议
  ...
```

---

### 5. 新建/更新计划作息（新增）

**触发词**：
- "新建计划"
- "更新计划"
- "插入计划"
- "设置今日计划"

**调用 CLI**：
```bash
# 简单方式（只填非空小时）
python3 scripts/schedule_cli.py upsert-plan <日期> "睡觉" "" "" "" "" "" "" "通勤+工作" "工作"

# JSON方式
python3 scripts/schedule_cli.py upsert-plan <日期> --json '{"hour_0": "睡觉", "hour_8": "30min通勤+30min工作", "hour_9": "40min工作+20min摸鱼"}'
```

**效果**：
- 如果日期不存在 → **插入**新计划
- 如果日期已存在 → **更新**计划（保留未填的小时）
- 不提供删除接口

## 接口清单（简览）

### schedule_records 操作

| 接口 | 类型 | 说明 |
|------|------|------|
| `add_record_full(...)` | 增加 | 9字段全校验 |
| `update_record(id, **kwargs)` | 更新 | 按id定位 |
| `get_records_by_date(date)` | 查询 | 按日期查 |

### daily_summary 操作

| 接口 | 类型 | 说明 |
|------|------|------|
| `add_summary_full(...)` | 增加 | 9字段全校验，满24h才可调用 |
| `update_summary(date, **kwargs)` | 更新 | 按date定位 |
| `get_daily_summary(date)` | 查询 | 按日期查 |

### schedule_plans 操作

| 接口 | 类型 | 说明 |
|------|------|------|
| `upsert_plan(date, hour_plans)` | 插入/更新 | 存在则更新，不存在则插入 |
| `get_plan(date)` | 查询 | 获取单日计划 |
| `get_plans(dates)` | 查询 | 获取多日计划（逗号分隔） |

### 游标查询
