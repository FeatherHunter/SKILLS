# 操作规范（强制）

> 本文件为作息管家技能的核心约束，所有 AI 和脚本必须严格遵守。

---

## 强制规则

### 1. 严禁直连数据库

**所有数据操作必须通过 CLI 接口，禁止直接连接数据库进行读写。**

- ✅ 正确：`python schedule_cli.py query-plans 2026-05-22` → 分析 → `upsert_plan()`
- ❌ 错误：直接用 sqlite3 连接数据库写入

### 2. 增加操作全字段校验

**增加接口必须传入全部字段，少传一个字段直接报错。**

```python
# add_record_full 必填9字段
add_record_full(date, time_start, time_end, duration_minutes, 
                activity, category, source_contents, 
                source_timestamps, analysis_reasoning)

# add_summary_full 必填9字段
add_summary_full(date, total_sleep_minutes, total_work_minutes,
                total_exercise_minutes, total_commute_minutes,
                total_eating_minutes, total_learning_minutes,
                total_entertainment_minutes, total_unknown_minutes)
```

### 3. 禁止删除操作

**数据库中不得出现任何 DELETE 语句，禁止删除记录。**

- ❌ 禁止：`DELETE FROM schedule_records WHERE ...`
- ✅ 正确：只能增加（INSERT）和更新（UPDATE）

### 4. 游标机制

**靠 schedule_records 表中最后一条记录的时间定位，不需要额外的 checkpoint 表。**

- 游标 = `SELECT ... ORDER BY date DESC, time_end DESC LIMIT 1`
- 下次同步从游标位置继续

---

## 违反规范的后果

违反以上任何一条规范，将导致：
1. 数据不一致
2. 同步逻辑混乱
3. 与其他技能联动失败
