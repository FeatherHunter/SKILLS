# 睡眠归属规则

> 本文件定义睡眠记录的归属逻辑。

---

## 核心规则

**睡眠记录归属于「就寝那天」，不是起床日。**

---

## 规则说明

| 场景 | 记录日期 | 示例 |
|------|---------|------|
| 23:00 睡，次日 07:00 起 | 就寝日（23:00那天） | 5月20日23:00睡 → 记录在5月20日 |
| 凌晨 01:00 睡，次日 08:00 起 | 就寝日（01:00那天） | 5月20日01:00睡 → 记录在5月20日 |
| 凌晨 03:00 睡，次日 09:00 起 | 就寝日（03:00那天） | 5月20日03:00睡 → 记录在5月20日 |

---

## 为什么这样设计

### 1. 符合生活习惯

人们通常说"昨晚几点睡"，而不是"今天早上几点醒"。

```
✅ 正确理解：
  "我昨晚1点睡的" → 睡眠归属于昨晚（就寝日）

❌ 错误理解：
  "我今早7点醒的" → 不应该把睡眠归属于今天
```

### 2. 保证数据连续性

如果按起床日计算，会出现：
- 5月20日的睡眠记录在5月21日
- 5月21日的睡眠记录在5月22日
- 导致日期碎片化

按就寝日计算：
- 所有睡眠记录都归属于同一个自然日
- 便于统计每日睡眠总时长

---

## 实际示例

### 场景1：正常作息

```
5月20日 23:00 → 5月21日 07:00 睡觉
睡眠记录：
  - date: 2026-05-20
  - time_start: 23:00
  - time_end: 07:00（次日）
  - duration_minutes: 480（8小时）
```

### 场景2：晚睡

```
5月20日 01:30 → 5月20日 08:00 睡觉（跨日但就寝仍是20日）
睡眠记录：
  - date: 2026-05-20
  - time_start: 01:30
  - time_end: 08:00
  - duration_minutes: 390（6.5小时）
```

### 场景3：熬夜到凌晨

```
5月20日 23:00 → 5月21日 04:00 睡觉
睡眠记录：
  - date: 2026-05-20
  - time_start: 23:00
  - time_end: 04:00（次日）
  - duration_minutes: 300（5小时）
```

---

## 睡眠与作息表的关系

睡眠记录和其他活动记录一样，都存储在 `schedule_records` 表中：

```
2026-05-20 作息记录：
  23:00-01:30 工作（加班）
  01:30-08:00 睡眠 ← 跨日但归属就寝日
  08:00-08:30 洗漱
  08:30-09:00 通勤
  ...
```

---

## 注意事项

1. **就寝时间可能在凌晨**（如1点、2点、3点），但仍然归属于昨天
2. **起床时间是次日**，但 sleep 字段 duration 计算要包含跨日部分
3. **连续性**：睡眠记录的 time_end 是下一条记录的 time_start
