# Cron 任务 - 作息管家配套

> 本文件列出与作息管家技能配合使用的最佳 cron 任务设计。
> 实际部署时需填入具体的 Cron ID、路径、推送地址等。

---

## Cron 1：定时同步作息表

| 项目 | 内容 |
|------|------|
| **名称** | 作息管家-定时同步 |
| **调度** | `0 */3 * * *`（每3小时整点） |
| **时区** | Asia/Shanghai |
| **会话** | isolated |
| **推送** | 静默（not requested） |
| **超时** | 300 秒 |

**执行内容：**
```
请完整加载「作息管家」技能（读取 <技能目录>/SKILL.md + references/同步流程.md + references/分类清单.md）。

本 Cron 任务执行模块：「定时同步作息表」——调用 prepare-messages 获取游标到现在的消息，分析消息生成细粒度作息记录，调用 add_record_full() 逐条写入数据库。

完成后：严格按照作息管家的要求，以最严格标准检查生成的每一条记录。

静默执行，不需要推送任何消息。
```

---

## Cron 2：每日作息报告（07:30）

| 项目 | 内容 |
|------|------|
| **名称** | 作息管家-每日报告 |
| **调度** | `30 7 * * *`（每天 07:30） |
| **时区** | Asia/Shanghai |
| **会话** | isolated |
| **推送** | WeChat（openclaw-weixin） |
| **超时** | 600 秒 |

**执行内容：**
```
请完整加载「作息管家」技能（读取 <技能目录>/SKILL.md + references/同步流程.md）。

本 Cron 任务执行模块：「每日作息报告」——读取昨日 schedule_records，验证是否满24小时，调用 add_summary() 写入 daily_summary，生成 HTML 作息报告并发送给用户。

完成后：严格按照作息管家的要求，以最严格标准检查生成的报告。
```

---

## 设计原则

| 项目 | 说明 |
|------|------|
| **时间连续性** | Cron 1 写入时必须保证记录首尾相接 |
| **粒度控制** | 每次活动切换即切割 block，不为省记录数而合并；宁可多条记录，不要漏记活动切换点 |
| **睡眠归属** | 睡眠记录归属于就寝日，不归属于起床日 |
| **禁止删除** | 两个 cron 都不执行任何 DELETE 操作 |
| **静默同步** | Cron 1 只写数据库，不推送 |
| **警告机制** | Cron 2 发现不满24小时先警告再决定是否生成报告 |
