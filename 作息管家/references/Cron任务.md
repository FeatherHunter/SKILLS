# Cron 任务 - 作息管家配套

> 本文件列出与作息管家技能配合使用的最佳 cron 任务设计。
> 实际部署时需填入具体的 Cron ID、路径、推送地址等。

---

## Cron 1：定时同步作息表

| 项目 | 内容 |
|------|------|
| **名称** | 作息管家-定时同步 |
| **调度** | `0 */3 * * *`（每3小时整点） |
| **时区** | Asia/Shanghai |
| **会话** | isolated |
| **推送** | 静默（not requested） |
| **超时** | 300 秒 |

**执行内容：**
```
请读取 <技能目录>/SKILL.md 并严格按照其中「推荐 Cron 任务」的设计执行「定时同步作息表」：

1. 调用 prepare-messages 获取游标到现在的消息
2. AI 分析消息 → 生成细粒度记录
3. **【重要】严格按日期分组**：不同日期的消息不得互相参考上下文
4. **前5后5推断**：以每个时间段的「开始时间点」为中心，取其之前5条消息分析；以「结束时间点」为中心，取其之后5条消息（若存在）分析，综合推断该时间段的合理活动类型。严格按日期分组，不同日期的消息不得互相参考。
5. 调用 add_record_full() 逐条写入数据库
6. 保证记录首尾相接（time_start = 上一条 time_end）
7. **禁止将无消息的将来时段补齐**：最后一条消息之后到当前时刻之间的空白不写入任何记录

完成后静默执行，不需要推送任何消息。
```

## Cron 2：每日作息报告（07:30）

| 项目 | 内容 |
|------|------|
| **名称** | 作息管家-每日报告 |
| **调度** | `30 7 * * *`（每天 07:30） |
| **时区** | Asia/Shanghai |
| **会话** | isolated |
| **推送** | WeChat（openclaw-weixin） |
| **超时** | 600 秒 |

**执行内容：**
```
请读取 <技能目录>/SKILL.md 并严格按照其中「推荐 Cron 任务」的设计执行：

1. 读取昨日（date - 1）的 schedule_records
2. 验证是否覆盖 00:00-23:59（满24小时）
3. 如果不满24小时 → 通知用户「当日作息记录不满24小时，无法生成摘要，请先补全记录」，停止执行
4. 如果满24小时 →
   a. 调用 add_summary() 按分类写入 daily_summary（KV结构）
   b. 生成 HTML 作息报告（禁止emoji，可用SVG，活泼生动）
   c. 通过 WeChat 发送报告给用户

报告要求：
- 无 emoji 表情
- 使用 SVG 绘制图表（如时间轴、分类饼图）
- 页面生动形象
- 包含昨日各时段活动详情 + 分类汇总
```

## 设计原则

| 项目 | 说明 |
|------|------|
| **时间连续性** | Cron 1 写入时必须保证记录首尾相接 |
| **粒度控制** | 每次活动切换即切割 block，不为省记录数而合并；宁可多条记录，不要漏记活动切换点 |
| **睡眠归属** | 睡眠记录归属于就寝日，不归属于起床日 |
| **禁止删除** | 两个 cron 都不执行任何 DELETE 操作 |
| **静默同步** | Cron 1 只写数据库，不推送 |
| **警告机制** | Cron 2 发现不满24小时先警告再决定是否生成报告 |
