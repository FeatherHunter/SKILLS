## 心法（4 条核心规则）

### 1. **一句话可能暗示 N 件事 → 拆 N 段，每段归类**

用户一句话里可能说了连续做的几件事。**按动作切换切段**，每段单独归类。

- 例："剪指甲 + 戴隐形眼镜 + 弄护理液" = 3 段，但都可归 `维持.洗漱`（category 一致，activity 文字记具体动作）
- 例："贴好面膜一边看剧" = 贴面膜（30 秒瞬时）+ 看剧为主 → 整段 `调整.追剧`
- 例："刚才到现在都在买菜，还在回家路上" = 买菜 + 通勤 2 段

### 2. **模糊兜底**：真无法拆解时 → 选最相似分类

但**兜底率应 < 15%**。如果某天兜底率过高，要回去看用户原话。

- 例："躺床上了" 没说干啥 → `调整.过渡`
- 例："处理杂事" → `日常.杂事`
- 例："送好了" 没说送完干啥 → 默认推断 `日常.过渡`

### 3. **意图 vs 实际**：按当前段已说出的话判，不判打算

用户说"我准备做 X"是意图；用户说"我正在做 Y"是实际。**实际优先**。

- 例："我又困了再睡会"（意图）但 5 分钟后说"没睡着在玩手机"（实际）→ **实际为准**
- 例："准备睡觉"（意图）但实际段是视频对话 → 实际段 `投入.伴侣`

### 4. **AI 对话不入作息**：除非用户明确说"把对话归档进作息"

AI 之间的对话（用户和 AI 的闲聊、查询、归档请求）默认**不归类到作息**。除非用户明确说"把这条对话录进去"。

- 例：用户说"把我们聊天记录写到文件" → 归 `工作.AI调优`（用户主动指令）
- 例：用户问"今天怎么样" → **不归类**

### 5. **遇到白名单外活动 → 主动提议，不要兜底**

当一段活动**无法匹配任何现有白名单二级**时（典型例子："给香草浇水"没有现成分类），**不要静默用 `日常.杂事` 兜底**。

正确流程：
1. 先按心法 #1-4 判定一级（看目的）
2. 如果二级不在白名单 → 调 `propose-category --code "X.Y" --hint "场景"`
3. **等用户口头确认** → 调 `approve-category --code "X.Y"` 写入 YAML
4. 再写入作息记录

反模式：
- ❌ 静默改成"日常.杂事"（丢失信息）
- ❌ 编造一个新名字写入（绕过校验）
- ❌ 跳过这条不录（数据缺失）

> 申请制是分类系统的"自演化"机制 —— **白名单不预先膨胀，但遇到真实需求就扩展**。

## 默认归类表（常见歧义）

| 动作 | 默认 category | 备注 |
|---|---|---|
| 玩手机（无明确目的） | `调整.手机` | 看剧/学习有别的信号会覆盖 |
| 散步（短） | `维持.通勤` | 走路去某地 |
| 散步（长，健身目的） | `健康.运动` | 用户说"去散步锻炼"会覆盖 |
| 做饭（给自己/家人吃） | `维持.做饭` | 例外：研发菜谱 = `创作.菜谱` |
| 买东西 | `维持.采购` | 例外：为开店囤货 = `工作.财务`（语境） |
| 躺床上没说干啥 | `调整.过渡` | 后续有动作就并入 |
| 聊天 / 通电话 | `投入.沟通`（默认）| 看亲密程度可选 `投入.伴侣`/`投入.朋友`/`投入.家人` |

## 一级 vs 二级

- **一级**：8 个固定不变（维持/健康/工作/学习/创作/投入/调整/日常）
- **二级**：白名单里挑，初始 ~70 个，可申请扩充
- **不用三级**：`工作.AI调优.x` 不允许（会爆炸）

## 申请新分类流程

> 触发条件：心法 #5 —— 遇到白名单外活动时主动提议，不要兜底。

1. AI 录入时发现 category 不在白名单 → 校验报错
2. AI 输出："提议新增 `调整.散步`，原因是用户常去小区散步"
3. **用户口头说"可以"或"加"** → AI 调 `approve-category --code "调整.散步"`
4. 白名单 YAML 自动更新，下一条 add_record 立即可用

CLI 命令：
```bash
python scripts/schedule_cli.py list-categories [--level 1|2]
python scripts/schedule_cli.py propose-category --code "一级.二级" --hint "场景"
python scripts/schedule_cli.py approve-category --code "一级.二级"
```

**真实例子（来自本心法作者）**：
- 用户说"我刚才在给香草浇水"
- AI 心法推演：
  - 心法 #1: 1 段 1 动作"浇水"
  - 心法 #2-3: 不模糊、实际发生
  - 心法 #4: 不是 AI 对话
  - 心法 #5: "浇水" + "香草" 二级白名单无匹配
- AI 输出：💬 "提议新增 `日常.园艺` 或 `学习.园艺`（视目的），原因是用户种香草是常见活动"
- 用户说"为吃，入菜用" → AI 归 `维持.做饭`（种香草是做饭链路的一部分）
- 如果用户说"为研究怎么种" → AI 提议 `学习.园艺`，批准后再录入

## 校验失败时的处理

当 `add_record_full` 抛出 `ValueError("category 校验失败: ...")`：

1. AI **不要尝试绕过** —— 没有 `--skip-validation` 跳过通道
2. AI 把错误信息展示给用户（含：字段名 + 当前值 + 期望值 + 怎么修）
3. AI 提议走申请制流程（如上）
