# 接口清单

> 本文件列出所有可用的数据库操作接口。**严禁直连数据库，必须通过 CLI 调用。**

---

## schedule_records 操作接口

### 增加记录

```python
add_record_full(
    date,                  # 日期（YYYY-MM-DD）
    time_start,           # 开始时间（HH:MM）
    time_end,             # 结束时间（HH:MM）
    duration_minutes,     # 持续时长（分钟）
    activity,             # 活动描述
    category,             # 分类（AI自由填写，参考建议列表）
    source_contents,       # 消息原文
    source_timestamps,     # 消息时间
    analysis_reasoning     # AI分析推理
)
```
⚠️ **9个字段缺一不可，少传直接报错**

### 更新记录

```python
update_record(record_id, **kwargs)
# record_id: 必填，定位记录
# kwargs: 可选字段 date/time_start/time_end/duration_minutes/
#         activity/category/source_contents/source_timestamps/analysis_reasoning
```

### 查询接口

| 接口 | 说明 |
|------|------|
| `get_last_record_full()` | 获取最后一条完整记录 |
| `get_records_by_date(date_str)` | 获取指定日期所有记录 |
| `get_records_range(start_date, end_date)` | 获取日期范围内所有记录 |
| `has_records_for_date(date_str)` | 检查是否有记录 |

## daily_summary 操作接口

### 增加摘要（KV结构）

```python
add_summary(date, category, total_minutes)
# date: 日期（YYYY-MM-DD）
# category: 分类名（AI自由填写）
# total_minutes: 该分类总分钟数
```
⚠️ **3个字段缺一不可**

⚠️ **前提条件**：该日期的 schedule_records 必须已填满 24 小时

### 查询接口

| 接口 | 说明 |
|------|------|
| `get_daily_summary(date_str)` | 获取每日摘要，返回 `list of {category, total_minutes}` |
| `get_summaries_range(start_date, end_date)` | 获取日期范围摘要，返回 `list of {date, category, total_minutes}` |

## 游标查询接口（用于 AI 同步）

| 接口 | 说明 |
|------|------|
| `get_messages_for_sync(cursor_datetime_str, end_time_str=None)` | 获取游标前10条到指定时间的消息 |
| `get_messages_before(before_time_str, limit=10)` | 获取游标前 N 条消息 |
| `get_messages_from(from_time_str)` | 获取某时间后的所有消息 |

---

## schedule_plans 操作接口

| 接口 | 说明 |
|------|------|
| `upsert_plan(date, hour_plans)` | 新增或更新计划作息 |
| `get_plan(date)` | 获取单日计划 |
| `get_plans(dates)` | 获取多日计划（逗号分隔） |
| `search_plan_event(date, title)` | 按日期+标题查活跃计划事件，返回 dict 或 None（2026-07-12 新增） |
| `ensure_plan_event(date, time_start, time_end, title, notes, category)` | 缺则建，返回 `{"action":"found"/"created", "id":N}`（2026-07-12 新增） |
| `update_plan_event(event_id, fields)` | 单条修改，fields 可含 title/notes/category/time_start/time_end/completion/completion_note |

## 调用方式

**所有接口通过 CLI 调用，禁止直接 import schedule_db。**

```bash
# 查询记录
python scripts/schedule_cli.py list 2026-05-22

# 查询摘要
python scripts/schedule_cli.py summary 2026-05-22

# 查询计划
python scripts/schedule_cli.py query-plans 2026-05-22

# 轻量查询（2026-07-12 新增，供程序化调用）
python scripts/schedule_cli.py search-plan-event 2026-07-12 --title "健身"

# 补计划（2026-07-12 新增，单条追加，幂等）
python scripts/schedule_cli.py ensure-plan-event 2026-07-12 --time-start 17:00 --time-end 18:00 --title "健身" --notes "练背" --category 运动

# 标记完成情况（2026-07-12 新增）
python scripts/schedule_cli.py update-event 562 --completion "未完成" --completion-note "下雨没跑"

# 写入作息记录（由 AI 分析后调用 add_record_full）
# 写入摘要（由 AI 分析后调用 add_summary）
# 这两个接口通过 schedule_cli.py 中的 Python 函数调用，不提供独立 CLI 命令
```
