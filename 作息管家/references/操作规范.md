# 操作规范（强制 · 2026-06-29 更新）

> 本文件为作息管家技能的核心约束，所有 AI 和脚本必须严格遵守。

---

## 强制规则

### 1. 严禁直连数据库
**所有数据操作必须通过 CLI 接口，禁止直接连接数据库进行读写。**
- ✅ 正确：`python schedule_cli.py query-plans 2026-05-22`
- ❌ 错误：直接用 sqlite3 连接数据库写入

### 2. 增加操作全字段校验
**增加接口必须传入全部字段，少传一个字段直接报错。**

```python
add_record_full(date, time_start, time_end, duration_minutes,
                activity, category, source_contents,
                source_timestamps, analysis_reasoning)
```

### 3. 禁止删除记录（schedule_records / daily_summary）
数据库中**作息记录数据**不得出现任何 DELETE 语句，禁止删除记录。
- ❌ 禁止：`DELETE FROM schedule_records WHERE ...`
- ✅ 正确：只能增加（INSERT）和更新（UPDATE）
- 适用表：`schedule_records`、`daily_summary`

### 4. 日程计划数据的软删豁免（schedule_plans）
`schedule_plans` 表允许软删除（`UPDATE is_active=0`），不视为违反规则 3。
- 原因：日程计划是对未来的假设，迭代修订是合理操作
- 实现：`is_active` 字段（1=活跃，0=停用）
- 查询：默认 `WHERE is_active=1`，需要历史快照时用 `WHERE is_active=0`
- 物理 DELETE 仍禁止

### 5. 游标机制
**靠 schedule_records 表中最后一条记录的时间定位，不需要额外的 checkpoint 表。**
- 游标 = `SELECT ... ORDER BY date DESC, time_end DESC LIMIT 1`

### 6. 飞书日历同步约定
**每次涉飞书的 CRUD 操作，由 CLI 层询问用户是否同步**（决策 C），禁止静默同步。
- 默认行为：CLI 询问 [Y/n]，用户主动确认
- 飞书不可用时：CLI 自动跳过 + 提示安装价值，不阻塞主流程
- 关联主键：`schedule_plans.feishu_event_id` 内联维护，**禁止另起映射表**

## 时间格式规范

| 场景 | 格式 |
|---|---|
| DB 存储 | `time_start` / `time_end` 均为 `HH:MM`（24 小时制） |
| 末段哨兵 | `time_end` 可为 `24:00`（仅 DB 层合法） |
| 飞书 ISO | `YYYY-MM-DDTHH:MM:SS+08:00`（24:00 自动转次日 00:00） |
| 联合覆盖 | 所有事件的联合区间 ⊇ [00:00, 24:00] |

---

## 违反规范的后果

违反以上任何一条规范，将导致：
1. 数据不一致
2. 同步逻辑混乱
3. 与其他技能联动失败
4. 飞书日历错乱（写错时段或漏写）
