准备消息 获取待同步消息供 AI 分析
CLI 命令 python3 scripts/schedule_cli.py prepare-messages
执行流程（始终分页，默认每页 200 条）
查询 schedule_records 最后一条记录，确定游标位置（date + time_end）
从 daily_recorder.db 的 user_messages 表获取游标前 10 条消息（上下文参考）
获取游标时间到当前时间的消息（分页：--page 默认 1，--page-size 默认 200）
输出 JSON（含 pagination 元数据：page, total_pages, has_next），不写入任何表
若 has_next=true，AI 处理完当前页后用 --page N 获取下一页
操作数据库表字段类型
READschedule_data.dbschedule_recordsdate, time_end查询游标
READdaily_recorder.dbuser_messagesid, timestamp, channel, content获取消息（分页）
副作用：无写入操作。仅输出 JSON 供后续 AI 分析使用。
分页：始终分页，默认每页 200 条。JSON 中 pagination.has_next=true 时需继续获取下一页。
同步作息 完整同步流程：获取消息 → AI 分析 → 写入记录
CLI 命令 prepare-messages → AI 分析 → add_record_full()
执行流程
调用 prepare-messages（始终分页，默认每页 200 条）获取消息
前置：调用 get_required_block_count() 获取最少 block 数量
AI 用滑动窗口（前5后5条消息）判断活动边界

后置：调用 validate_record_count() 验证 block 数量是否达标

保证首尾相接：每条 time_start = 上一条 time_end
若 has_next=true，处理完当前页后用 --page N 获取下一页继续
操作数据库表字段类型

副作用：向 schedule_records 表插入 N 条记录。source_contents 为消息原文（一字不差）。

生成摘要 按分类汇总当日各活动时长
CLI 命令 python3 scripts/schedule_cli.py summary <日期>
执行流程
读取 schedule_records 中该日期的所有记录
按 category 分组，计算 duration_minutes 之和
调用 add_summary() 写入 daily_summary 表（KV 结构）
输出分类汇总（睡眠 7h、工作 4h 等）
操作数据库表字段类型
READschedule_data.dbschedule_recordsdate, category, duration_minutes按分类汇总
UPSERTschedule_data.dbdaily_summarydate, category, total_minutes写入摘要
前提条件：schedule_records 必须已覆盖该日期 00:00-23:59（满 24 小时）。
副作用：向 daily_summary 表写入 N 条记录（每个分类一条，upsert 语义）。
汇总作息 日期范围内的分类汇总统计
CLI 命令 python3 scripts/schedule_cli.py range <开始> <结束>
执行流程
从 daily_summary 表查询指定日期范围内的所有摘要
按日期分组，输出每天的分类时长和总计
操作数据库表字段类型
READschedule_data.dbdaily_summarydate, category, total_minutes范围查询
副作用：纯读取，无写入操作。

初始化数据库 创建三张核心数据表
CLI 命令 python3 scripts/schedule_cli.py init
执行流程
CREATE TABLE IF NOT EXISTS schedule_records（作息记录主表，11 字段）
CREATE TABLE IF NOT EXISTS daily_summary（每日摘要，KV 结构，联合主键）
CREATE TABLE IF NOT EXISTS schedule_plans（计划作息，24 小时字段）
操作数据库表说明
CREATEschedule_data.dbschedule_recordsid, date, time_start, time_end, duration_minutes, activity, category, source_contents, source_timestamps, analysis_reasoning, created_at
CREATEschedule_data.dbdaily_summarydate, category, total_minutes, generated_at（主键: date+category）
CREATEschedule_data.dbschedule_plansdate, hour_0_planned ~ hour_23_planned, created_at, updated_at（主键: date）
副作用：CREATE TABLE IF NOT EXISTS，已存在则跳过，不会覆盖已有数据。

配置每日报告 每日 07:30 验证并推送作息报告

执行流程

副作用：向 daily_summary 写入摘要 + 通过 WeChat 推送报告。
