# 共享单车提醒

## 触发条件

用户提到以下关键词时激活本技能：
- "骑车了""正在骑车""骑完车""骑车中""骑车回家""骑车上班""扫了小黄车""扫了美团""共享单车""小黄车""美团单车""哈啰""青桔""电动车""电单车"等

## 执行流程

### Step 1：识别并确认

当用户说开始骑车，立即回复：
> "🚲 好的，记住了！会定期提醒你关锁~"

同时执行 Step 2 创建循环提醒。

### Step 2：创建循环 cron 任务

使用 `exec` 工具调用 OpenClaw 的 cron 命令创建周期性提醒：

```bash
openclaw cron add \
  --name "共享单车提醒" \
  --cron "*/3 * * * *" \
  --tz "Asia/Shanghai" \
  --message "🚲 哥们，车关了吗？别忘了锁车哦！" \
  --channel qqbot \
  --to "18CD32F5999F760615E9862E343E59FC" \
  --session isolated
```

**注意**：必须使用 `exec` 工具实际执行 `openclaw cron add` 命令，不能只是口头承诺。

### Step 3：等待用户回复

- **用户回复"关了""锁了""已关""车关了"** → 执行 Step 4 取消提醒
- **用户没回复** → cron 会继续每3分钟发提醒
- **用户说"还没""等一下"** → 继续等待，不取消提醒

### Step 4：取消循环提醒

当用户确认已关锁，使用 `exec` 工具调用：

```bash
openclaw cron list 2>/dev/null | grep "共享单车提醒"
# 找到 ID 后
openclaw cron rm <job-id>
```

并回复：
> "✅ 收到，车已锁好，放心啦！"

## 关键规则

1. **每次用户说骑车就重新创建提醒**，即使之前有未取消的提醒（先 list 找到 ID → rm → 再 add 新提醒）
2. **提醒内容要简洁**，2-3句话，配合 emoji
3. **只有关锁相关词才取消**："关了""锁了""已关""车关了"，其他回复都继续提醒
4. **提醒频率是每3分钟**，不是更频繁，避免打扰
5. **必须用 OpenClaw 内置 cron**（`openclaw cron add`），不要用 qqbot_remind 工具

## 防重复机制

创建提醒前先检查是否已有同名 cron：
```bash
openclaw cron list 2>/dev/null | grep "共享单车提醒"
```
如果有，先删再创建，避免重复定时器。
