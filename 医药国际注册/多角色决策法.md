# 多角色决策法

## 核心思想

模拟完整团队（注册、法务、市场、代理、质量），多角度分析帮你决策。

## 适用场景

- "帮我分析先去哪个国家注册"
- "这个产品值不值得做"
- "从多个角度评估XX策略的风险"
- "帮我做注册策略规划"

## 方法论流程

```
定义决策问题 → 各角色独立分析 → 角色间辩论 → 综合建议 → 你拍板
```

## 底层调用

- 👥 多方评估决策支持 - 多角色分析
- 📚 查法规找资料 - 提供决策依据

## 可以模拟的角色

| 角色 | 看问题的角度 | 关心什么 |
|------|------------|---------|
| 注册专员 | 干活的人 | 资料好不好编、时间够不够、操作上可不可行 |
| 注册经理 | 管事的人 | 人够不够、进度会不会卡、怎么协调 |
| 法务顾问 | 合规的人 | 法规风险、知识产权、有没有合规漏洞 |
| 市场分析师 | 做生意的人 | 市场大不大、竞争怎么样、划不划算 |
| 代理商对接人 | 在当地的人 | 当地好不好操作、药监局好不好沟通、文化差异 |
| 质量负责人 | 管质量的人 | GMP合规、数据完整性、质量风险 |

## 对话示例

```
你：我有阿莫西林胶囊，想去马达加斯加、尼日利亚、喀麦隆，先去哪个国家？

我：好的，我用【多角色决策法】帮你分析。我会模拟注册专员、注册经理、法务顾问、市场分析师、代理商对接人、质量负责人6个角色，每个角色从自己的角度分析。

[调用：查法规找资料]
先让我查一下这3个国家的注册要求...

[调用：多方评估决策支持]
现在各角色开始分析：

【注册专员怎么看】
核心观点：尼日利亚资料要求最明确，操作最规范
最担心的风险：马达加斯加法规不透明，可能反复补资料
建议：先去尼日利亚，积累经验后再去其他国家

【法务顾问怎么看】
核心观点：喀麦隆是CEMAC区域，一次注册多国有效
最担心的风险：知识产权风险、数据保护
建议：如果产品有IP风险，优先CEMAC

【市场分析师怎么看】
核心观点：尼日利亚人口最多，市场最大
建议：从商业角度，尼日利亚优先

【角色间辩论】
法务顾问认为应该优先CEMAC（区域优势）
市场分析师认为应该优先尼日利亚（市场最大）
综合建议：如果资源允许，尼日利亚和CEMAC并行启动

【最终建议】
推荐方案：尼日利亚 + 喀麦隆（CEMAC）并行启动

⚠️ 需要你拍板的事：
- 是否接受并行启动的方案？
- 资源（人力、预算）是否足够支持并行？

你觉得呢？
```

## 执行步骤

### 第1步：确认分析什么

问你：
1. 分析什么主题（如：阿莫西林胶囊在5个非规范市场，先去哪个国家）
2. 要哪些角色参与（从上面选，或者自定义）
3. 分析多深（快速评估/详细分析/深度辩论）

默认角色组合：注册专员 + 注册经理 + 法务顾问 + 代理商对接人

### 第2步：每个角色各自分析

每个角色按这个框架说：

```markdown
## {角色名} 怎么看

### 核心观点
{1-3句话说清楚这个角色的判断}

### 最担心的风险（从大到小排）
1. {风险1} — 严重程度：高/中/低 — 可能性：高/中/低
2. {风险2}
3. ...

### 看到的机会
1. {机会1}
2. ...

### 建议怎么做
1. {建议1}
2. ...
```

### 第3步：角色之间辩论

找出各角色观点冲突的地方，让他们辩论：

```markdown
## 角色间辩论

### 辩论点1：{冲突主题}

**{角色A}认为**：{观点}

**{角色B}认为**：{观点}

**客观分析**：{对两方观点的分析}

**综合建议**：{建议倾向}
```

### 第4步：综合建议

```markdown
# 策略分析报告：{主题}

## 概要
- 参与角色：{角色列表}
- 分析时间：{日期}

## 各角色核心观点

| 角色 | 核心判断 | 风险等级 | 建议倾向 |
|------|---------|---------|---------|
| {角色1} | | | |
| {角色2} | | | |

## 风险矩阵

| 风险 | 严重程度 | 可能性 | 谁受影响 | 怎么缓解 |
|------|---------|--------|---------|---------|
| | | | | |

## 辩论焦点

### 焦点1：{主题}
- 冲突：{描述}
- 建议：{综合建议}

## 最终建议

### 推荐方案
{综合各角色观点后的推荐}

### 备选方案
{备选}

### ⚠️ 需要你拍板的事
{列出需要你最终决定的事项}

## 接下来该做什么
1. {行动1}
2. ...
```
