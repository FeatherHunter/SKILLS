# 快速探索法

## 核心思想

用自然语言对话快速探索，降低启动阻力，快速验证想法。

## 适用场景

- "我想了解XX国家的注册要求"
- "这个产品在XX市场能不能做"
- "帮我快速对比几个国家的注册难度"
- "XX国家最近有什么政策变化"

## 方法论流程

```
你提出想法 → 我快速调研 → 给你初步判断 → 你决定要不要深入
```

## 底层调用

- 📚 查法规找资料 - 快速获取信息
- 👥 多方评估决策支持 - 快速判断可行性

## 对话示例

```
你：我想了解一下尼日利亚NAFDAC的仿制药注册

我：好的，我用【快速探索法】帮你了解。

[调用：查法规找资料]
我查到尼日利亚NAFDAC的基本情况：
- 药监局：NAFDAC（尼日利亚国家食品药品管理局）
- 注册类型：仿制药简化注册
- 审评周期：6-12个月
- 关键要求：必须通过当地代理商提交

[调用：多方评估决策支持]
从注册难度看：中等
从市场机会看：人口多，市场大
从风险看：代理商依赖度高

你想深入了解哪个方面？还是想对比其他国家？
```

## 执行步骤

### 第1步：确认探索目标

问你：
1. 查哪个国家（可以选多个）
2. 查什么主题（注册流程/要交什么资料/法规最新变化/费用和周期/其他）
3. 查多深（快速看一眼/详细对比/深度分析）

### 第3步：输出

1. 调研报告存到 `output/` 目录
2. 所有拿不准的地方标"⚠️ 待确认"
3. 问你要不要补充或深入某个方面

## 注意

- 拿不准的信息必须标"⚠️ 待确认"，绝不编造
- 优先参考官方来源，其次参考行业报告
- 调研结果仅供参考，正式决策前你要自己确认
