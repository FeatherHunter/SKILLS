# 持续迭代法

## 核心思想

收到问题 → 回复 → 再收到问题 → 再回复，循环直到通过。

## 适用场景

- "帮我回复NAFDAC的缺陷信"
- "收到药监局的问题，帮我处理"
- "整改XX问题"

## 方法论流程

```
收到缺陷 → 分析 → 回复 → 提交 → [收到新缺陷？] → 循环 → [通过？] → 结束
```

## 底层调用

- 🔍 审文件查合规 - 分析缺陷
- 📄 按模板写标准化 - 生成回复
- ✅ 逐项核对反复检查 - 确保回复完整

## 对话示例

```
你：NAFDAC发来缺陷信了，帮我回复

我：好的，我用【持续迭代法】帮你处理。这个方法论会持续循环，直到药监局满意为止。

【第1轮】
[调用：审文件查合规]
我分析了缺陷信，共3个缺陷项：
1. 稳定性数据不完整
2. BE报告格式不符合NAFDAC要求
3. 申请人信息有误

[调用：按模板写标准化]
我生成了回复：
缺陷项1回复：补充了长期稳定性数据（附件1）
缺陷项2回复：重新格式化了BE报告（附件2）
缺陷项3回复：修正申请人信息为代理商名称

[调用：逐项核对反复检查]
确认：3个缺陷项都已回复，支持文件都附上了。

回复已生成，请确认后提交给代理商。

你：[提交了]

我：好的，等NAFDAC反馈。

【第2轮】（假设收到新缺陷）
你：NAFDAC又发来缺陷信了

我：继续处理...

[循环直到NAFDAC通过]

【结束】
你：NAFDAC通过了！

我：恭喜！缺陷信回复完成。
我会把这次的经验记录到经验库，下次遇到类似问题可以参考。
```

## 执行步骤

### 第1步：缺陷信接收与分析

- 输入：药监局缺陷信
- 操作：
  1. 翻译缺陷信（如非英语）
  2. 逐条列出缺陷项
  3. 分类：资料缺陷/数据缺陷/格式缺陷/其他
  4. 评估严重程度和回复难度
- 输出：缺陷项分析表

### 第2步：回复编制

- 输入：缺陷项分析表
- 操作：
  1. 逐条编制回复
  2. 每条回复格式：
     ```
     缺陷项{编号}：{原文}
     回复：{回复内容}
     支持文件：{附件列表}
     ```
  3. ⚠️ 申请人字段填写代理商公司名称
  4. 准备支持性附件
- 输出：回复草稿

### 第3步：回复审核

- 输入：回复草稿
- 操作：
  1. 注册经理审核技术内容
  2. 合规检查
  3. 确认所有缺陷项均已回复
- 输出：审核通过的回复

### 第4步：提交回复

- 输入：审核通过的回复
- 操作：通过代理商提交
- 输出：提交确认

### 第5步：循环迭代

收到新缺陷信 → 重复步骤1-4，直到通过。

## 回复时限

- 一般缺陷信：收到后30天内回复
- 严重缺陷：收到后15天内回复
- 具体时限以各国药监局要求为准
