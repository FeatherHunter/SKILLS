# 流水线作业法

## 核心思想

多Agent协作，内置品质关卡，标准化批量作业。

## 适用场景

- "帮我批量编制CTD资料"
- "按SOP生成XX产品的注册文件"
- "标准化处理一批文件"

## 方法论流程

```
定义作业标准 → Agent1生成 → Agent2审核 → Agent3校验 → 输出
```

## 底层调用

- 📄 按模板写标准化 - 生成文档
- 🔍 审文件查合规 - 合规检查
- ✅ 逐项核对反复检查 - 完整性检查

## 对话示例

```
你：帮我按SOP生成阿莫西林胶囊的CTD Module 3

我：好的，我用【流水线作业法】帮你生成。这个方法论会启动多个Agent协作，每个Agent负责一个环节，确保质量。

[Agent1：生成Agent]
[调用：按模板写标准化]
我按CTD Module 3的SOP生成了质量部分：
- 3.2.S 原料药数据
- 3.2.P 成品数据
生成完成，交给审核Agent。

[Agent2：审核Agent]
[调用：审文件查合规]
我检查了生成的文档：
✅ 申请人字段正确（代理商名称）
✅ 格式符合CTD要求
✅ 数据引用有来源
发现1个问题：稳定性数据部分缺少加速试验数据
→ 请补充加速试验数据，或标注"未进行"

[Agent3：校验Agent]
[调用：逐项核对反复检查]
我对照CTD Module 3清单逐项核对：
✅ 3.2.S.1 一般信息
✅ 3.2.S.2 生产工艺
✅ 3.2.S.3 特性鉴定
...
⚠️ 3.2.P.7 稳定性数据 - 缺少加速试验

【输出】
文档已生成：output/阿莫西林胶囊_Module3.md
状态：基本完成，需补充加速试验数据

下一步建议：
1. 补充加速试验数据后，我可以重新生成
2. 或者标注"未进行加速试验"，继续下一步
```

## 执行步骤

### 第1步：确认任务

问你：
1. 生成什么文档（CTD某个模块/注册申请信/缺陷信回复/其他）
2. 用什么模板或规范
3. 给我需要填的信息

### 第2步：生成文档

按模板生成，同时内置品质关卡：
- Agent1 生成文档
- Agent2 审核合规性
- Agent3 完整性校验

### 第3步：质量检查

1. 申请人字段是否填写代理商名称
2. 格式是否符合规范
3. 所有必填字段是否完整

### 第4步：输出

1. 文档存到 `output/`
