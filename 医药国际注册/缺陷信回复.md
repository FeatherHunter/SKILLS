# 缺陷信回复

## 基本信息

- 编号：SOP-REG-003
- 版本：V1.0
- 生效日期：2026-04-09
- 适用范围：非规范市场药监局缺陷信的回复

## 目的

规范缺陷信回复流程，确保回复完整、合规、及时。

## 流程

### 步骤1：缺陷信接收与分析

- 输入：药监局缺陷信
- 操作：
  1. 翻译缺陷信（如非英语）
  2. 逐条列出缺陷项
  3. 分类：资料缺陷/数据缺陷/格式缺陷/其他
  4. 评估严重程度和回复难度
- 输出：缺陷项分析表
- 负责人：注册专员

### 步骤2：回复编制

- 输入：缺陷项分析表
- 操作：
  1. 逐条编制回复
  2. 每条回复格式：
     ```
     缺陷项{编号}：{原文}
     回复：{回复内容}
     支持文件：{附件列表}
     ```
  3. ⚠️ 申请人字段填写代理商公司名称
  4. 准备支持性附件
- 输出：回复草稿
- 负责人：注册专员

### 步骤3：回复审核

- 输入：回复草稿
- 操作：
  1. 注册经理审核技术内容
  2. 合规检查
  3. 确认所有缺陷项均已回复
- 输出：审核通过的回复
- 负责人：注册经理

### 步骤4：提交回复

- 输入：审核通过的回复
- 操作：通过代理商提交
- 输出：提交确认
- 负责人：代理商

## 回复时限

- 一般缺陷信：收到后30天内回复
- 严重缺陷：收到后15天内回复
- 具体时限以各国药监局要求为准

## 合规校验项

| 字段 | 校验规则 | 常见错误 |
|------|---------|---------|
| 申请人 | **必须填写代理商公司名称** | 填写了生产企业名称 |
| 缺陷项逐条回复 | 每条缺陷必须对应一条回复 | 遗漏缺陷项 |
| 支持文件 | 每条回复须附支持文件 | 附件不完整 |
