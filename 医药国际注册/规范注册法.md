# 规范注册法

## 核心思想

规范先行，意图即真理。先定义清楚要做什么（spec），再严格执行。

## 适用场景

- "我要启动一个正式的注册项目"
- "帮我完成XX产品在XX国家的注册"
- "从0到1完成整个注册流程"

## 方法论流程

```
定义规范(spec) → 制定计划 → 编制资料 → 合规检查 → 提交 → 跟踪 → 完成
```

## 底层调用

- 📋 排计划管项目 - 制定项目计划
- 📚 查法规找资料 - 明确注册要求
- 📄 按模板写标准化 - 按SOP编制CTD资料
- 🔍 审文件查合规 - 合规性检查
- ✅ 逐项核对反复检查 - 完整性检查

## 对话示例

```
你：我要启动阿莫西林胶囊在尼日利亚的注册项目

我：好的，我用【规范注册法】帮你完成。这个方法论的核心是：先定义清楚要做什么，再严格执行。

第一步：定义规范（spec）
我需要你告诉我：
- 产品名称：阿莫西林胶囊
- 剂型和规格：[请告诉我]
- 目标国家：尼日利亚
- 代理商名称：⚠️ 这个必须告诉我
- 参比制剂：[请告诉我]
- 希望什么时候完成：[请告诉我]

你：0.5g，代理商是Lagos Pharma Ltd，参比制剂是Amoxil，希望年底前完成

我：好的，规范定义清楚了。现在开始执行。

[调用：排计划管项目]
我生成了项目计划：
- 总周期：约8个月
- 关键节点：资料编制(3个月) → 提交 → 审评(6个月) → 批准

[调用：查法规找资料]
我查到尼日利亚NAFDAC的具体要求：
- CTD格式
- 英语文件
- 必须通过代理商提交
- 申请人字段必须填代理商公司名

[调用：按模板写标准化]
现在开始编制CTD资料，我会按SOP逐模块生成：
- Module 1 行政信息
- Module 2 质量综述
- Module 3 质量（核心）
- Module 4 非临床
- Module 5 临床/BE

[调用：审文件查合规]
资料编制完成后，我会自动检查：
- 申请人字段是否填了代理商名称
- 所有必填字段是否完整
- 格式是否符合NAFDAC要求

[调用：逐项核对反复检查]
最后对照CTD清单，逐项核对，确保什么都不缺。

整个过程中，我会主动问你缺什么信息，你只需要回答我的问题，剩下的我来做。
```

## 执行步骤

### 第1步：定义规范（spec）

问你以下信息（缺什么问什么）：

```
告诉我项目的基本情况：
- 产品名称：[如：阿莫西林胶囊]
- 剂型和规格：[如：胶囊/0.5g]
- 目标国家：[可多选]
- 代理商名称：⚠️ 这个必须告诉我
- 参比制剂：[如：Amoxil]
- 希望什么时候完成：[如：年底前]
```

⚠️ 代理商名称是必填的，缺了必须问你要。

### 第2步：生成项目计划

按这个框架来：

```markdown
# {产品名称} 非规范市场注册项目计划

## 项目概览
- 产品：{产品名称} {剂型/规格}
- 类型：仿制药
- 去哪些国家：{国家列表}
- 时间：{开始日期} → {希望完成日期}

## 各国任务拆解

### {国家} — 代理商：{代理商名称}

| 阶段 | 要做什么 | 谁来做 | 大概多久 | 前置条件 | 状态 |
|------|---------|--------|---------|---------|------|
| 前期准备 | 确认代理商资质 | 注册部 | 1周 | 无 | 待开始 |
| 前期准备 | 搞清楚当地注册要求 | 注册部 | 2周 | 无 | 待开始 |
| 编资料 | Module 1 行政信息 | 注册部 | 1周 | 代理商信息 | 待开始 |
| 编资料 | Module 3 质量部分 | 质控部 | 4周 | 产品数据 | 待开始 |
| 编资料 | Module 4 非临床 | 注册部 | 1周 | 参比制剂信息 | 待开始 |
| 编资料 | Module 5 临床/BE | 研发部 | 6周 | BE报告 | 待开始 |
| 编资料 | 翻译和公证 | 外部 | 2周 | 全部资料 | 待开始 |
| 提交 | 通过代理商提交 | 代理商 | 1周 | 全部资料+翻译 | 待开始 |
| 审评 | 等技术审评 | 药监局 | 3-6月 | 提交完成 | 待开始 |
| 审评 | 回复缺陷信 | 注册部 | 2周 | 收到缺陷信 | 待开始 |
| 批准 | 拿注册证书 | 代理商 | 2周 | 审评通过 | 待开始 |

## 关键时间节点
1. 全部资料编完：预计 {日期}
2. 全部国家提交完：预计 {日期}
3. 第一个国家批下来：预计 {日期}
4. 全部国家批下来：预计 {日期}

## 可能的风险
- [根据各国特点列出]

## ⚠️ 待确认
- [需要你确认的信息]
```

### 第3步：合规检查

资料编制完成后，执行：
- 申请人字段是否填了代理商名称
- 所有必填字段是否完整
- 格式是否符合要求

### 第4步：完整性检查

最后对照CTD清单，逐项核对，确保什么都不缺。
