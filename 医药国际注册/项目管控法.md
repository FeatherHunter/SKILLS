# 项目管控法

## 核心思想

系统化管理项目全生命周期，控制进度、风险、质量。

## 适用场景

- "帮我管理XX产品在多国的注册项目"
- "跟踪项目进度"
- "管理多国注册项目"

## 方法论流程

```
项目启动 → 制定计划 → 分配资源 → 跟踪进度 → 风险预警 → 调整 → 完成
```

## 底层调用

- 📋 排计划管项目 - 项目计划与跟踪
- 👥 多方评估决策支持 - 风险评估
- 🔍 审文件查合规 - 质量控制

## 对话示例

```
你：我要管理阿莫西林胶囊在5个非洲国家的注册项目

我：好的，我用【项目管控法】帮你管理。这个方法论会系统化管理整个项目生命周期。

[调用：排计划管项目]
我建立了项目计划：
- 产品：阿莫西林胶囊 0.5g
- 目标国家：尼日利亚、马达加斯加、喀麦隆、加纳、肯尼亚
- 总周期：18个月
- 关键节点：资料编制(3个月) → 各国提交 → 审评跟踪 → 批准

各国进度看板：
| 国家 | 状态 | 当前进度 | 风险等级 |
|------|------|---------|---------|
| 尼日利亚 | 资料编制中 | 60% | 低 |
| 马达加斯加 | 等代理商确认 | 10% | 中 |
| 喀麦隆 | 未启动 | 0% | 低 |
| 加纳 | 未启动 | 0% | 中 |
| 肯尼亚 | 未启动 | 0% | 高 |

[调用：多方评估决策支持]
风险评估：
- 高风险：肯尼亚代理商未确定
- 中风险：马达加斯加、加纳法规不明确
- 建议：优先推进尼日利亚和喀麦隆

我会持续跟踪项目进度，你可以随时告诉我：
- "XX国家进展到哪了"
- "XX国家遇到问题了"
- "帮我看看项目整体情况"

我会主动预警风险，提醒你注意关键节点。
```

## 执行步骤

### 第1步：收集项目信息

问你以下信息（缺什么问什么）：

```
告诉我项目的基本情况：
- 产品叫什么：[如：阿莫西林胶囊]
- 剂型和规格：[如：胶囊/0.5g]
- 去哪些国家：[可多选]
- 各国的代理商叫什么：⚠️ 这个必须告诉我
- 什么时候开始：
- 希望什么时候做完：
- 现在到哪一步了：[还没开始/正在编资料/已经交了/正在审评/其他]
```

### 第2步：生成项目计划

按这个框架来：

```markdown
# {产品名称} 非规范市场注册项目计划

## 项目概览
- 产品：{产品名称} {剂型/规格}
- 类型：仿制药
- 去哪些国家：{国家列表}
- 时间：{开始日期} → {希望完成日期}

## 各国任务拆解

### {国家} — 代理商：{代理商名称}

| 阶段 | 要做什么 | 谁来做 | 大概多久 | 前置条件 | 状态 |
|------|---------|--------|---------|---------|------|
| 前期准备 | 确认代理商资质 | 注册部 | 1周 | 无 | 待开始 |
| ... | ... | ... | ... | ... | ... |

## 关键时间节点
1. 全部资料编完：预计 {日期}
2. 全部国家提交完：预计 {日期}
3. 第一个国家批下来：预计 {日期}

## 可能的风险
- [根据各国特点列出]
```

### 第3步：建项目文件夹

在 `projects/` 目录下建：
```
projects/{产品名称}/
├── 项目计划.md
├── 进度跟踪.md
├── 各国资料/
│   ├── {国家1}/
│   ├── {国家2}/
│   └── ...
└── 通信记录/
```

### 第4步：跟踪进度

你可以随时告诉我哪步做完了，我来更新进度表。

进度表长这样：
```markdown
# {产品名称} 进度跟踪

更新时间：{日期}

## 总体进度
- 总共{N}件事
- 做完了{M}件
- 正在做{K}件
- 卡住了{L}件
- 完成率：{M/N}%

## 各国进度

### {国家}
| 事情 | 状态 | 完成日期 | 备注 |
|------|------|---------|------|

## ⚠️ 卡住的事
- [哪件事卡住了、为什么]

## 📋 接下来最该做什么
- [最紧急的待办]
```

### 第5步：输出

1. 项目计划存到 `projects/{产品名称}/项目计划.md`
2. 进度跟踪存到 `projects/{产品名称}/进度跟踪.md`
