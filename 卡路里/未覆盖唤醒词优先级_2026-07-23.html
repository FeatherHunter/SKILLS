未覆盖唤醒词 · HTML 价值评估

~52 个未覆盖的唤醒词中，按"价值类型 + 复用频率 + 实施难度"评估，8 个候选。但首先必须理解当前系统的根本约束。

⚠ HTML 单向渲染 · 不能自动与 AI 互动
唯一互动路径：HTML "复制"按钮 → 用户粘贴 → AI 执行
⚠ 关键约束 📊 3 类价值 🏆 8 个候选 🔴 3 个 S 级
🔴 当前技术约束（必须先理解）
• HTML 通过 <media> 标签附在飞书/Chrome 中打开 · 不是 localhost 服务
• HTML 不能反向调用 LLM · 无法触发 AI 操作
• 唯一 AI 互动路径：HTML 提供"复制"按钮 → 用户把复制内容粘贴回 AI 对话框 → AI 完成下一步
• 用户 = 承上启下的桥梁，把 HTML 输出"翻译"成 AI 指令
↓ 设计影响 ↓
❌ "一键保存" / "一键替换"  ← 不可能（HTML 不能操作 DB）
✅ "复制给 AI" 按钮         ← HTML 是编辑器 + 复制模板

🎯 判据模型 · HTML 的 3 类价值（修正版）

📊

A · 结果型

单纯展示 · 不需要 AI 互动

输出本身就是结构化数据,用户只看。
无约束 · 不需要"复制"按钮（除非用户想分享给 AI）。

例:健康报告、5 食物排行、健身计划视图
⚙️

B · 过程型

AI 协同模板 · 必须有"复制"按钮

操作 3+ 步 / 用户要做选择 / AI 提取待核对。
HTML 是"编辑器 + 复制模板",用户改完复制给 AI。

例:拍营养表、批量导入、4 步流程进度
本类必须设计为"完成后复制 prompt"
✓

C · 回执型

单条 CRUD 后的视觉反馈

单条录入后给个"漂亮回执",不只是"✓ 已记录"。
回执本身无 AI 互动需求,但可附"复制今日进度"按钮。

例:记体重 → 趋势叠加新点

🤝 过程型 HTML 的标准 AI 协同模板

所有"过程型 HTML"必须有 3 个元素

1. 编辑表单（HTML 可视化,字段可改） 2. "采纳"按钮（无修改时用,复制"已确认"短文本） 3. "复制修改指令"按钮（修改后用,复制完整 prompt）

复制 prompt 标准格式

场景: "扫禁忌后修改" ━━━━━━━━━━━━━━━━━━━━━━━━ 我扫描了禁忌,需要修改健身计划。请帮我把以下动作替换为安全变体: 原动作: 杠铃硬拉 替换为: 支撑式悍马划船 原因: 腰椎间盘突出禁忌的髋铰链轴向压力 原动作: 罗马尼亚硬拉 替换为: 俯卧 T-bar 划船 原因: 同上 请执行: python scripts/plan_generator.py update-action \ --week 1 --day 1 --old "杠铃硬拉" --new "支撑式悍马划船" 数据来源: scanning 禁忌扫描报告 2026-07-23 计划 ID: v14.4（请先 plan_generator validate 再写入）

关键设计原则

  • HTML 永远不做最终动作——只做"看清 + 编辑 + 准备指令"
  • prompt 含 4 部分：场景描述 + 数据 + 期望动作 + 数据来源
  • 不要寄希望"无修改"——大多数情况都有修改,所以"复制修改指令"是主按钮
  • "采纳"按钮复制短文本"已确认,请按 AI 路由执行"——AI 自己知道怎么调 CLI
