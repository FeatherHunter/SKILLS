图片路由

图片内容智能识别与技能路由。用户发送图片时自动触发，或通过文字指令唤醒。识别图片内容后判断归属领域，路由到对应技能执行。

v1.0 13 唤醒词 mmx vision describe
图片识别
图片内容识别 识别图片
对图片做快速内容识别，返回图片中包含的物体、场景、文字等描述信息。
CLI mmx vision describe --image <path> --prompt "描述这张图片的内容"
执行流程
  1. 调用 MiniMax Vision API，传入图片路径
  2. API 返回图片内容的自然语言描述
  3. 无数据库操作，无副作用
副作用：无。仅返回识别结果，不写入任何数据。
深度解析 解析图片
对图片做深度解析，强调细节提取——包括 OCR 文字、数字、表格结构、小字备注等。
CLI mmx vision describe --image <path> --prompt "详细解析图片内容，提取所有文字、数字、表格"
执行流程
  1. 调用 MiniMax Vision API，prompt 侧重细节提取
  2. API 返回包含文字、数字、结构的详细描述
  3. 无数据库操作，无副作用
副作用：无。与「识别图片」使用同一工具，区别在于 prompt 侧重点不同。
自然语言描述 描述图片
用自然语言描述图片整体内容，适合用户分享图片时了解 AI 看到了什么。
CLI mmx vision describe --image <path> --prompt "描述这张图片的内容"
执行流程
  1. 调用 MiniMax Vision API
  2. API 返回自然语言描述
  3. 无数据库操作，无副作用
副作用：无。与「识别图片」同一命令，语义侧重不同（描述 vs 识别）。
OCR 文字提取 提取文字
专门做 OCR，提取图片中的文字和数字。适合小票、账单、截图、文档照片等场景。
CLI mmx vision describe --image <path> --prompt "提取图片中所有文字和数字，保持原始格式"
执行流程
  1. 调用 MiniMax Vision API，prompt 专注 OCR
  2. API 返回提取的文字和数字，保留原始格式
  3. 无数据库操作，无副作用
副作用：无。纯识别，不写入数据。提取结果可用于后续路由或记录。
路由判断
分析归属领域 判断路由
识别图片内容后，分析图片属于哪个技能领域（记账/管家/卡路里），只判断不执行。适合用户想先了解图片归类再决定操作。
CLI mmx vision describe → 归属判断
执行流程
  1. subagent 调用 mmx vision describe 识别图片内容
  2. AI 基于识别结果判断归属领域：
    · 涉及金额/交易 → 饼干记账
    · 涉及物品存放 → 居家管家
    · 涉及食物/营养 → 卡路里
  3. 向用户展示判断结论，不执行任何操作
副作用：无。纯判断，不触发下游技能，不写入数据。
路由执行
路由到饼干记账 路由记账
识别图片后直接路由到饼干记账。适用于金额/交易类图片：小票、转账截图、账单页面。
CLI mmx vision describe → 饼干记账
执行流程
  1. subagent 调用 mmx vision describe 识别图片
  2. AI 提取金额信息（优先「实付/实收」，其次「合计/总计」）
  3. 调用饼干记账 CLI：python scripts/cookie_cli.py add -a <金额> -t "图片记账"
  4. 写入 cookie.db → transactions 表 → amount、description、category 字段
副作用：向饼干记账数据库写入一条交易记录。需用户确认后执行。
路由到居家管家 路由管家
识别图片后直接路由到居家管家。适用于物品类图片：快递包裹、收纳柜、物品照片。
CLI mmx vision describe → 居家管家
执行流程
  1. subagent 调用 mmx vision describe 识别图片
  2. AI 提取物品名称、位置、数量等信息
  3. 调用居家管家 CLI：python scripts/home_cli.py add --name "XX" --location "XX"
  4. 写入 home.db → items 表 → name、location、category、status 字段
副作用：向居家管家数据库写入一条物品记录。需用户确认后执行。
路由到卡路里 路由卡路里
识别图片后直接路由到卡路里。适用于食物/营养类图片：饭菜照片、营养成分表、菜单。
CLI mmx vision describe → 卡路里
执行流程
  1. subagent 调用 mmx vision describe 识别图片
  2. AI 提取食物名称、热量、营养成分
  3. 调用卡路里 CLI：python scripts/calorie_tracker.py add "食物名" <热量> <蛋白> <碳水> <脂肪> <克数>
  4. 写入 calorie.db → entries 表 → name、calories、protein、carbs、fat、grams 字段
副作用：向卡路里数据库写入一条饮食记录。需用户确认后执行。
AI 自动判断推荐 自动路由
AI 自动识别图片内容、判断归属领域、推荐对应技能，用户确认后执行。默认路由模式。
CLI mmx vision describe → 判断 → 推荐
执行流程
  1. subagent 调用 mmx vision describe 识别图片
  2. AI 判断归属领域（记账/管家/卡路里/不相关）
  3. 向用户展示识别结果 + 推荐操作
  4. 用户确认后，调用对应技能 CLI 执行
副作用：确认前无副作用。确认后根据推荐技能写入对应数据库。
批量组合
多图分别路由 批量路由
连续发送多张图片时，一次性识别并分别路由到对应技能。每张图片独立判断归属。
CLI 循环 mmx vision describe → 分别路由
执行流程
  1. 遍历所有待处理图片
  2. 逐张调用 mmx vision describe 识别
  3. 每张图片独立判断归属领域
  4. 汇总展示所有路由建议，用户逐一确认执行
副作用：确认后根据各图片路由结果写入对应数据库。每张图片独立处理，互不影响。
一图多技能联动 组合执行
一张图片触发多个技能。例如食物照片同时触发记账（记录消费）和卡路里（记录摄入）。
CLI mmx vision describe → 多技能顺序执行
执行流程
  1. subagent 调用 mmx vision describe 识别图片
  2. AI 检测到多信号叠加（如食物+金额）
  3. 推荐多个技能组合，展示给用户
  4. 用户逐一确认，依次调用各技能 CLI 执行
副作用：向多个数据库写入记录。每个技能执行完成后单独询问是否继续。
查询重试
查看最近路由结果 查看路由
查看最近一次图片路由的识别结果和执行状态。无需重新识别，直接从上下文读取。
CLI 读取上下文
执行流程
  1. 从当前会话上下文中读取最近的路由记录
  2. 展示：识别内容、判断结果、执行状态
  3. 无 API 调用，无数据库操作
副作用：无。纯查询，不修改任何数据。
重新判断路由 重新路由
对同一张图片重新识别并判断路由。用于上次判断不准确时的纠错场景。
CLI mmx vision describe → 重新判断
执行流程
  1. 重新调用 mmx vision describe 识别同一图片
  2. AI 重新判断归属领域
  3. 展示新的路由建议，用户确认后执行
副作用：确认前无副作用。重新识别不影响之前已执行的记录。
特殊触发：图片事件
用户发送图片时自动触发此技能，无需文字唤醒词。图片事件优先于文字指令匹配。
