# 备忘录 (Memorandum)

## 描述
私人多功能备忘录，支持文字记录、分类、媒体附件、全文搜索和定时提醒。帮助用户将所有临时信息外置，并在需要时快速检索或主动提醒。

## 触发示例
- “记一下：明天下午3点开会”
- “我的社交夹里记了什么？”
- “搜一下关于旅行的笔记”
- “设置每周五早上9点提醒我还信用卡”
- “我的愿望单里有什么？”
- “拍张照存起来”
- “今天最大的收获是学会了FTS5”

## 核心功能
### 1. 笔记记录
- 快速添加文字笔记，可选指定分类（category）。
- 支持媒体附件（图片/录音/文件），路径存储在笔记中，文件加密由上层处理。
- 笔记会自动生成摘要（summary），用于列表展示。

### 2. 笔记组织与检索
- 使用 category 扁平化分类，常用值如 `social`、`wish`、`inspiration`、`achievement`，也允许用户自创。
- 全文搜索：通过 FTS5 进行高效中文搜索，支持关键词匹配。
- 列表查看：可按分类、时间倒序浏览。

### 3. 定时提醒
- 每条提醒必须关联一条已存在的笔记。
- 支持一次性提醒和重复提醒（每天/每周/每月/每年）。
- 重复规则存储在 `repeat_rule` 字段，格式严格定义。
- 系统每分钟自动检查未来10分钟内需触发的提醒，主动通知用户。
- 过期提醒自动废弃。

## 系统架构
- 底层：SQLite 数据库 `memo.db`
- 中层：Python CLI 工具 `memo_cli.py`
- 上层：当前 SKILL.md 与 module 模块协作，将自然语言转换为 CLI 调用。

## 依赖模块
- `script/intent_handler.py`：负责意图解析、参数提取和命令构造。
- `script/reminder_scheduler.py`：用于 openclaw cron 调用的定时检查逻辑，会调用 `memo_cli.py due`。

## 环境变量
- `MEMO_MEDIA_DIR`: 媒体文件存储的根目录绝对路径。

## 注意事项
- 媒体文件加密请在上层 Skill 调用时完成，CLI 仅记录相对路径。
- 提醒不自动计算下一次触发时间，规则解析由 `due` 命令实时计算。
