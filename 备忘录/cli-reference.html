笔记操作
add 添加笔记 ▶
参数类型必填说明
contentstr必填笔记内容（不能为空）
--category -cstr可选分类，默认 general
--media -mstr可选媒体路径（须以 MEMO_MEDIA_DIR 开头）
# 基本添加 python memo_cli.py add "去医院复查" -c "心愿" # 带媒体附件 python memo_cli.py add "今天的照片" -c "打卡" -m "media/photos/img.jpg" # 错误示例 python memo_cli.py add "" → 笔记内容不能为空 python memo_cli.py add "test" -c "xxx" → 无效分类: xxx python memo_cli.py add "test" -m "other" → 媒体路径必须以 media 开头
  • content 不能为空或纯空格
  • category 必须在白名单内
  • media 路径必须以 MEMO_MEDIA_DIR 环境变量值开头，存储时自动去掉前缀
search 搜索笔记（全文搜索） ▶
参数类型必填说明
keywordstr可选搜索关键词（空则列出全部）
--category -cstr可选分类过滤
--limit -lint可选结果数量，默认 20
# 搜索关键词 python memo_cli.py search "医院" # 按分类搜索 python memo_cli.py search "" -c "工作" -l 10 # 错误示例 python memo_cli.py search "test" -l -1 → limit 必须是正整数
  • limit 必须是正整数（> 0）
update 更新笔记 ▶
参数类型必填说明
idint必填笔记 ID（正整数）
--contentstr可选新内容
--category -cstr可选新分类
--media -mstr可选新媒体路径
--reminder-idint可选关联提醒 ID
# 更新内容 python memo_cli.py update 1 --content "去医院复查（已改期）" # 更新分类 python memo_cli.py update 1 -c "工作" # 错误示例 python memo_cli.py update 0 --content "test" → 笔记 ID 必须是正整数 python memo_cli.py update 1 → 至少需要提供一个更新字段 python memo_cli.py update 999 --content "test" → 笔记不存在: id=999
  • id 必须是正整数（> 0）
  • 至少提供一个更新字段（content / category / media / reminder-id）
  • category 必须在白名单内
  • media 路径必须以 MEMO_MEDIA_DIR 开头
  • 笔记必须存在

  • id 必须是正整数（> 0）
  • 笔记必须存在

search-date 按日期搜索笔记 ▶
参数类型必填说明
startstr必填开始日期 YYYY-MM-DD
endstr必填结束日期 YYYY-MM-DD
--category -cstr可选分类过滤
--limit -lint可选结果数量，默认 20
# 搜索本月笔记 python memo_cli.py search-date "2026-05-01" "2026-05-31" # 按分类搜索 python memo_cli.py search-date "2026-05-01" "2026-05-31" -c "工作" # 错误示例 python memo_cli.py search-date "abc" "def" → start 格式错误，要求 YYYY-MM-DD python memo_cli.py search-date "2026-01-01" "2025-01-01" → 开始日期不能晚于结束日期
  • start 和 end 必须是 YYYY-MM-DD 格式
  • start 不能晚于 end
  • limit 必须是正整数

  • id 必须是正整数（> 0）
  • category 必须在白名单内
  • 笔记必须存在
提醒操作
remind 添加提醒 ▶
参数类型必填说明
note_idint必填关联笔记 ID
--atstr可选一次性时间 YYYY-MM-DD HH:MM
--repeat-typeenum可选一次性 / 每天 / 每周 / 每月 / 每年
--rulestr可选重复规则（格式取决于 repeat-type）

--rule 格式要求：

repeat-typerule 格式示例说明
一次性--不需要 --rule，必须有 --at
每天HH:MM09:00每天 9 点
每周W HH:MM5 17:00每周五 17 点（W: 0=周日, 1-6=周一至周六）
每月D HH:MM15 08:30每月 15 号 8:30（D: 1-31）
每年MM-DD HH:MM12-25 10:00每年 12 月 25 日 10 点
# 一次性提醒 python memo_cli.py remind 1 --at "2026-05-28 09:00" # 每天重复 python memo_cli.py remind 1 --repeat-type "每天" --rule "09:00" # 每周重复（周五 17:00） python memo_cli.py remind 1 --repeat-type "每周" --rule "5 17:00" # 每月重复（15 号 08:30） python memo_cli.py remind 1 --repeat-type "每月" --rule "15 08:30" # 每年重复（12 月 25 日 10:00） python memo_cli.py remind 1 --repeat-type "每年" --rule "12-25 10:00" # 错误示例 python memo_cli.py remind 1 --at "abc" → --at 格式错误 python memo_cli.py remind 1 --repeat-type "每天" → 必须提供 --rule python memo_cli.py remind 1 --repeat-type "每周" --rule "abc" → --rule 格式错误 python memo_cli.py remind 1 --repeat-type "每周" --rule "8 09:00" → 星期超出范围 (0-6) python memo_cli.py remind 1 --repeat-type "每月" --rule "32 09:00" → 日期超出范围 (1-31) python memo_cli.py remind 1 --repeat-type "每年" --rule "13-01 09:00" → 月份超出范围 (1-12) python memo_cli.py remind 1 --repeat-type "每天" --rule "25:00" → 时间超出范围
  • note_id 必须是正整数，且笔记必须存在
  • --at 格式必须是 YYYY-MM-DD HH:MM
  • --repeat-type=一次性 时必须提供 --at
  • --repeat-type 非一次性时必须提供 --rule
  • --rule 格式必须匹配对应的 repeat-type
  • 时间范围：小时 0-23，分钟 0-59
  • 星期范围：0-6（0=周日）
  • 日期范围：1-31
  • 月份范围：1-12
due 获取到期提醒（内部命令，由 cron 调用） ▶
参数类型必填说明
--dbstr可选数据库路径覆盖
python memo_cli.py due python memo_cli.py due --db "/custom/path/memo.db"
dismiss 废弃提醒 ▶
参数类型必填说明
idint必填提醒 ID（正整数）
python memo_cli.py dismiss 1 # 错误示例 python memo_cli.py dismiss 0 → 提醒 ID 必须是正整数 python memo_cli.py dismiss 999 → 提醒不存在: id=999 python memo_cli.py dismiss 1 → 提醒已经是废弃状态（重复废弃）
  • id 必须是正整数（> 0）
  • 提醒必须存在
  • 提醒状态必须是 active（不能重复废弃）
reminders 查看提醒列表 ▶
参数类型必填说明
--status -senum可选active（默认）/ dismissed
# 查看活跃提醒 python memo_cli.py reminders # 查看已废弃提醒 python memo_cli.py reminders -s "dismissed" # 错误示例 python memo_cli.py reminders -s "xxx" → 无效状态: xxx，允许的值: active, dismissed
  • status 只能是 active 或 dismissed
completed 查看已完成提醒（打卡关联） ▶

无参数。查询已完成提醒：一次性提醒（已通知+打卡）或有打卡关联的重复提醒。

python memo_cli.py completed

匹配逻辑：

类型完成条件
一次性已通知（notified_at 存在）+ 有打卡笔记
每天打卡日期 == 今天
每周打卡在当前自然周 + 星期匹配
每月打卡在当月 + 日期匹配
每年打卡在年内 + 月日匹配
