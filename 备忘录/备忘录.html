快速开始

  1. 读 SKILL.md 了解功能(AI 会按这个文件路由)
  2. 检查环境变量,交互式帮你配置 SKILLS_DB_PATH / MEMO_MEDIA_DIR
  3. 运行 script/init.sql 初始化数据库
  4. 设置 cron 任务:每分钟检查提醒,QQ 渠道推送(详见 定时提醒)
  5. 验证:python3 script/memo_cli.py --help

第一个示例(添加一条笔记):

python3 script/memo_cli.py add "今天开了个会" -c 备忘 -s 工作

环境变量

变量必填说明默认
SKILLS_DB_PATH否数据库根目录(统一配置)父目录 .db/ 层层找 / D:/.db/(Win) / /mnt/d/.db/(WSL)
MEMO_MEDIA_DIR否媒体文件目录media

操作规范

  • 所有操作通过 script/memo_cli.py 执行
  • 提醒必须关联笔记,不可独立存在
  • 媒体文件路径使用相对路径存储
  • CLI 返回 JSON:{"status": "ok/error", "data": ..., "message": "..."}

媒体附件

  • 参数:--media <文件名>
  • 支持类型:图片(jpg/png/gif)、音频、视频
  • 存储路径:MEMO_MEDIA_DIR/ 目录下
  • 示例:script/memo_cli.py add "购物小票" -c 记账 --media slip_20260522.jpg

⚠️ 提醒路由(防用错工具)

所有「提醒」类请求(无论是否以「备忘录」开头):

  • 必须走备忘录 CLI(先 add → 再 remind)
  • 禁止使用 qqbot_remind 或其他提醒工具

触发词详细 · 记录类

记备忘

用法: add "内容" [-c 备忘|心愿|打卡|情绪日记] [-s 子分类] [--due YYYY-MM-DD]

添加前 AI 会问"是工作相关还是心愿?"(除非你已明确指定分类或用子唤醒词)。

🔍 查看底层原理
  • CLI: memo_cli.py add "内容" [-c 备忘|心愿|打卡|情绪日记] [-s 子分类] [--due YYYY-MM-DD]
  • SQL: INSERT INTO notes (content, category, sub_category, media_path, due, ...) VALUES (...)
  • 飞书 hook: category == "心愿" 时自动建飞书 task(失败降级)
  • 失败: 空内容 / 无效分类 / due 格式错 → status: error + 自动 rollback

改备忘

用法: 先搜 → 找到 ID → update <id> [--content "新内容"] [-c 分类] [-s 子分类]
🔍 查看底层原理
  • SQL: UPDATE notes SET ..., updated_at = datetime('now','localtime') WHERE id = ?
  • 飞书 hook: category=心愿 + 内容变 → 同步飞书 task 标题

删备忘

用法: 先搜 → 找到 ID → delete <id> [--with-reminders]
🔍 查看底层原理
  • SQL: 先 DELETE FROM reminders WHERE note_id = ? → 再 DELETE FROM notes WHERE id = ?(避开 FK 约束)
  • 原子性: 事务包裹,失败回滚
  • 飞书 task: 不会被删除(飞书无 delete 概念)

备忘改分类

单条: update-category <id> <新分类>
批量(说"都/全部/多 id"): AI 调 batch-update-category --from-category X --html,在 HTML 里勾选 + 选目标分类
🔍 查看底层原理
  • SQL: UPDATE notes SET category = ?, updated_at = ... WHERE id = ?
  • 副作用: 不动 sub_category(二阶属性独立)
  • 硬规则: --from-category ≠ --to-category

备忘改子分类

用法: update-sub-category <id> <子分类 | null>

传 null 表示清除子分类。适用于所有 category。

🔍 查看底层原理
  • SQL: UPDATE notes SET sub_category = ?, updated_at = ... WHERE id = ?
  • null 处理: 传 "null" / "" → 清除

sub_category 原则

sub_category 是自由文本字段,AI 智能从用户原话推断 1 个 2 字(推断不出则 NULL):

  • 1 个,2 字(简短但比 1 字精确)
  • 不预设白名单:任何有意义的 2 字都可以(如"工作"/"学习"/"跑步"/"社交")
  • 适用于所有 category:不限于"备忘",心愿/打卡/情绪日记都能有 sub_category

例子:

  • "今天跑了 5 公里" → -c 打卡 -s 跑步
  • "今天学 Python" → -c 备忘 -s 学习
  • "张三生日 10/3" → -c 备忘 -s 社交
  • "今天去医院" → -c 备忘,sub_category=NULL

触发词详细 · 查找类

搜备忘 / 查备忘(别名)

用法: search "关键词" [-c 分类] [-s 子分类]

AI 会自动调 --html 渲染网页给你,带 chip 筛选 + 复制 ID。

🔍 查看底层原理
  • 底层: FTS5 全文搜索(notes_fts MATCH ?),无关键字时按分类列
  • SQL: SELECT n.* FROM notes n JOIN notes_fts f ON n.id = f.rowid WHERE notes_fts MATCH ? ...
  • 返回: {status: "ok", data: [items], message: "找到 N 条"}

看备忘

用法: 搜索找到 ID → get <id>

AI 会自动调 --html 渲染详情页。

🔍 查看底层原理
  • SQL: SELECT * FROM notes WHERE id = ?

按时间搜备忘

用法: search-date <start> <end> [-c 分类]

日期格式 YYYY-MM-DD,start ≤ end。AI 自动调 --html。

🔍 查看底层原理
  • SQL: SELECT * FROM notes WHERE created_at BETWEEN ? AND ? [AND category = ?]

触发词详细 · 提醒类

记提醒(添笔记 + 设提醒)

用法: add "内容" -c 备忘 → 拿到 id → remind <id> --at "时间" --content "提醒内容"

AI 通常自动完成两步。

🔍 查看底层原理
  • 两步: add 笔记 → remind 加提醒
  • 笔记与提醒分别存储,关联通过 note_id

设提醒

用法: remind <note_id> [--at "YYYY-MM-DD HH:MM"] [--content "提醒内容"] [--repeat-type 每天|每周|每月|每年] [--rule "HH:MM"]
🔍 查看底层原理
  • SQL: INSERT INTO reminders (note_id, remind_at, repeat_type, repeat_rule, ...) VALUES (...)
  • FK: note_id → notes(id) ON DELETE NO ACTION
  • 重复规则格式: "HH:MM"(每天)/"W HH:MM"(每周)/"D HH:MM"(每月)/"MM-DD HH:MM"(每年)

看提醒

用法: reminders [--status active|dismissed]

AI 自动调 --html,按重复类型提供筛选 chip。

🔍 查看底层原理
  • SQL: SELECT * FROM reminders WHERE status = ?

查已提醒备忘

用法: completed

AI 自动调 --html,支持复制提醒 ID / 打卡 ID。

🔍 查看底层原理
  • 匹配逻辑: 一次性已通知+关联打卡 / 重复型有关联打卡

废弃提醒

用法: dismiss <reminder_id>

触发词详细 · 心愿类

心愿排期(过程型 HTML)

用法: wish-batch-plan [--ids 1 2 3] [--all] [--suggest-due YYYY-MM-DD] --html

说"这 3 个都排到 7/3" / "心愿 #36 #48 安排到 6/30" → AI 调此命令生成过程型 HTML 给你在网页里调 → 采纳复制 → AI 调 set-due。

第一性:不在对话中逐条报 ID,先在 HTML 里看清全部、勾选、确认。

🔍 查看底层原理
  • 底层命令: set-due <id1> <id2> ... --due X(批量 SQL UPDATE)
  • HTML: 过程型 wish_plan.html(默认未勾选,你主动勾要排期的)
  • 飞书: 同步飞书 task due(UTC ms → 北京日期)
  • 失败: 心愿无 feishu_task_guid → 需先跑"备忘录同步"补建

完成心愿 / 完成打卡(别名 · 过程型 HTML)

用法: wish-complete [--ids 1 2 3] [--only-overdue] [--content "打卡内容"] --html

说"这些心愿都完成了" / "完成打卡 #36 #48" → AI 调此命令。你在 HTML 里勾要完成的 + 填打卡内容 → 采纳复制 → AI 调 complete-wish 原子转打卡。

第一性:过程型 HTML 的价值是让你主动表达"完成哪些",默认未勾选(不是反向删勾)。

🔍 查看底层原理
  • 底层: complete-wish <id> [--content "打卡内容"](每次 1 条原子操作)
  • 原子操作: DELETE reminders → DELETE notes → INSERT notes(category=打卡) 事务包裹
  • HTML: 过程型 wish_complete.html(默认未勾选)
  • 飞书: 心愿有 feishu_task_guid → 标飞书 task 完成

触发词详细 · 批量类

备忘改分类(批量 · 过程型 HTML)

用法: batch-update-category --from-category X [--to-category Y] --html

说"把 X 分类下这 10 条都改到 Y 分类" → AI 调此命令。你在 HTML 里勾选 + 选目标分类 → 采纳复制 → AI 调多条 update-category。

触发词路由:

  • 原话只含 1 个具体 ID(如 #15 / 笔记 A)→ 单条 update-category
  • 原话含"都/全部/这 N 条"或多个 ID → 批量向导
  • 原话没有 ID 也没有"都/全部" → AI 反问用户"是想单条还是批量?"
🔍 查看底层原理
  • 触发路由: 含"都/全部/多 id" → 批量(否则走单条 update-category)
  • SQL: 单条 UPDATE notes SET category = ?, updated_at = ... WHERE id = ?
  • HTML: 过程型 change_category.html
  • 硬规则: --from-category ≠ --to-category
  • 副作用: 不动 sub_category

触发词详细 · 跨 Skill

备忘录同步

用法: sync-from-feishu [--html]

本地心愿 ↔ 飞书任务双向对账(3 步)。AI 会自动加 --html 渲染对账报告给你看(11 个统计字段、3 步折叠、错误高亮)。

3 步对账:

  1. 本地补建(本地 → 飞书):本地心愿无 feishu_task_guid → 调 add_wish_sync 建飞书 task
  2. 反向同步 done(飞书 → 本地):飞书 status=done → 本地 complete-wish
  3. 反向同步 due(飞书 → 本地 · 仅 status=todo):飞书 task.due → 本地 notes.due(飞书优先 4 象限)
🔍 查看底层原理
  • 自动检测: is_feishu_available() 检查 lark-cli
  • 失败降级: 飞书 API 失败不阻塞本地(本地写入照常)
  • 报告字段: backfilled / scanned_done / synced / scanned_pending / due_added/overridden/removed / skipped_* / errors[]

子唤醒词(12 个)

子唤醒词继承主唤醒词的 CLI 调用,只在以下 3 处差异化:

维度差异
命令参数 必加 -c <顶层分类>(记心愿=心愿 / 记打卡=打卡 / 记情绪=情绪日记)
飞书 hook 仅 category == "心愿" 触发飞书 task 同步(打卡/情绪日记不触发)
子分类 自由文本 2 字(AI 智能推断)

记心愿 / 删心愿 / 改心愿 / 查心愿

等价于"记备忘/删备忘/改备忘/搜备忘"+ 必加 -c 心愿。查心愿 AI 自动 --html。

🔍 记心愿底层
  • 等价于"记备忘"+ -c 心愿
  • 飞书: add_wish_sync 自动建飞书 task(失败降级)
🔍 查心愿底层
  • CLI: search -c 心愿 "关键词" --html
  • 等价于"搜备忘" + category='心愿' 过滤

记打卡 / 删打卡 / 改打卡 / 查打卡

等价于"记备忘/删备忘/改备忘/搜备忘"+ 必加 -c 打卡。子分类 AI 智能推断(如"跑步")。查打卡 AI 自动 --html。

🔍 查打卡底层
  • CLI: search -c 打卡 "关键词" --html

记情绪 / 删情绪 / 改情绪 / 查情绪

等价于"记备忘/删备忘/改备忘/搜备忘"+ 必加 -c 情绪日记。查情绪 AI 自动 --html。

🔍 查情绪底层
  • CLI: search -c 情绪日记 "关键词" --html

定时提醒(Cron)

备忘录技能需要设置 cron 任务,每 {MEMO_CRON_INTERVAL} 分钟检查一次到期提醒,推送到 QQ(或其他已配置的消息渠道)。

  • 执行命令:python3 ${SKILL_DIR}/script/reminder_scheduler.py
  • 无提醒时:静默处理,输出 NO_REPLY,不发送任何消息
  • 有待提醒时:按格式 🔔 {内容}\n⏰ {时间} · {重复类型} 输出,通过 message 工具发送

Cron 配置示例

# 每分钟检查提醒
* * * * * python3 /mnt/d/2Study/StudyNotes/SKILLS/备忘录/script/reminder_scheduler.py
