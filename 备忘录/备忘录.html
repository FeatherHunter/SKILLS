笔记管理
添加笔记
记录一条新备忘，支持分类和媒体附件

CLI 完整语法

script/memo_cli.py add "内容" -c 分类 -m 媒体路径

执行流程

  • 校验 content 非空
  • 校验 category 在白名单内（社交/心愿/灵感/成就/工作/学习/记账/打卡/情绪/general）
  • 校验 media_path 以 MEMO_MEDIA_DIR 开头，去掉前缀存储相对路径
  • INSERT INTO notes（content, category, media_path, created_at, updated_at）
  • 触发器 notes_ai 自动同步到 notes_fts 虚拟表

涉及表 & 字段

表字段操作
notescontent, category, media_path, created_at, updated_atINSERT
notes_ftsrowid, content, category触发器同步

AI 交互规范

添加前必须先问用户确认分类，不得直接使用默认分类。示例：用户说「去医院」→ AI 问「工作还是心愿？」→ 用户选 → 写入。

搜索笔记
按关键词全文搜索，支持分类过滤

CLI 完整语法

script/memo_cli.py search "关键词" -c 分类 -l 数量

执行流程

  • 有关键词 → 使用 FTS5 全文搜索 notes_fts MATCH ?，JOIN notes 表
  • 无关键词 → 直接查 notes 表，按分类过滤
  • 按 updated_at DESC 排序，limit 截断（默认 20）

涉及表 & 字段

表字段操作

notes_ftsrowidMATCH

副作用

无。只读操作。

更新笔记
修改已有笔记的内容、分类或媒体

CLI 完整语法

script/memo_cli.py update <id> --content "新内容" -c 分类 -m 媒体 --reminder-id 提醒ID

执行流程

  • 校验 id 存在于 notes 表
  • 校验 category 白名单（如提供）
  • 拼接动态 SET 子句（只更新提供的字段）
  • UPDATE notes SET ... WHERE id = ?，同时更新 updated_at
  • 触发器 notes_au 同步更新 notes_fts（先删旧、再插新）

涉及表 & 字段

表字段操作
notescontent / category / media_path / reminder_id / updated_atUPDATE
notes_ftsrowid, content, category触发器 DELETE+INSERT

副作用

至少提供一个更新字段，否则报错。updated_at 始终自动更新。

删除笔记
删除笔记及其关联提醒（级联删除）

CLI 完整语法

script/memo_cli.py delete <id>

执行流程

  • 校验 id 存在于 notes 表
  • DELETE FROM notes WHERE id = ?
  • 外键 ON DELETE CASCADE 自动删除 reminders 中 note_id = 该 id 的所有记录
  • 触发器 notes_ad 自动删除 notes_fts 对应行

涉及表 & 字段

表字段操作
notes整行DELETE
remindersnote_id 匹配的整行CASCADE DELETE
notes_ftsrowid触发器同步

副作用

级联删除：关联的提醒会被一并删除，不可恢复。

提醒管理
设置提醒
为笔记添加定时提醒，支持一次性或周期重复

CLI 完整语法

script/memo_cli.py remind <note_id> --at "YYYY-MM-DD HH:MM" --content "提醒内容" --repeat-type 类型 --rule "规则"

执行流程

  • 校验 content 非空
  • 校验 note_id 存在于 notes（如提供）
  • 校验 --at 格式（一次性必须有 --at）
  • 校验 --rule 格式匹配 repeat_type
  • INSERT INTO reminders（note_id, remind_at, repeat_type, repeat_rule, content, created_at）

涉及表 & 字段

表字段操作
remindersnote_id, remind_at, repeat_type, repeat_rule, content, created_atINSERT

重复规则格式

类型格式示例
一次性不需要 --rule--at "2026-05-28 09:00"
每天HH:MM"09:00"
每周W HH:MM"5 17:00" 周五
每月D HH:MM"15 08:30" 每月15号
每年MM-DD HH:MM"12-25 10:00"
记提醒（添笔记 + 设提醒）
一步完成：先创建笔记，再设置提醒

执行流程

  • 调用 add 创建笔记 → 获得 note_id
  • 调用 remind 设置提醒，关联 note_id
  • 笔记内容和提醒内容分别存储（reminders.content 可不同于 notes.content）

涉及表 & 字段

表字段操作
notescontent, category, created_at, updated_atINSERT
notes_ftsrowid, content, category触发器同步
remindersnote_id, remind_at, repeat_type, repeat_rule, content, created_atINSERT

与「设提醒」的区别

「设提醒」直接创建提醒（可不关联笔记），「记提醒」强制先创建笔记再设提醒。

查看提醒
列出当前活跃或已废弃的提醒

CLI 完整语法

script/memo_cli.py reminders -s active|dismissed

执行流程

  • SELECT r.*, n.content as note_content FROM reminders r LEFT JOIN notes n
  • WHERE r.status = ?（默认 active）
  • 按 remind_at, repeat_type 排序
  • content 字段优先取 reminders.content，fallback 到 notes.content

涉及表 & 字段

表字段操作
reminders全字段SELECT
notescontentLEFT JOIN
废弃提醒

执行流程

  • 校验 id 存在于 reminders
  • 校验 status 必须是 active（不能重复废弃）
  • UPDATE reminders SET status = 'dismissed' WHERE id = ?

remindersstatusUPDATE

副作用

查询已完成提醒
查看已完成的打卡记录，按提醒类型匹配周期

执行流程

  • SELECT FROM notes JOIN reminders WHERE notes.category = '打卡' AND reminders.status = 'active'
  • LEFT JOIN notes（原笔记）获取原始内容
  • 按 repeat_type 匹配周期：一次性需 notified_at 存在；每天/每周/每月/每年检查打卡时间是否在当前周期内

涉及表 & 字段

表字段操作
notesid, content, category, created_at, reminder_idSELECT
remindersid, repeat_type, repeat_rule, notified_at, content, note_idSELECT

完成匹配逻辑

类型完成条件
一次性notified_at 存在 + 有打卡笔记
每天打卡日期 == 今天
每周打卡在当前自然周 + 星期匹配
每月打卡在当月 + 日期匹配
每年打卡在年内 + 月日匹配
系统机制
定时提醒调度
cron 每隔 N 分钟调用，检查到期提醒并推送

执行流程

  • 三层查找 DB 路径：环境变量 SKILLS_DB_PATH → 父目录 .db → 技能目录 .db
  • 调用 memo_cli.py due 获取到期提醒
  • 有提醒 → 输出格式化消息 🔔 内容 ⏰ 时间 · 类型
  • 无提醒 → 输出 NO_REPLY（静默，不推送）

触发时机

由 cron job 每 {MEMO_CRON_INTERVAL} 分钟触发。提前 {MEMO_ADVANCE_MINUTES} 分钟预通知，准点触发 + {GRACE_PERIOD} 分钟容错窗口。

due 命令内部逻辑

  • 查询所有 active 提醒，LEFT JOIN notes
  • 一次性：判断 remind_at 日期 == 今天，检查触发窗口
  • 循环提醒：按 repeat_rule 解析，判断是否在当天/当周/当月/当年周期内
  • 新周期开始时重置 notified_at
  • 触发后：提前触发 → 设置 notified_at；准点触发 → 一次性标记 dismissed，循环只设 notified_at

数据库结构
SQLite + WAL 模式，两张核心表 + FTS5 全文索引

FTS5 虚拟表 & 触发器

notes_fts 使用 FTS5 全文索引 content 和 category 字段，通过三个触发器（INSERT/UPDATE/DELETE）自动同步。

索引

idx_notes_category（category）、idx_reminders_status_remind（status, remind_at）
