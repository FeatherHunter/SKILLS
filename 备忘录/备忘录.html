添加笔记
记录一条新备忘，支持顶层分类 + 子分类 + 媒体附件

CLI 完整语法

script/memo_cli.py add "内容" -c 顶层分类 -s 子分类 -m 媒体路径 --tasklist-guid 飞书清单GUID

--tasklist-guid：仅当 category=心愿 时生效。指定飞书 tasklist GUID，task 进指定清单；不传则飞书 task 进"我的任务"主页。

顶层分类（4 种）

分类语义
备忘想到什么记什么（默认）
心愿任务/待办（可被 complete-wish）
打卡行为记录（已完成的事情）
情绪日记情绪状态记录

子分类（自由文本字段）

1 个，2 字。AI 智能从用户原话推断内容维度（如"工作"/"学习"/"跑步"/"社交"等），推断不出 → NULL。适用于所有顶层分类，不预设任何白名单。

执行流程

  • 校验 content 非空
  • 校验顶层分类在白名单内（备忘/心愿/打卡/情绪日记，默认 备忘）
  • 如 AI 推断出 sub_category（≤2 字），写入；否则 NULL（不做白名单校验）
  • 校验 media_path 以 MEMO_MEDIA_DIR 开头，去掉前缀存储相对路径
  • INSERT INTO notes（content, category, sub_category, media_path, created_at, updated_at）
  • 触发器 notes_ai 自动同步到 notes_fts 虚拟表

完成心愿流式工作流
原子删除心愿 + 新建打卡，两步必须同时成功

CLI 完整语法

script/memo_cli.py complete-wish <心愿id> --content "打卡内容"

执行流程

  • 校验 id 存在且 category='心愿'（不是心愿分类则报错）
  • 决定打卡 content：用户提供 → 用用户的；没提供 → 拷贝原心愿 content
  • 事务原子执行（兼容 NO ACTION / CASCADE 两种 FK 行为）：
    • DELETE FROM reminders WHERE note_id = ?（先删提醒，避开 FK 约束）
    • DELETE FROM notes WHERE id = ?
    • INSERT INTO notes (content, category='打卡', created_at, updated_at)
  • 返回结果：{deleted_wish_id, created_checkin_id, checkin_content}

涉及表 & 字段

表字段操作
remindersnote_id 匹配的整行DELETE
notes原心愿整行DELETE
notescontent, category='打卡', created_at, updated_atINSERT
notes_fts对应行触发器同步

副作用

原子事务：失败回滚，心愿和打卡要么都成功要么都没变。不联动飞书（飞书联动是后续 Phase 2 工作）。

设计取舍

  • 不写 reminder_id：CASCADE 删 reminders 后该字段会悬空，留 NULL 更干净
  • 硬删除：流式工作流意味着心愿生命终结于"完成"那一刻
  • 兼容 NO ACTION FK：手动先删 reminders，不论 FK 是 NO ACTION 还是 CASCADE 都安全

与旧「完成提醒：提醒与打卡的完整流程」的关系

维度旧流程（4 步手动）新流程（1 步原子）
操作add → remind → 关联 → 打卡追加complete-wish
原心愿保留删除
打卡新建（关联 reminder_id）新建（不关联 reminder_id）
追溯通过 reminder_id通过 content 关键词 + 时间窗

推荐用新流程。

设计原则

  • 无开关：自动检测 CLI 是否安装，不靠环境变量
  • 零配置可用：add 心愿不传 --tasklist-guid → 飞书 task 进"我的任务"主页
  • 少用场景：显式传 tasklist：用户偶尔想分到指定 tasklist 时，AI 跑 list-tasklists 列飞书清单 → 用户选 → AI 传 --tasklist-guid
  • 本地优先：飞书 API 失败不阻塞本地操作（仅 stderr 记录 warning）
  • 仅心愿联动：备忘/打卡/情绪日记不联动飞书（不必要）
  • 需要 task_guid：add 触发时自动建 + 写回 notes.feishu_task_guid；update/delete/complete-wish 依赖这个字段
  • memo_id 编码进 description：飞书 task create 不支持 extra 字段，用 description="原备忘 #N" + 正则反查
反向同步（飞书 → 本地）
主动拉飞书已完成 task，反查 memo_id → 触发本地 complete-wish

执行流程

  • 拉飞书所有 status=done 的 task（task +get-related-tasks）
  • 对每个 done task 从 description 反查 原备忘 #N 提取 memo_id（task tasks get --task-guid）
  • 反查本地：SELECT id, category FROM notes WHERE id = ? AND feishu_task_guid = ?
  • 如果 category=心愿 且本地还在 → 触发 complete-wish memo_id
  • 输出报告：scanned / synced / skipped

为什么需要这个

V2 联动只能处理"本地改 → 飞书改"。如果用户在飞书 App 手动 complete task，本地心愿 note 不会被删除（状态不一致）。

用户说"备忘录同步飞书"主动触发反向同步，保持两边一致。

不起定时任务

设计上选择"用户主动触发"而非"cron 定时 polling"——避免无人值守时的意外操作。

系统机制
定时提醒调度
cron 每隔 N 分钟调用，检查到期提醒并推送

执行流程

  • 三层查找 DB 路径：环境变量 SKILLS_DB_PATH → 父目录 .db → 技能目录 .db
  • 调用 memo_cli.py due 获取到期提醒
  • 有提醒 → 输出格式化消息 🔔 内容 ⏰ 时间 · 类型
  • 无提醒 → 输出 NO_REPLY（静默，不推送）

触发时机

由 cron job 每 {MEMO_CRON_INTERVAL} 分钟触发。提前 {MEMO_ADVANCE_MINUTES} 分钟预通知，准点触发 + {GRACE_PERIOD} 分钟容错窗口。

due 命令内部逻辑

  • 查询所有 active 提醒，LEFT JOIN notes
  • 一次性：判断 remind_at 日期 == 今天，检查触发窗口
  • 循环提醒：按 repeat_rule 解析，判断是否在当天/当周/当月/当年周期内
  • 新周期开始时重置 notified_at
  • 触发后：提前触发 → 设置 notified_at；准点触发 → 一次性标记 dismissed，循环只设 notified_at

SQLite + WAL 模式，两张核心表 + FTS5 全文索引

notes 表

字段类型说明
idINTEGER PK自增主键
contentTEXT NOT NULL笔记内容
summaryTEXTAI 摘要，可空
categoryTEXT顶层分类：备忘/心愿/打卡/情绪日记（默认 备忘）
sub_categoryTEXT子分类（自由文本字段）：AI 智能从用户原话推断 1 个 2 字（如"工作"/"学习"/"跑步"等任意值），适用于所有 category；推断不出 → NULL，不预设白名单
media_pathTEXT附件相对路径
reminder_idINTEGER关联提醒 ID（提醒完成后可追溯来源）
feishu_task_guidTEXT飞书 task GUID（心愿同步飞书时记录，用于反向查找）
created_atTEXT创建时间
updated_atTEXT最后更新时间

reminders 表

字段类型说明
idINTEGER PK自增主键
note_idINTEGER FK关联笔记 ID，真实 DB 是 NO ACTION（init.sql 描述为 CASCADE）
remind_atTEXT一次性提醒时间
repeat_typeTEXTnone/daily/weekly/monthly/yearly
repeat_ruleTEXT重复规则
statusTEXTactive / dismissed
notified_atTEXT上次通知时间
contentTEXT提醒内容（可不同于 notes.content，提醒独有）
created_atTEXT创建时间

FTS5 虚拟表 & 触发器

notes_fts 使用 FTS5 全文索引 content 和 category 字段，通过三个触发器（INSERT/UPDATE/DELETE）自动同步。

索引

idx_notes_category（category）、idx_reminders_status_remind（status, remind_at）

环境变量配置
数据库路径、媒体目录、定时参数均可通过环境变量自定义
变量说明默认值
SKILLS_DB_PATH数据库根目录向上查找 .db/
MEMO_MEDIA_DIR媒体文件目录media
MEMO_CRON_INTERVALcron 执行间隔（分钟）2
MEMO_ADVANCE_MINUTES提前触发分钟数10
MEMO_GRACE_MULTIPLIER延后窗口倍数2
