# 基础复习题型规则

## 一、stage × 轮次 矩阵

| 知识 stage | R1（5 题） | R2（8 题） | R3（12 题） | R4（1 项目） | R5（6 题） |
|-----------|-----------|-----------|------------|------------|-----------|
| **stage1-2** | 概念回忆 ×4 + 反直觉 ×1 | 概念 ×2 + 填空 ×3 + 代码 ×2 + 解释 ×1 | 概念 ×2 + 填空 ×3 + 代码 ×3 + 解释 ×1 + 交错 ×1 | 实战项目 | 解释 ×1 + 代码 ×1 + 交错 ×4 |
| **stage3-4** | 概念回忆 ×3 + 反直觉 ×1 + 源码填空 ×1 | 概念 ×2 + 填空 ×2 + 代码 ×2 + 解释 ×1 + 调用链 ×1 | 填空 ×2 + 代码 ×3 + 解释 ×2 + 调用链 ×1 + 交错 ×2 | 实战项目 | 解释 ×1 + 代码 ×1 + 调用链 ×1 + 交错 ×3 |

**查表方法**：取 `current_stage >= 3` 用第二行，否则用第一行。

## 二、题型定义

| 题型 | 出题依据 | 评分标准 |
|------|---------|---------|
| **概念回忆** | entity 页的定义和特性 | 核心概念准确即满分 |
| **反直觉** | entity 页的误区/常见错误 | 选对即满分 |
| **填空** | entity 页的关键机制（如"volatile 写后插入 ___ 屏障"） | 空值精确匹配 |
| **代码题** | raw notes 或 sources 中的代码示例，变参/变形出题 | 功能正确 50%+ 最佳实践 30%+ 错误处理 20% |
| **解释题** | entity 页的原理章节，要求用自己的话说 | 核心概念准确 1/3 + 逻辑清晰 1/3 + 无重大错误 1/3 |
| **调用链** | sources 中的调用链/入口文件，要求写出 A→B→C | 关键节点不漏即满分 |
| **交错题** | 选 tags 最相似的另一个已完成知识点，出对比题 | 两知识区分正确即满分 |
| **源码填空** | sources 中的关键代码段，空掉关键行或参数 | 空值精确匹配 |

## 三、出题规则

### 规则 1：题目必须来自 Wiki

```
出题前执行 LLM-WIKI QUERY：
  → 得到 entity 页 + raw notes + sources 片段

每道题标注"出处"：entity/{知识点}.md 的「{章节}」或 raw/sources/{文件名}
```

### 规则 2：内容不足以支撑时降级

```
如果 QUERY 返回的内容无法支撑某道题的要求：
  → 降为该轮次中内容最低要求的题型，或减少该题
  → 在步骤 7 写回时标记该知识"内容缺失"，提示用户 Ingest
```

### 规则 3：source 阶段优先用 sources 片段

```
当前知识的 current_stage >= 3（即学到源码追踪阶段）时：
  → 调用链题和源码填空题必须取自 raw/sources/ 中标注过的片段
  → 不可凭空编造调用链
```

### 规则 4：薄弱点覆盖

```
如果步骤 2 中检测到该知识有未解决的薄弱点：
  → R2/R3 的代码题或解释题优先覆盖薄弱点 topic
  → 题目标记 [薄弱] 提醒用户
```

## 四、代码题文件规则

```
文件路径：exercises/{knowledge_id}_R{round}_{n}.{ext}
扩展名：执行 CLI：python3 learning.py knowledge get <id> 获取 language 字段
文件内容：题干注释 + TODO 标记 + 模板代码
生命周期：AI 创建 → 用户编辑 → 提交/跳过 → 评分后手动删除
```
