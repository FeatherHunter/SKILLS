# 复习设计原理参考

## 原理 → 设计 映射表

| 原理 | 一句话 | 应用到哪 | 为什么 |
|------|--------|---------|--------|
| **预测误差** | 猜错比猜对记得更牢 | R1 反直觉题 | 预测试触发认知惊讶，错误预测加深编码 |
| **交错练习** | 混着练比单练记得牢 | R3/R5 交错题 | 混合不同知识点，提高辨别能力，防死记硬背 |
| **自我解释** | 说给自己听比看更有效 | R2/R3/R5 解释题 | 深度加工，用自己的话重新编码 |
| **加工深度** | 写代码是深层，选 ABCD 是浅层 | R3/R5 代码题 | 生成效应——主动产出的记忆强度远大于被动识别 |
| **编码特异性** | 和真实场景越像，迁移能力越强 | R4 实战项目 + R5 场景化 | 学习时的情境和用时的情境一致，记忆提取更容易 |
| **成功体验** | 前 30% 确保 80% 正确率，建立信心 | 全轮次题目排序 | 小成功激励持续，挫败感导致放弃 |
| **间隔重复** | 1→3→7→14→30 天递减 | 轮次设计 | 艾宾浩斯遗忘曲线：刚学完遗忘最快，随间隔增大遗忘速度减慢 |
| **即时反馈** | 错了立刻知道对的是什么 | 即时验证 | 错误不被纠正会固化，即时反馈阻断错误记忆强化 |

## 题目排序原则

```
前 30% 题 → 确保 80% 正确率（建立信心）
中间 50% → 有挑战但可完成（维持动力）  
后 20% → 有难度（激发探索欲）
```

## 即时验证原理

```
错题数 >= 2 时触发 → 元认知监控
讲解后问"你觉得理解了吗？" → 元认知反思
变参验证题 → 测试效应 + 生成效应
最多 3 次 → 认知负荷理论（避免疲劳）
失败归因于知识点难度 → 成长型思维
```

## 薄弱点利用原理

```
每次复习前查上次的 verification → 薄弱点在记忆中最脆弱
本次复习优先覆盖 → 提取练习效应——反复从记忆中提取比重复阅读更有效
```
