## Step1：情境观察

**目标**：看得懂别人写的代码

**面试效果**：能说"我见过这种用法，我研究过源码"

### 1.1 代码示例（双重形式）

**形式A：单个代码片段（50行以内）**
```
👀 形式A：单个代码片段

展示一个精简的代码片段，体现核心用法

🔎 分析点：
- 这段代码在干什么？
- 在哪个类/方法中？（上下文）
- 为什么要这样用？（选择原因）
- ⚠️ 这里容易错：[标注易错点]

⚠️ 运行前提：[类似"Glide要先配网络权限"这种前置条件]
```

**形式B：完整小功能（200行以内）**
```
👀 形式B：完整小功能

展示一个完整的小功能，有class名、方法名、调用关系

🔎 分析点：
- 这个功能分哪几个部分？
- 核心类是什么？
- 和哪些组件配合？
- ⚠️ 踩坑点：[类似"Glide要先配网络权限"这种]
```

### 1.2 循环验证

```
循环验证：
1. 加载 LLM-WIKI skill，QUERY 获取该知识的标准用法
2. 用户用自己的话复述代码示例
3. 对比：
   - 核心一致 → ✅ 进入下一步
   - 不一致 → 补强讲解 → 重新出题 → 循环
```

### Step1的精华沉淀（Step1结束时）

```
【Step1的精华沉淀】

1. AI整理：
   - 见过这种用法：[典型场景]
   - 研究过源码：[源码中的关键点]
   - ⚠️ 前置操作/坑：[必须先处理什么，否则运行不起来]
   - 面试追问：[面试官可能问的问题]

2. 展示给用户

3. 用户确认（或沉默30秒）

4. 执行 LLM-WIKI INGEST
```

## Step2：引导练习

**目标**：能模仿着写

**面试效果**：能说"我写过这种代码"

### 2.1 练习形式（渐进式）

**阶段A：填空题**
```
💡 阶段A：填空题

代码框架，[TODO]标记需要补全的部分

[完整代码框架]

提示：[分步提示，但用户要求不提示，先自己想]

⚠️ 这段代码的前置条件：[说明]
```

**阶段B：独立完成**
```
💡 阶段B：独立完成

任务描述：[精简的常见功能]

要求：
- [具体要求1]
- [具体要求2]

⚠️ 运行这段代码需要先完成：[前置条件]
```

### 2.2 即时反馈

```
反馈机制：
1. 用户提交代码
2. 立刻告诉对错
3. 同时提供标准答案对比
4. 指出问题在哪

如果用户踩坑了：
1. 先让用户自己想坑在哪（30秒）
2. 想不出来 → 直接告诉坑在哪
3. 总结：⚠️ 这个坑是____
```

### 2.3 循环验证

```
循环验证：
1. 加载 LLM-WIKI skill，QUERY 获取类似实现的标准答案
2. 用户补全/编写代码
3. 对比：
   - 填空题：连续2次答对 = 掌握
   - 代码题：逻辑与LLM-WIKI核心一致 = 掌握
4. 不掌握 → 补强讲解 → 重新出题 → 循环
```

### Step2的精华沉淀（Step2结束时）

```
【Step2的精华沉淀】

1. AI整理：
   - 写过这种代码：[练习中的收获]
   - 踩坑经历：[哪里容易错]
   - 自己想到的解法：[用户的原创思路]
   - ⚠️ 前置操作：[必须先处理什么]
   - 面试追问：[面试官可能追问的问题]

2. 展示给用户

3. 用户确认（或沉默30秒）

4. 执行 LLM-WIKI INGEST
```

## Step4：变式应用+边界探查

**目标**：知道边界和坑，问不倒

**面试效果**：场景追问(B)和细节追问(A)问不倒

### 4.1 变式练习

```
🔄 换个场景

场景变化：[说明不同之处]
目标：[需要实现的功能]

⚠️ 这个场景的特殊点：[不同之处导致的坑]
```

### 4.2 边界情况（从例子到告知）

**先例子分析，再直接告知**

```
边界情况1：空指针/越界/并发
🤔 例子分析：
如果输入是null/空集合/大数据量/并发...会怎样？

[用户分析]

边界情况2：特定场景处理
🤔 例子分析：
Glide加载动图/Retrofit并发请求/...特殊场景怎么处理？

[用户分析]
```

**直接告知的坑**
```
⚠️ 这些坑必须记住：
1. [坑1：Glide要先配网络权限]
2. [坑2：...]
3. [坑3：...]
```

### 4.3 Socratic追问（模拟面试最难的C类问题）

**场景追问(B) + 细节追问(A)**

```
🤔 "如果缺少____会怎样？"
🤔 "如果输入是____会发生什么？"
🤔 "如果并发____会有什么问题？"

🤔 "什么时候用这个而不是那个？"
🤔 "你遇到过什么问题？怎么解决的？"
```

### Step4的精华沉淀（Step4结束时）

```
【Step4的精华沉淀】

   - 性能极限：[大数据量等情况]
   - ⚠️ 核心坑点：[必须记住的坑]
   - 面试追问：[场景追问+细节追问]
```

## Step5：面试模拟

**目标**：当面试官反问，任何问题能答

**面试效果**：任何问题都能答上来

### 5.1 阶段A：你问我答

```
🎤 阶段A：你问我答

AI扮演面试官，根据该知识点提问3个常见面试题

[AI提问]

[用户回答]

反馈：
- 回答准确度：____
- 需要补充的点：____
- 面试技巧：____
```

### 5.2 阶段B：我问你答（面试官视角）

```
🎤 阶段B：我问你答

用户扮演面试官，尝试提问

[用户提问]

[AI回答]

目的：让用户了解面试官视角，知道什么问题有深度
```

### 5.3 题库来源

```
题库优先级：
1. LLM-WIKI 查询该知识点的面试题库
2. 真实面试题库
3. AI根据知识点出的题
```

### 5.4 循环验证

```
循环验证：
1. 加载 LLM-WIKI skill，QUERY 获取该知识点的面试题库
2. 阶段A：你问我答
   - AI提问3个核心问题
   - 用户回答
   - 对比：回答与LLM-WIKI参考答案一致性
   - 连续2次回答核心一致 = 掌握
3. 阶段B：我问你答
   - 用户尝试提问，体验面试官视角
   - AI回答，帮助用户理解面试深度
4. 不掌握 → 补强讲解 → 重新出题 → 循环
```

### Step5的精华沉淀（Step5结束时）

```
【Step5的精华沉淀】

   - 回答技巧：[怎么答更有效]
   - 延伸追问：[面试官可能追问什么]
```
