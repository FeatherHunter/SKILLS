**达到Level**：Level 3（能说为什么）+ Level 4部分（了解实现）

- [ ] 步骤4：主动回忆

### 步骤1 | 设计动机追溯

【使用技能 baoyu-diagram 生成 原理故事图】

**苏格拉底式提问（推荐使用）**：

| 类型 | 使用场景 |
|------|----------|
| Trace | 追溯因果链：为什么要这样做？ |
| Assumption | 探查隐含假设 |
| Reverse | 逆向思考：不这样做的代价 |
| Analogy | 和已学知识建立联系 |
| Evidence | 支撑设计决定的证据/理由 |
| Implication | 这个设计还会导出什么后果/Trade-off |

**不推荐**：Judgment（原理探究阶段是理解，不是评判）

---

**流程**：

1. 【确定轮次】调用 LLM-WIKI QUERY，分析该知识点包含几个独立原理，每个原理 = 1轮
2. 【每轮开始】调用 LLM-WIKI QUERY 获取本轮原理的要点
3. 【苏格拉底引导】依次使用以下6种提问类型，每种至少问1个问题：
   - Trace：追溯因果链（为什么要这样设计？）
   - Assumption：探查隐含假设（这个设计的假设是什么？）
   - Reverse：逆向思考（不这样做的代价是什么？）
   - Analogy：类比迁移（这和你学过的哪个知识类似？）
   - Evidence：证据追溯（支撑这个设计的理由是什么？）
   - Implication：后果推断（这个设计还会导出什么？）
4. 【验证】出1道问答题，用户用自己的话回答
5. 【比对】调用 LLM-WIKI QUERY 获取标准答案，逐点比对用户作答与标准答案
6. 【判定】
   - 全部要点都对 → 本轮完成，进下一轮（如有）
   - 有遗漏 → 展示标准答案对照 → 回到步骤3继续本轮
7. 【全部轮次完成后】执行精华沉淀

**精华沉淀**：

1. 【整理】AI按以下6项逐一整理：
   - 因果链突破：用户最终理解的那条"为什么这样设计"的因果线
   - 精彩追问：用户自己问出的好问题（证明理解深度）
   - 模糊点澄清：过程中的困惑+最终如何想通
   - 类比：用户自创的类比
   - 关键记忆点：用户提取的关键词
   - 薄弱点+解决：说不清楚的地方及解决过程
2. 【展示】将整理结果展示给用户
3. 【确认】用户确认无误
4. 【执行】调用 LLM-WIKI INGEST，将"全部内容"作为素材

### 步骤2 | 对比分析

**目标**：找到与已学知识的相似和不同，在知识体系中建立位置感

**流程**：

1. 【确定对比组】调用 LLM-WIKI QUERY，查询与当前知识点相关的 entities（同一 category 或有 Relationships 的知识点），选择1-2组最相关的作为对比对象
2. 【因果链衔接】先回顾步骤1建立的因果链："当前设计在[XX维度]做了[YY取舍]，因此产生了[ZZ效果]"
3. 【用户初答】问用户："你觉得A和B有什么相似和不同？"（用户凭直觉回答，不要求准确）
4. 【系统补充】调用 LLM-WIKI QUERY 获取对比知识点的核心要点，展示给用户
5. 【共同构建】与用户一起填写对比表：
   - 维度1：原理/机制
   - 维度2：优势
   - 维度3：局限/代价
   - 维度4：适用场景
6. 【验证】出1道问答题（"在XX场景下，应该选A还是B？为什么？"），用户用自己的话回答
7. 【比对】调用 LLM-WIKI QUERY 获取标准答案，逐点比对
8. 【判定】
   - 全部要点都对 → 本组完成，进下一组对比（如有）
   - 有遗漏 → 展示标准答案对照 → 回到步骤6继续本组
9. 【全部组完成后】执行精华沉淀

**精华沉淀**：

1. 【整理】AI按以下4项逐一整理：
   - 异同点总结：A和B的核心差异
   - 场景选择结论：什么场景选什么
   - 记忆技巧：用户自己想出来的记忆方法/口诀
   - 有趣洞察：过程中的意外发现
2. 【展示】将整理结果展示给用户
3. 【确认】用户确认无误
4. 【执行】调用 LLM-WIKI INGEST，将"全部内容"作为素材

5. 【自动关联建立】对比完成后，AI 建议将对比关系写入 entity 页：
   ```
   对当前知识点的 entity 页 frontmatter 写入：
   related_to:
     - knowledge_id: [对比对象知识点ID]
       relation: similar 或 alternative（根据对比结论判断）
       established_at: [今天日期]

   展示建议 → 用户确认 → 执行 LLM-WIKI INGEST（更新 entity 页）
   ```

### 步骤3 | 本质提炼

**目标**：透过现象看本质，提炼跨领域的共同规律

**流程**：

1. 【回顾基础】回顾步骤1的因果链和步骤2的异同点总结
2. 【初步提炼】问用户："这个知识的本质是什么？用一句话说"
3. 【探索本质】调用 LLM-WIKI QUERY 获取知识点核心，层层追问：
   - "这个本质背后更底层的是什么？"
   - "这个本质在其他领域也存在吗？找1-2个类比"
4. 【提炼多个本质】与用户共同提炼1-3个相关本质（层层深入）
5. 【验证】每个本质出1道问答题，用户用自己的话回答
   - 题目来源：调用 LLM-WIKI QUERY 获取该本质的标准表述
6. 【比对+判断】
   - 逐点比对：调用 LLM-WIKI QUERY 获取标准答案，对照是否说到要点
   - 主观判断：AI补充"用户是否真正抓住了本质"
7. 【判定】
   - 全部要点都对且抓住本质 → 完成
   - 有遗漏（客观+主观）→ 展示标准答案对照 → 回到步骤5继续验证错误的本质
8. 【完成后】执行精华沉淀

**精华沉淀**：

1. 【整理】AI按以下4项逐一整理：
   - 提炼的本质：用户提炼的1-3个本质（层层深入的结构）
   - 跨领域类比：其他领域印证该本质的例子
   - 抽象过程：用户从现象到本质的思考路径
   - 模糊点澄清：过程中的困惑+最终如何想通
2. 【展示】将整理结果展示给用户
3. 【确认】用户确认无误
4. 【执行】调用 LLM-WIKI INGEST，将"全部内容"作为素材

### 步骤4 | 主动回忆

**目标**：验证前三步学习效果，强化长期记忆

**流程**：

1. 【告知】告诉用户："现在不看笔记，请用自己的话复述：因果链、异同点、本质"
2. 【用户复述】用户完整复述
3. 【验证】出3道问答题，每个维度1道：
   - 维度1：因果链（为什么这样设计）
   - 维度2：异同点（与其他知识的对比）
   - 维度3：本质（跨领域的共同规律）
   - 题目来源：调用 LLM-WIKI QUERY 分别获取三个维度的标准表述，各出1道验证题
4. 【比对+判断】
   - 逐点比对：调用 LLM-WIKI QUERY 获取标准答案，对照是否说到要点
   - 主观判断：AI补充"用户表达是否准确抓住了本质"
5. 【判定】
   - 全部要点都对且主观判断通过 → 完成
   - 有遗漏 → 展示标准答案对照 → 回到步骤3继续验证答错的题
   - 有重大遗漏（复述内容缺失大段）→ 回到步骤2重新复述
6. 【完成后】执行精华沉淀

**精华沉淀**：

1. 【整理】AI按以下2项逐一整理：
   - 薄弱点：过程中的困惑+未说清楚的点
   - 新知识：过程中发现的新的理解角度
2. 【展示】将整理结果展示给用户
3. 【确认】用户确认无误
4. 【执行】调用 LLM-WIKI INGEST，将"全部内容"作为素材

### 步骤5 | 实现探究

**目标**：了解底层实现方式，先猜后学验证理解

**流程**：

1. 【确定轮次】调用 LLM-WIKI QUERY："这个知识有多少个独立的实现维度？分别是哪些？"每个维度 = 1轮
2. 【每轮开始】问用户："你觉得[维度X]的底层是怎么实现的？"
3. 【用户预测】用户凭直觉预测实现方式
4. 【展示实现】调用 LLM-WIKI QUERY 获取该维度的实际实现
5. 【对比差异】调用 LLM-WIKI QUERY 获取定性描述，引导用户对比预测与实际的差距
6. 【验证】出1道问答题验证理解，题目来源：调用 LLM-WIKI QUERY 获取标准表述
7. 【判定】
   - 答对 → 本轮完成，进下一轮（如有）
   - 答错 → 调用 LLM-WIKI QUERY 获取更深入的描述 → 回到步骤4 → 回到步骤5对比 → 回到步骤6再验证
8. 【全部轮次完成后】执行精华沉淀

**精华沉淀**：

根据知识类型沉淀不同内容：

| 知识类型 | 精华沉淀重点 |
|----------|-------------|
| 开源库/框架 | 设计模式、精彩设计、架构取舍 |
| 语言特性 | 底层机制思想、JVM/编译器处理方式 |
| 数据结构 | 核心思想、时间空间复杂度分析 |
| 算法 | 核心思想、优化思路、应用场景 |
| 设计模式 | 典型应用场景、权衡取舍 |
| 协议/标准 | 设计哲学、约束与扩展性 |
| 其他 | 根据知识特点灵活沉淀：聚焦"实现层面最有价值的东西" |

1. 【整理】AI按该知识类型对应的沉淀重点进行整理
2. 【展示】将整理结果展示给用户
3. 【确认】用户确认无误
4. 【执行】调用 LLM-WIKI INGEST，将"全部内容"作为素材

### 阶段3 精华沉淀（阶段性）

【可选：使用技能 baoyu-infographic 生成 原理全景图】

**目标**：将步骤1-5的精华串成完整的知识线

**流程**：

1. 【整合】AI将步骤1-5的精华整合：因果链→异同→本质→记忆→实现
2. 【展示】将整合结果展示给用户
3. 【确认】问用户："需要沉淀吗？"用户确认
4. 【执行】如用户确认 → 调用 LLM-WIKI INGEST；如用户认为没必要 → 跳过
