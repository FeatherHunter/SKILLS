## 阶段5：实战深化

**目标**：积累真实使用经验，能举出具体案例

**达到Level**：Level 5（能举案例）

**面试效果**：面试问"你在项目中使用过XXX吗？讲个例子"，能流畅讲述场景+问题+方案+结果

**进度跟踪**：
```
- [ ] 步骤1：案例学习
- [ ] 步骤2：模拟项目
- [ ] 步骤3：踩坑复盘
- [ ] 步骤4：案例沉淀
- [ ] 步骤5：经典问题预判
- [ ] 步骤6：面试回答框架训练
- [ ] 步骤7：精华沉淀汇总
- [ ] S5_GATE：阶段门控检查
```

**AI 强制约束**：
- 每完成一个步骤，在对话中更新对应 checkbox 为 `[x]`
- GATE 前必须汇报当前进度

| 维度 | 设定 |
|------|------|
| **角色** | 终面评审官 |

## 步骤6：面试回答框架训练

### 面试六步回答框架

```
🎤 面试六步回答框架：

当被问到一个问题时，按以下步骤回答：

Step 1: 澄清（如果需要）
"你说的 XXX 是指...对吗？"

Step 2: 结论先行
"我认为...（核心答案）"

Step 3: 原因解释
"因为...（2-3个理由）"

Step 4: 证据支撑
"具体来说，源码中可以看到... / 项目中遇到过... / 数据表明..."

Step 5: 承认边界
"不过这个方案也有局限，比如...（1-2个）"

Step 6: 权衡分析（如果问方案选择）
"如果不用这个方案，另一个选择是...，
 权衡点是...（时间/空间/复杂度/可维护性）"
```

### 训练流程

```
AI扮演面试官出题：
"能给我讲一个你在项目中使用XXX的实际案例吗？"

用户按6步框架回答：
1. 澄清 → 2. 结论先行 → 3. 原因 → 4. 证据 → 5. 边界 → 6. 权衡

AI评估（使用检查清单）：
- [ ] 是否澄清了问题？
- [ ] 是否结论先行？
- [ ] 原因是否清晰（2-3个）？
- [ ] 是否有具体证据？
- [ ] 是否承认了局限？
- [ ] 是否分析了权衡？

未完整回答 → 继续练习，直到完整回答6个步骤
```

**交互要点**：
- 用户先尝试按框架回答
- AI评估并给出具体反馈
- 用户修正后再次练习
- 练到能完整回答为止

## 阶段完成后的 related_to 关联写入

阶段5完成时，AI 应检查步骤4中涉及的替代技术/对比方案，自动建议 `related_to` 关联：

```
1. 回顾步骤4的案例：该案例涉及哪些替代方案的对比？
2. 对每个替代方案对应的知识点，在 entity 页 frontmatter 写入：
   related_to:
     - knowledge_id: [替代方案知识点ID]
       relation: alternative
       established_at: [今天日期]
3. 展示建议的关联给用户
4. 用户确认后 → 执行 LLM-WIKI INGEST（更新 entity 页 frontmatter）
```
