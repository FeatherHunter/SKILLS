## 阶段7：迁移设计

**目标**：综合运用知识，进行架构思考和设计

**达到Level**：Level 7（能延伸设计）

**面试效果**：面试问"如果让你设计XXX，你会怎么做？"，能画出架构图+说清设计原则+权衡决策

**进度跟踪**：
```
- [ ] 步骤1：跨知识关联
- [ ] 步骤2：设计思考
- [ ] 步骤3：架构分析
- [ ] 步骤4：创新组合
- [ ] 步骤5：设计验证
- [ ] 步骤6：精华沉淀
- [ ] S7_GATE：阶段门控检查
```

**AI 强制约束**：
- 每完成一个步骤，在对话中更新对应 checkbox 为 `[x]`
- GATE 前必须汇报当前进度

### ADR（架构决策记录）产出

**这是整个阶段7的核心产出物**。面试三面（架构师面）的核心素材。

```
【ADR 产出】

AI 将步骤 1-5 的架构思考整合为 ADR：

## 架构决策记录：[决策标题]

### 背景
[需要做什么，为什么需要做这个决策]

### 方案对比
| 维度 | 方案A（选了） | 方案B | 方案C |
|------|------------|-------|-------|
| 性能 | ... | ... | ... |
| 复杂度 | ... | ... | ... |
| 扩展性 | ... | ... | ... |
| 可维护性 | ... | ... | ... |
| 社区/生态 | ... | ... | ... |

### 决策
选了 **方案A**，因为 [1-2 个核心原因]

### 放弃的
- 放弃了方案B的 [优势]，因为 [为什么这个场景下这点不重要]
- 放弃了方案C的 [优势]，因为 [权衡考量]

### 风险 & 缓解
| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|---------|
| [风险1] | 中 | 高 | ... |
| [风险2] | 低 | 高 | ... |

### 复审时间
建议在 [条件] 时重新评估此决策

## 阶段完成后的 related_to 关联写入

阶段7完成时，AI 应将步骤1中建立的跨知识关联网络写入 entity 页：

```
1. 提取步骤1（跨知识关联）中建立的所有关联关系
2. 区分关系类型：
   - 同类知识（similar）
   - 上下游知识（prerequisite / depends_on）
   - 设计模式关联（similar / composed_of）
3. 对每个关联目标，在当前 entity 页 frontmatter 写入：
   related_to:
     - knowledge_id: [关联知识点ID]
       relation: [prerequisite | similar | composed_of | depends_on]
       established_at: [今天日期]
4. 对关联目标 entity 页也写入反向关联（如有必要）
5. 展示建议的关联网络给用户
6. 用户确认后 → 执行 LLM-WIKI INGEST（更新相关 entity 页 frontmatter）
```
