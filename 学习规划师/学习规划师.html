学习规划师

制定高效学习计划，数据驱动，简洁执行

16 个唤醒词 7 个功能分类 cron 每日 08:00 自动触发

强制性规定（优先级最高）

  1. 同步更新：该技能的所有优化和变动、脚本的所有变动都必须体现在本 HTML 文件上
  2. 优先级：本规定优先级高于技能文件中所有其他规则
  3. 修改确认：对该技能的所有文件、脚本的任何一行修改都需要明确得到用户的 1 次确认
输入关键词筛选，如「百日」「JD」「优化」
智能入口
🧠
制定学习计划
智能判断缺失层级，按优先级自动补全
制定学习计划
CLI 命令
skill 学习规划师 --action smart-plan
执行流程
1. 检测 plans/ 目录是否存在百日计划
2. 无百日计划 → 调用「制定百日计划」流程
3. 无当月计划 → 调用「制定月计划」流程
4. 有当月计划 → 调用「优化月计划」流程
5. 无本周计划 → 调用「制定周计划」流程
6. 有本周计划 → 调用「优化周计划」流程
7. 执行「制定今日计划」
副作用：可能生成/优化多个层级的计划文件，同时写入主存储和备份

制定百日计划
生成长期大计划，含JD分析+用户采访
制定百日计划
CLI 命令
skill 学习规划师 --action create-hundred-day-plan
执行流程
1. 前置：调用「采集用户信息」（不采访不生成）
2. 读取 jd/ 目录下所有 md 文件，统计技能关键词词频
3. 从「学习系统」获取当前学习进度和 Level
4. 结合以上数据生成百日计划内容
5. 写入 plans/{YYYYMMDD}_百日计划.md
6. 备份至 个人/{YYYYMMDD}_百日计划.md
写入文件：plans/ 百日计划 + 个人/ 备份。依赖：jd/*.md、学习系统进度、用户采访数据

制定月计划
CLI 命令
skill 学习规划师 --action create-monthly-plan
执行流程

制定周计划
从月计划分解+学习系统完成度，生成本周计划
制定周计划
CLI 命令
skill 学习规划师 --action create-weekly-plan
执行流程
1. 前置检查：确认 plans/ 下存在当月计划
2. 读取当月计划本周任务
3. 从「学习系统」获取上周完成度
4. 月计划分解 + 完成度 → 生成本周计划
5. 写入 plans/{YYYYWW}_周计划.md
6. 备份至 个人/{周一日期}/{YYYYWW}_周计划.md
前置条件：月计划存在。依赖：学习系统上周完成度。文件名使用 ISO 周数

制定今日计划
综合作息管家+学习系统+周计划，生成今日安排
制定今日计划
CLI 命令
skill 学习规划师 --action create-daily-plan
执行流程
1. 从「作息管家」获取过去14天每日摘要
2. 从「作息管家」获取过去7天作息记录明细
3. 从「学习系统」获取昨日完成度
4. 从「学习系统」获取整体进度
5. 判断周几/节假日，调整任务量
6. 生成作息时间表（含修行20min+锻炼20min）
7. 通过「作息管家」设置计划保存
8. 生成 HTML 文件
9. 通过 QQ 发送给用户
10. 写入 plans/{YYYYMMDD}_今日计划.md + 备份
副作用：写入计划文件、调用作息管家设置计划、生成HTML、发送QQ消息。必须符合今日计划格式模板

计划优化
🔧
优化月计划
对照百日计划+作息管家数据，优化当月计划
优化月计划
CLI 命令
skill 学习规划师 --action optimize-monthly-plan
执行流程
1. 读取百日计划当前阶段目标，对照当月计划完成度
2. 从「作息管家」获取过去一个月每日摘要（按日统计各分类时长）
3. 从「学习系统」获取月进度
4. 判断：当月目标是否合理？进度超前/滞后多少？
5. 滞后 > 20% → 调整下月目标，或顺延未完成任务
6. 超前 → 评估是否加速或补充额外目标
7. 同步更新百日计划的阶段进度
8. 写入主存储 + 备份
副作用：修改月计划文件、可能修改百日计划进度、写入备份。输出调整说明
🔧
优化周计划
对照月计划+作息管家数据，优化本周计划
优化周计划
CLI 命令
skill 学习规划师 --action optimize-weekly-plan
执行流程
1. 读取当月计划本周任务，对照完成度
2. 从「作息管家」获取本周每日摘要（今天之前实际投入时长）
3. 从「学习系统」获取本周进度（昨天完成了什么）
4. 判断：本周目标是否还能达成？
5. 滞后 → 调整剩余任务，优先核心内容（可删减次要任务）
6. 超前 → 评估补充额外内容或留出复习时间
7. 同步更新当月计划的周进度
8. 写入主存储 + 备份
副作用：修改周计划文件、可能修改月计划周进度、写入备份

3. 保存截图至 jd/images/{YYYYMMDD}_{公司简称}.jpg
4. 生成 md 记录关键信息
5. 保存到 jd/{YYYYMMDD}_{公司简称}.md
写入文件：jd/images/ 截图 + jd/ md记录。输入：用户截图

用户信息
🎤
采集用户信息
采访用户真实情况（时间/项目/短板/经验）
采集用户信息
CLI 命令
skill 学习规划师 --action interview-user
执行流程
1. 通过对话采访用户，收集核心信息：
· 每天真实可用的学习时间（工作日/周末）
· 最近的真实项目经历（技术细节/用户规模/个人贡献）
· 最大的技术短板
· 是否有面试经验
2. 对话记录可存档保存到 个人/ 目录
原则：不采访就不生成计划。可能写入个人/ 目录存档

自动触发

cron 每日触发
每日 08:00 自动检测并执行计划层级补全+优化
自动触发（无需唤醒词）
CLI 命令
0 8 * * * skill 学习规划师
执行流程
1. 检测 cron 任务，不存在则创建
2. 无百日计划 → 生成百日计划
3. 无当月计划 → 生成当月计划
4. 有当月计划 → 优化当月计划
5. 无本周计划 → 生成本周计划
6. 有本周计划 → 优化本周计划
7. 生成今日计划
副作用：可能生成/优化多个层级的计划文件、调用作息管家、发送QQ消息
