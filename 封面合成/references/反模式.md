## #1 ❌ `Image.rotate(expand=True)` on RGB

**症状**:旋转后图片的 4 个角是**纯黑色实色**方块,看起来很丑

**根因**:
```python
im = Image.open(path).convert("RGB")  # RGB 模式
im.rotate(45, expand=True)             # expand 后新角用 fillcolor=0=(0,0,0) 黑填
```

**正解**:**先转 RGBA,再 rotate**
```python
im = im.convert("RGBA")  # alpha 通道
im = im.rotate(45, expand=True)  # RGBA 模式 fillcolor 默认 (0,0,0,0) 透明
```

## #2 ❌ alpha 通道 GaussianBlur 羽化

**症状**:旋转图边缘有几像素是"半透明",跟画布叠加后看起来是**深色边缘**

**根因**:
```python
r, g, b, a = im.split()
a = a.filter(ImageFilter.GaussianBlur(6))  # 模糊让 a 从 255 渐变到 0
im = Image.merge("RGBA", (r, g, b, a))
```

渐变的 alpha 跟画布背景混合 → 半透明区 = "画布色 + 图片色"

**正解**:**alpha 二值化**(0/255 二值,不要渐变)
```python
a_arr = np.array(a)
a_bin = (a_arr > 128).astype(np.uint8) * 255
im = Image.merge("RGBA", (r, g, b, Image.fromarray(a_bin, 'L')))
```

## #3 ❌ shadow 扩散造成半透明黑

**症状**:每个图层下面带阴影,阴影在主图上叠加成"半透明黑色"

**根因**:
```python
shadow = Image.new("RGBA", img.size, (0, 0, 0, 180))  # alpha 180 半透明黑
shadow = shadow.filter(ImageFilter.GaussianBlur(15))       # 模糊扩散
canvas.paste(shadow, (x+12, y+12), shadow)               # 偏移贴
```

shadow 区域覆盖在主图上 → 主图被"半透明黑"叠加 → 看起来像有黑雾

**正解**:**完全去掉 shadow**。如果非要 shadow,要做 all-around(四周一致),且 alpha 限制在 80 以下

## #4 ❌ RGB 画布转 RGBA,alpha 默认 255

**症状**:"我创建了透明画布,为什么粘贴后是黑色实色?"

**根因**:
```python
canvas = Image.new("RGB", (W, H), (0, 0, 0))  # RGB
canvas = canvas.convert("RGBA")                # 转 RGBA,alpha 默认 255(不透明)
```

转 RGBA 时,RGB 像素的 alpha 默认 255。画布**不透明**,看不到底下的东西。

**正解**:**直接用 RGBA 创建,fill 用 (R,G,B,0) 透明**
```python
canvas = Image.new("RGBA", (W, H), (0, 0, 0, 0))  # alpha 0 = 透明
```

## #5 ❌ JPG 保存 RGBA 丢 alpha

**症状**:合成图看着有半透明,导出 JPG 后全是黑色实色

**根因**:JPG 不支持 alpha 通道,保存时 alpha 直接丢

**正解**:
- 透明需求 → 用 `.png` 输出
- 不用透明 → 显式 `convert("RGB")` 后再 JPG

```python
if output_path.endswith(".jpg"):
    canvas = canvas.convert("RGB")
canvas.save(output_path)
```

## #6 ❌ 旋转后角是黑色实色而非透明

**症状**:#1 的延伸,RGBA 模式下旋转角居然也是黑

**根因**:`rotate(expand=True)` 对 RGBA 默认 fillcolor=(0,0,0,0),但旧版 PIL 或某些 mode 下不生效

**正解**:**用临时大画布 + crop bbox,确保 alpha=0 是真的透明**

```python
diag = int(sqrt(w**2 + h**2)) + 20
temp = Image.new("RGBA", (diag, diag), (0, 0, 0, 0))
temp.paste(rgba, ((diag-w)//2, (diag-h)//2), rgba)
rotated = temp.rotate(angle, resample=Image.BICUBIC, expand=False)
cropped = rotated.crop(rotated.getbbox())  # 裁掉 alpha=0 区域
```

## #7 ❌ PIL rotate 抗锯齿产生 alpha 中间值

**症状**:旋转后 alpha 通道不是 0/255 二值,有 100/200 等中间值

**根因**:`Image.BICUBIC` 抗锯齿在像素边界做插值,产生中间值

**正解**:**二值化强制转 0/255**(见 #2)

## #9 ❌ 不显式 z-order

**症状**:改了图层顺序,某图层突然被遮,不知道为什么

**根因**:靠 paste 顺序隐式决定 z-order,后人看不懂

**正解**:**LayerSpec.z 显式声明**
```python
@dataclass
class LayerSpec:
    path: str
    z: int  # 显式 z-order,数字越大越上层
```

compose 时 `sorted(layers, key=lambda l: l.z)` 后逐个 paste

## #10 ❌ 文字无黑色描边

**症状**:黄字 + 亮色照片背景上,字读不清

**根因**:`draw.text((x, y), text, fill=黄, font=font)` 没指定描边

**正解**:**黑色描边 + 描边宽度 ≈ 字号 × 5%**
```python
draw.text(
    (x, y), text, font=font, fill=黄,
    stroke_width=8,          # 200 字号下 8 像素
    stroke_fill=(0, 0, 0),   # 黑色
)
```

## #11 ❌ 重复缩小主图,主图消失

**症状**:"我想让左右图不挡主图",把主图越缩越小,最后主图太小看不见

**根因**:**主图大小应该固定,左右图缩小后入侵主图** 才正确

**正解**:
```python
main_size = 0.65 * min(W, H)   # 固定
side_size = 0.50 * min(W, H)   # 左右图缩小
```

## #12 ❌ 4:3 安全区在 9:16/16:9 画布上不一致

**症状**:"我在 16:9 画布上画了文字,怎么裁成 4:3 视频封面后字跑偏了"

**根因**:不同画布比例的"安全区"算法不一样,得分别处理

**正解**:**每个 layout spec 显式声明 safe_area**
```python
SAFE_AREA_4_3_IN_16_9 = {
    "x_min": 240, "x_max": 1680,  # 16:9 画布 1920x1080 居中
    "y_min": 0, "y_max": 1080,
}
SAFE_AREA_9_16 = {
    "x_min": 0, "x_max": 1080,
    "y_min": 200, "y_max": 1680,  # 避开顶部标题栏 + 底部评论栏
}
```

## #13 ❌ RGB 转 RGBA 后 fill 非黑

**症状**:#4 的另一种情况,fill 用 RGB tuple 转 RGBA 时没补 alpha

**根因**:
```python
canvas = Image.new("RGB", (W, H), (255, 0, 0))  # 红色
canvas = canvas.convert("RGBA")  # (255, 0, 0, 255) 不透明红色
# 用户以为"红色画布"变"半透明红色"
```

**正解**:**直接用 RGBA 创建,fill 写 4-tuple**
```python
canvas = Image.new("RGBA", (W, H), (255, 0, 0, 255))  # 显式不透明
# 透明 = (0, 0, 0, 0)
```

## #14 ❌ 字号过大撑爆 4:3 安全区

**症状**:200 字号写"挑战 44 斤"在 720 宽 4:3 区域里,字溢出

**根因**:字号按"画布宽"算的,没考虑"安全区宽"

**正解**:`fit_text_to_area()` 自动缩字号到 fit
```python
size = start_size
while size > 20:
    bbox = font.getbbox(content)
    if bbox[2] - bbox[0] <= max_width and bbox[3] - bbox[1] <= max_height:
        return size
    size -= 5
return 20
```

## #15 ❌ JPG 不支持 alpha → 透明失效

**症状**:#5 的"症状描述",导出的 JPG 没有透明效果

**根因**:JPG 是有损格式,不支持 alpha 通道

**正解**:
- 透明需求 → `.png`
- 不要透明 → 显式 `convert("RGB")` 后 JPG

```python
def safe_save(canvas, output_path):
    if output_path.endswith(".jpg"):
        if canvas.mode == "RGBA":
            canvas = canvas.convert("RGB")
    canvas.save(output_path)
```

## #16 ❌ 旋转/合成后没 diagnose 半透明黑

**症状**:用户反馈"封面有半透明黑",开发者 debug 一小时

**根因**:没自动检查,等用户发现

**正解**:`compose()` 末尾自动跑 `diagnose()`,返回 warnings
```python
diag = full_diagnose(canvas)
if diag["status"] == "warn":
    result["status"] = "warn"
    result["warnings"] = diag["issues"]
```

## 总结:合成多层图片的 3 条铁律

1. **alpha 必须是 0 或 255**(不要中间值)
2. **画布必须从一开始就是 RGBA**(不要从 RGB 转)
3. **永远显式 z-order**(不要靠 paste 顺序)

满足这 3 条,半透明黑问题消失 90%。剩下的 10% 来自 shadow / 抗锯齿,需要 diagnose() 兜底。
