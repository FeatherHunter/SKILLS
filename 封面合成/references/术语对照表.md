# 封面合成 · 术语对照表

> 专业词 ↔ 大白话。配合《优秀Skill指导手册》§⑦ 体系使用。

| 专业词 | 通俗替代词 | 生活类比 | 上下文 |
|---|---|---|---|
| **alpha 通道** | 透明度通道 / "看不见的图层" | 玻璃纸——玻璃越透明,后面东西越清楚 | RGBA 模式的第 4 个通道 |
| **alpha = 255** | 完全不透明 / 实心 | 厚纸板——完全挡住背景 | 像素完全可见 |
| **alpha = 0** | 完全透明 / 隐形 | 完全透明玻璃——后面东西原样 | 像素不可见 |
| **alpha = 128** | 半透明 / 半遮 | 磨砂玻璃——模糊看到背景 | **避免!** 跟画布混合会出色差 |
| **alpha 二值化** | 透明/不透明二选一 / 把半透明砍掉 | 把所有玻璃换成"要么完全透要么完全挡" | 旋转图必备,防半透明黑 |
| **alpha 羽化** | 边缘渐变透明 / 像渐变光晕 | 阴影柔化——但跟其他图层叠加会出色差 | **避免!** 用二值化替代 |
| **RGBA** | "带透明度的彩色图" / "4 通道图" | RGB = 位置 + 颜色;RGBA = 位置 + 颜色 + 透明度 | PIL 默认模式 |
| **z-order** | 图层上下顺序 / 哪个盖哪个 | 桌面图标——你点哪个哪个在最上面 | LayerSpec.z 字段 |
| **paste with mask** | "用图本身当遮罩" | 把图当橡皮章——只盖图本身有的部分 | `canvas.paste(img, (x,y), img)` |
| **expand=True** | 旋转后画布自动长大 | 转 45° 后画布不再是矩形,要装下对角线 | `im.rotate(angle, expand=True)` |
| **fillcolor** | 旋转后新角的填充色 | 画布不够大时,边缘用什么色补 | RGBA 模式默认 (0,0,0,0) 透明 |
| **bbox** | 内容边界框 / 图里"实际有像素"的区域 | 一张照片的"画了东西的部分" | `im.getbbox()` |
| **safe area** | 安全区 / 平台裁切不切到的地方 | 重要内容放中间,边缘随便裁 | 16:9 画布中央 4:3 |
| **aspect ratio** | 长宽比 / 9:16 / 16:9 | 手机屏 9:16,电视 16:9,老电视 4:3 | 输出尺寸 |
| **canvas** | 画布 / 所有图层叠在一起的底 | 画画前先铺一张纸 | Image.new("RGBA", (W,H), bg) |
| **raster image** | 位图 / 像素图 | 拼图——每块是固定色 | JPG / PNG |
| **vector image** | 矢量图 / 数学描述 | 方程式画的——无限放大不糊 | SVG / PDF |
| **anti-aliasing** | 抗锯齿 / 边缘平滑 | 斜线不"锯齿",混色模拟 | PIL rotate 默认开启 |
| **getbbox()** | 找图里"实际有像素"的最小矩形 | 在图上画个最紧的方框 | 用于裁切旋转后的空白 |

## 反模式速查(对应 references/反模式.md)

| 反模式 | 通俗替代词 | 一句话根因 |
|---|---|---|
| RGB rotate 黑色填充 | 玻璃没装好就转,玻璃角发黑 | rotate 前必须转 RGBA |
| alpha 羽化 | 边缘透明渐变,跟背景混出色差 | alpha 必须 0/255 二值 |
| shadow 扩散 | 图下面带"光晕",容易污染主图 | 干脆别加 shadow |
| RGB 转 RGBA | 转出来 alpha=255,跟你想的不一样 | 直接用 RGBA 创建,fill 4-tuple |
| JPG 不支持 alpha | JPG 是有损压缩,没"透明"维度 | 透明必须用 PNG |

## 业务术语

| 词 | 意思 |
|---|---|
| 主图(hero) | 封面上最大、最显眼的图,通常放中央 |
| 副图 / 配图 | 主图周围的小图,烘托氛围 |
| 层叠交错 | 多图叠加摆放,像散落的照片 |
| 镜像对称 | 左右图位置关于画布中心对称 |
| 钩子(hook) | 封面上吸引人的元素,通常是文字或数字 |
| 缩略图(thumbnail) | 视频/文章的封面图 |
| 钩子标题 | 封面最显眼的文字,用于吸引点击 |
