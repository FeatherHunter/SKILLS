封面合成 (cover-composer)

v1.0 · 2026-07-15 · SKILL.md · changelog · 反模式

多图旋转叠加 + 文字水印 + 半透明黑防御

管什么

  • 多张照片(+ 文字水印)在 16:9 / 9:16 / 4:3 / 1:1 画布上的层叠合成
  • 多图旋转 / 透明叠加 / 对称布局 / 文字水印
  • 半透明黑检测 / 对称性检测 / 暗像素区检测
  • 平台预设模板(抖音 / B 站 / 视频号 / 小红书 / YouTube / 微博)

不管什么

  • 单图磨皮 / 调色 / 滤镜(用其它 skill)
  • 视频帧提取 / 视频 thumbnail(用 ffmpeg)
  • GIF / 动图 / AI 生图(用其它 skill)
  • 多文字复杂海报(用 design-tool)

触发词

封面合成 多图拼图 做封面 缩略图 拼贴 海报合成 cover compose photo collage thumbnail

快速上手

# 1. 查平台预设
封面合成 presets --platform douyin
# → {"ratio": "9:16", "size": [1080, 1920], "name": "抖音"}

# 2. 合成(DAY14 真实用法)
封面合成 compose \
  --photos main.jpg left.jpg right.jpg \
  --layout symmetric-cascade \
  --aspect 16:9 \
  --text '{"main":"14 天","sub":"-7 斤","tags":"腰突 大基数"}' \
  -o cover.jpg

# 3. 自检
封面合成 diagnose cover.jpg

核心 API

双层 API 设计(关键)

                    ┌─────────────────┐
                    │  ② 智能 CLI       │  ← operations/auto_compose.py
                    │  (智能决策)        │     (分析图 → 决策 → 调 ①)
                    └────────┬────────┘
                             │
                             ▼
                    ┌─────────────────┐
                    │  ① 基础 API       │  ← core/pipeline.py:compose()
                    │  (参数化)          │     (接收完整参数,直接合成)
                    └─────────────────┘

手动挡 compose(用户传完整参数)→ ① 直接执行
自动挡 auto(用户只传 --photos)→ ② 分析决策 → 调 ①
两者都通过 core/pipeline.py:compose() 这个唯一基础 API落地

4 种 layout

layout用途z 分布
symmetric-cascade(默认)主图 + 左右副图,镜像对称主=0, 副=1
cascade主图 + 副图堆叠(右下偏移)主=0, 副=1+
polaroid主图中央,副图四角散落主=1, 副=2+
grid网格平铺(多图无主图)全 =0

8 条核心反模式(详见 反模式.md)

❌ #1 RGB rotate 角是黑实色

必须先转 RGBA 再 rotate,否则 expand 后的新角用 0 填黑

❌ #2 alpha 羽化导致半透明黑

GaussianBlur 羽化让 alpha 有中间值,跟画布混出色差。二值化(>128 → 255)替代

❌ #3 shadow 扩散污染主图

每个图层加 shadow 会叠加成"半透明黑雾",直接去掉 shadow

❌ #4 RGB 转 RGBA alpha 默认 255

转出来不透明。直接用 Image.new("RGBA", (W,H), (0,0,0,0)) 创建透明画布

❌ #5 JPG 保存丢 alpha

JPG 不支持 alpha。透明需求用 PNG,不需要透明用 .convert("RGB") 显式转

❌ #7 PIL rotate 抗锯齿 alpha 中间值

Image.BICUBIC 抗锯齿在像素边界做插值,产生中间值 alpha。必须二值化

❌ #9 隐式 z-order

LayerSpec.z 显式声明,不要靠 paste 顺序

❌ #10 文字无描边在亮色照片上读不清

stroke_fill=(0,0,0) + stroke_width=字号×5% 黑色描边

3 条铁律(踩了 16 个坑后总结)

  1. alpha 必须是 0 或 255(不要中间值)
  2. 画布必须从一开始就是 RGBA(不要从 RGB 转)
  3. 永远显式 z-order(不要靠 paste 顺序)

满足这 3 条,半透明黑问题消失 90%。剩下 10% 来自 shadow / 抗锯齿,需要 diagnose() 兜底。
