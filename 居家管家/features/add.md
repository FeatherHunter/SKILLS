# 物品录入

## 流程概述

1. 解析用户描述，提取基本信息
2. 位置智能推荐（同类物品位置）
3. 补全缺失字段
4. 检查重复物品
5. 确认并写入数据库
6. 提示录入图片（可选）

---

## Step 1: 解析用户描述

AI 从用户自然语言中提取：
- 名称
- 分类
- 位置
- 数量
- 标签
- 备注
- 购买日期（可选）
- 过期日期（可选）

**位置推荐**：根据物品分类查询 `item_locations` 表，找出同类物品用过的位置（按使用次数排名），展示给用户选择。用户选择已有位置或输入新位置。

对照 `references/categories.md` 推断分类，不确定则询问用户。

### 状态确认

对照 `references/statuses.md` 确认状态（默认"在家"）。

**AI自动推断规则（最小实现）**：
- 位置路径包含"快递" → 自动设置状态为"快递中"
- 其他情况默认"在家"

AI 仍需询问用户确认最终状态，但可以提示推断依据。

### 图片解析

AI 从图片中提取以下信息作为 tags：
- 颜色（白色、黑色、蓝色、红色等）
- 品牌/厂商（海天、可口可乐、优衣库等）
- 规格容量（500ml、2L、大瓶、小瓶等）
- 材质/面料（纯棉、羊毛、丝绸、塑料等）
- 口味/香型（橙味、原味、薄荷等）
- 适用场景（运动、职场、户外、居家等）
- 季节（春夏、秋冬、四季）
- 新旧程度（全新、九成新、八成新等）

**原则**：能识别多少就提取多少，不要遗漏图片中明显可见的信息。

## Step 2.5: 位置智能推荐

查询 `item_locations` 表，根据物品分类推荐位置：

```python
from home_manager.location_ops import suggest_locations

locations = suggest_locations(conn, category)
```

展示给用户：

```
📍 同类物品位置推荐：
  1. 卧室/东南角/小冰箱上/电子设备抽屉 (2件同类)
  2. 卧室/东南角/小冰箱上/眼镜装扮抽屉 (3件同类)
  3. 客厅/东北角/桌子上 (1件同类)
  4. 其他位置（用户输入）
```

用户选择编号或输入新位置。

## Step 3: 补全缺失字段

AI 一句话问完所有缺失信息：

```
收到，先记个草稿 📝

{name} ｜ {category}

再补充几点就好：
· 它放在哪儿？（**必须至少两级路径，如`客厅/冰箱`、`厨房/灶台`**，不允许只写"客厅"）
· 谁的？（不说就默认「使用者」）
· 数量有几件/包？（不说就默认 1）
· 花了多少钱？（没记就跳过）
· 什么时候买的？（大概日期就行，跳过也可以）
· 会过期吗？（食品/化妆品类才需要，写个大概日期）
· 贴几个标签方便以后找？（**必填**，至少包含：颜色 + 品牌，其他可选：规格、口味、材质、季节、适用场景）
· 有没有图片要发过来？（有就发，没有说"无"就行）

一口气告诉我就行 👇
```

**原则**：缺什么问什么，不问已经给过的

**必需字段**：name / category / location 必须全部确认

**location 路径规则**：必须至少包含两级（如 `客厅/冰箱`，`厨房/灶台`），不允许单级位置。

---

## Step 4: 检查重复

AI 执行一次搜索，检查是否有重名或名称相似的物品。

如果找到以下情况，AI 必须列出让用户选择：
- 已有"已用完"状态的同名物品 → 询问"补充库存"还是"新建"
- 已有"已废弃"状态的同名物品 → 询问"恢复使用（补充）"还是"新建"
- 存在其他同名物品 → 询问"更新现有"还是"新建"

用户选择"补充"后，AI 询问新物品的状态："现在是在家、备用、还是快递中？"

**用户确认前不得写入数据库**

---

## Step 5: 确认并写入

AI 展示完整记录，用户说"对"后执行：

```bash
python home_manager.py add \
  --name "薯片" \
  --category "食品" \
  --location "厨房/上柜/左侧" \
  --quantity 3 \
  --tags "零食,乐事,原味"
```

**注意**：当前脚本不支持多位置同时录入。如需多位置，请分多次录入。

## Step 7: 提示录入图片

物品录入成功后，AI 必须询问用户是否需要添加图片：

```
物品已录入成功（ID: {物品ID}）。
需要为此物品添加图片吗？发给我图片即可，或回复"不用"跳过。
```

### 图片录入子流程

1. 先添加物品（不带 photo）→ 得到 ID
2. AI 使用视觉模型分析图片，提取 tags
3. 将提取的 tags 展示给用户确认
4. 用户确认 tags 后，保存图片到 `photos/{物品ID}_{名称}.jpg`
5. 执行 `update --id {ID} --photo "photos/{ID}_{名称}.jpg" --tags "用户确认的tags"`
