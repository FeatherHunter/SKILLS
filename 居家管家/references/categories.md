# 分类总览

> **权威源**:`categories` 表(SQLite)。本文档为人类阅读参考,**不**作为代码查询依据。
>
> **代码查询**:
> - `python category_manager.py tree` — 树形展示
> - `python category_manager.py list` — 列表
> - `python category_manager.py show --id N` — 详情
>
> **新增分类**:必须走 CLI(`category_manager.py import`)。**AI 不直接写库**。

---

## 设计原则(第一性)

1. **管理动作 > 物品本体** —— 按"怎么管它"组织,不是"是什么"
2. **三层为主,按需四级** —— 当前固定 3 层
3. **叶子 ≥ 3 件才独立** —— < 3 件合到上级或同级
4. **互斥原则** —— 一件物品只能归一个分类(按主用途)
5. **正交不耦合** —— 分类管"是什么";tag 由全局 tag 体系管(详见 `features/add.md` Step 3.5)

## 关键决策记录

- **2026-07-03**:老 28 大类设计废止;新建 `categories` 表为权威源
- **aliases / 必填 tag / 推荐 tag 全部删除** —— 分类只管"是什么",tag 由全局体系管
- **试点域 = 1. 食物与饮品**(老数据多+边界清晰)
- **`--category-id` 必选 int**,从 categories 表 derive 字符串写入老 category 字段
- **`category_manager.py import --merge`**:合并模式(同名跳过,异名新增),避免 `--force` 重排 id 致引用失效

## 与老数据的关系

- 老 `items.category` 字段保留(过渡期)
- 新物品走新分类(`category_id`)
- 老物品的迁移在 **Phase B:数据迁移** 时执行(逐件 review,从老 category 字符串映射到新 category_id)
- **过渡期**:老物品显示老 category 字符串,新物品显示新分类
