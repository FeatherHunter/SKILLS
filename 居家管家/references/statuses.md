# 物品状态参考

## 10种状态

| 状态 | 含义 | 常见触发场景 |
|------|------|-------------|
| 在家 | 在固定位置 | 默认状态、归位后 |
| 备用 | 备用库存 | 囤货、未拆封的替换品 |
| 穿着中 | 正在身上使用 | 衣服穿在身上 |
| 旅游中 | 带出去旅游了 | 出行前确认 |
| 洗护中 | 在洗衣机/晾衣架/烘干机 | 脏衣服送洗 |
| 借用中 | 借给朋友了 | 外借确认 |
| 维修中 | 坏了在修 | 送修确认 |
| 已用完 | 消耗品用完了 | quantity自动归0 |
| 快递中 | 已下单/发货了，在配送中 | 网购等待收货 |
| 待处理 | 需要清理或处理 | 待整理物品 |
| 已废弃 | 废弃了 | 用户明确表态 |

## 状态流转规则（衣物示例）

```
在家（衣柜）→ 穿着中/旅游中（出门）→ 洗护中（脏衣服）→ 在家（洗干净放回）
```

## 自动状态变更

| 条件 | 动作 |
|------|------|
| quantity 被更新为 0 | 自动将 status 设为 "已用完" |
| quantity > 0 但用户说废弃 | status 设为 "已废弃"，quantity 保持原值 |

## AI 推断规则

- 用户说"穿出去" → 建议"穿着中"
- 用户说"带出去旅游" → 建议"旅游中"
- 用户说"借给XX" → 建议"借用中"
- 用户说"坏了" → 建议"维修中"
- 用户说"洗了/在晾" → 建议"洗护中"
- 用户说"用完了" → 建议"已用完"
- 用户说"扔了/不要了" → 建议"已废弃"
- 用户说"下单了/买了/发货了" → 建议"快递中"
- 用户说"收到了/取件了" → 建议"在家"
- 用户说"放回去了/整理好了" → 建议"在家"
- 无法推断 → 让用户选择
