## 一、背景与目标

### 1.1 问题

- `home_manager.py` 当前 944 行（约19页），单文件过胖
- AI 每次读取要翻大量上下文，不利于分析和修改
- `is_primary` 主次位置设计已被否定，需要彻底改造
- `item_locations` 需要新增 `location_status` 字段

### 1.2 目标

1. 将脚本按职责拆分为多文件模块
2. **不改变任何外部可见行为**（CLI 参数、输出格式、调用方式完全兼容）
3. 为后续 `location_status` 加字段铺路

## 二、拆分方案（简化版）

### 2.1 目录结构

```
居家管家/scripts/
├── home_manager/           # 包目录
│   ├── __init__.py         # 空（或简单 re-export 主函数）
│   ├── models.py           # dataclass 数据模型（Item / ItemLocation / Tag）
│   ├── db.py              # 数据库连接 + 建表 + 迁移
│   ├── item_ops.py        # add / search / update / detail / list
│   ├── location_ops.py    # 位置读写（含 location_status 推导）
│   ├── tag_ops.py         # 标签相关操作
│   ├── inventory_ops.py   # 盘点、统计
│   └── home_manager.py   # CLI入口（argparse + 命令路由，保留原样不动）
├── home_manager.py        # thin wrapper入口（从包 import cli）
└── home_manager_backup.py # 原文件备份
```

> **注**：`home_manager/` 目录内的 `home_manager.py` 是包内文件，与顶层 `home_manager.py` 是两个文件。

## 五、后续：location_status 加字段（第二步）

> 本节内容仅供参考，第二步实施时可能调整

### 5.1 目标

- `item_locations.location_status`：`TEXT DEFAULT '在家'`
- 去掉 `is_primary` 字段
- `items.status` 废除，运行时从 location_status 推导

### 5.2 迁移（幂等）

```python
def migrate_location_status():
    conn = get_conn()
    cursor.execute("PRAGMA table_info(item_locations)")
    columns = [row[1] for row in cursor.fetchall()]
    if 'location_status' not in columns:
        cursor.execute("ALTER TABLE item_locations ADD COLUMN location_status TEXT DEFAULT '在家'")
    conn.commit()
```

### 5.3 is_primary 迁移

```python
def migrate_remove_is_primary():
    conn = get_conn()
    # 将所有 is_primary=1 的位置 location_status 设为 '在家'
    # is_primary=0 的位置 location_status 设为 '备用'
    cursor.execute("""
        UPDATE item_locations
        SET location_status = CASE is_primary WHEN 1 THEN '在家' ELSE '备用' END
        WHERE location_status IS NULL
    """)
    # 删除 is_primary 列
    cursor.execute("ALTER TABLE item_locations DROP COLUMN is_primary")
    conn.commit()
```

### 5.4 items.status 推导策略（方案A）

运行时推导，不存冗余：

| 场景 | 实现 |
|------|------|
| `detail` 展示 | `"卧室×3（在家）| 客厅×2（备用）"` |
| `search --status "在家"` | `WHERE location_status='在家' AND quantity>0` |
| `list` 展示 | 合并所有位置状态缩写 |
| `stats --type summary` | 从 item_locations 实时聚合 |
