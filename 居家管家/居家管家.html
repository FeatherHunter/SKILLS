3 分钟入门

本技能让你用自然语言管理家里所有物品。AI 负责理解、确认、执行；你只需要说清楚想做什么。

三类核心动作

① 查 —— "东西在哪？"
说查物品 黑色T恤 / 找冰箱里的牛奶
典型对话

👤 「找一下卧室里白色的T恤」

🤖 （搜索后弹出 HTML 页面，列出所有匹配物品）

👤 「我要看第二件的详情」

🤖 （打开详情页：标签、位置、购买日期、照片）

② 录 —— "家里新买了"
说录物品 一箱牛奶 放冰箱里 / 拍照发给 AI
典型流程

👤 「录物品 一箱牛奶 放在厨房冰箱」

🤖 「同类物品一般放在：① 厨房/冰箱 [12件] ② 厨房/储物柜 [3件]，放哪个？」

👤 「放厨房/冰箱」

🤖 「将写入：名称=牛奶 / 分类=食物与饮品 / 位置=厨房/冰箱 / 数量=1 / 状态=在家，对吗？」

👤 「对」

🤖 （写入成功，回读展示）

③ 算 —— "数据怎么看？"
说统物品 / 查过期 / 查低频
典型场景

👤 「查过期」

🤖 「未来 30 天内过期 5 件，已过期 2 件 —— [列表]」

💡 关键原则 所有 写操作（录、改、移、删）AI 都会先展示"将执行什么"，你说"对"才会真动手。放心试，错了不写库。

只展示你需要知道的变化。底层修改、bug 修复等不打扰你的事不会出现在这里。

① 查找类

找东西的 5 个唤醒词。涵盖按名称、位置、标签、状态、参考物品定位。

查物品
说查物品 [关键词] · 找 [关键词]

使用示例

👤 「查物品 黑色T恤」

🤖 （打开 HTML 搜索结果页，列出所有匹配物品）

👤 「找冰箱里的牛奶」

🤖 （按位置 + 名称搜索，结果按位置分组）

👤 「看看有哪些食品」

🤖 （按顶级分类"食物与饮品"搜索，包含所有子类）

可组合的搜索维度

  • 名称：模糊匹配，如"杯子"会匹配"水杯"、"马克杯"
  • 位置：模糊匹配路径片段，如"冰箱"会匹配所有含"冰箱"的位置
  • 标签：精确匹配，如"白色"
  • 分类：按顶级或二级 ID 筛选
  • 状态：在家 / 备用 / 快递中 / 旅游中 / 借用中 / 维修中 / 已用完 / 已废弃
底层工作流
Step 1 · 解析搜索条件

AI 从你的输入里识别出 5 个维度（名称/位置/标签/分类/状态）的取值。

Step 2 · 执行搜索
python home_manager.py search --name "T恤" --tag "黑色" --limit 20
Step 3 · 处理结果
  • 0 条：告诉你没找到，询问是否换个关键词或新建物品
  • 1 条：直接展示
  • 多条：列出来让你选（不默认选任何一个）
Step 4 · 输出

默认生成 templates/search_results.html 在浏览器打开。

推位置
说推位置 [分类]

使用示例

👤 「推位置 衣物」

🤖 「同类物品位置推荐（衣物）—— 共 3 个位置：① 卧室/衣柜/上层 [2件] 代表：黑色连帽卫衣、白色长袖T恤 ② 卧室/抽屉 [1件] 代表：深蓝牛仔裤 ③ 其他位置（请告诉我新位置）」

底层工作流

② 录入类

家里新买了什么？两种方式：文字描述 或 拍照。

录物品
说录物品 [描述]

使用示例

👤 「录物品 一箱特仑苏牛奶 保质期 6 个月 放厨房冰箱」

🤖 「将写入：名称=特仑苏牛奶 / 分类=食物与饮品/乳制品 / 位置=厨房/冰箱 / 数量=1 / 状态=在家 / 过期日期=2027-01-24，对吗？」

👤 「对」

完整流程（AI 内部走的 6 步）

  1. 解析描述：提取名称、数量、位置、标签、备注、购买日期、过期日期
  2. 位置推荐（必走）：自动调"推位置"，展示同类物品所在位置
  3. 补全字段：缺失的字段问你确认
  4. 查相似（必走）：先搜索库内是否有同名/相似物品，避免重复
  5. 确认并写入：展示预览，你说"对"才写库
  6. 回读确认：写入后再读一遍给你看
⚠️ 防重复机制 搜索到名称相同的"已用完"或"已废弃"物品时，AI 必须问你是"补充到现有记录"还是"新建记录"。不会自动决定。
底层工作流
前置校验
  • 分类必须先从 categories 表查，不硬编码
  • 位置路径自动建议 ≥ 2 级（如 厨房/冰箱）
  • 状态默认"在家"，路径含"快递"时自动改"快递中"
CLI 调用
python home_manager.py add \
  --name "特仑苏牛奶" \
  --category-id 145 \
  --location "厨房/冰箱" \
  --quantity 1 \
  --expiration-date "2027-01-24" \
  --tags "乳制品,白色,1L"
拍物品
说拍物品 + 发送照片

使用示例

👤 「拍物品」（发送一张洗衣液的照片）

🤖 「图中识别：蓝色塑料瓶、约 2L、品牌'蓝月亮'、瓶身文字'亮彩增艳洗衣液'。建议分类=家居与陈设/清洁用品。位置？」

AI 会从图片里自动提取的标签

  • 颜色：白/黑/蓝/红/...
  • 品牌：海天、可口可乐、优衣库...
  • 规格：500ml / 2L / 大瓶 / 小瓶
  • 材质：纯棉 / 羊毛 / 塑料
  • 口味/香型：橙味 / 原味 / 薄荷
  • 场景：运动 / 职场 / 户外 / 居家
  • 季节：春夏 / 秋冬 / 四季
  • 新旧：全新 / 九成新
底层工作流

图片理解 → 提取结构化信息 → 走"录物品"流程的 Step 1-6。add --photo 内部拿到 ID 后把图片复制为 YYYYMMDD_{ID}_{描述}.jpg 并写入 item.photo。

图片保存位置：环境变量 HOME_PHOTOS_DIR，默认为技能目录下 photos/。

③ 更新类

8 个唤醒词，覆盖物品的常见变更场景。"改物品"是通用入口，其他是常用快捷方式。

改物品
说改物品 [名称] [变更]

使用示例

👤 「改物品 黑色马克杯 把备注改成'老婆送的'」

🤖 「将更新：remark='老婆送的'，对吗？」

支持修改的字段

  • 名称 / 分类 / 备注 / 所有者

  • 数量 / 位置 / 状态
  • 购买日期 / 过期日期
底层工作流

搜索 → 推断变更内容 → 展示预览 → 确认 → update --id {ID} --field value。

减物品
说减物品 [名称] [数量] · [名称] 用了/吃了/喝了 N 个

标物品
说标物品 [名称] [新标签]

使用示例

👤 「标物品 黑色T恤 加标签'秋冬'」

🤖 「追加标签'秋冬'，当前标签：黑色、长袖、全棉、秋冬，对吗？」

④ 盘点类

核对指定位置 / 全屋的物品清单。出差回来、季度整理时用。

盘物品
说盘物品 [位置] · 盘点 [位置]

使用示例

👤 「盘物品 卧室衣柜」

🤖 「卧室/衣柜 共 23 件：① 黑色连帽卫衣 ② 白色长袖T恤 ... 请逐个确认状态是否正确」

盘点中可说的话

  • 「都对」→ 结束盘点
  • 「第 3 件数量改成 2」→ 自动进入"改物品"流程
  • 「第 5 件状态改成废弃」→ 同上
  • 中途退出 → 已确认的写入，未确认的保持原样
底层工作流

inventory --location "卧室/衣柜" --output inventory_check.html → 逐项确认 → 增量 update。

⑤ 穿搭

穿什么
说穿什么 · 今天穿啥

使用示例

👤 「今天穿什么？」

🤖 「今天 22°C，多云。建议：上装=白色长袖T恤（标签：四季、薄） / 下装=深蓝牛仔裤 / 鞋=白色运动鞋。需要标记某件为"穿着中"吗？」

AI 如何选

  • 第一步：查今日天气（天气获取失败则用纯标签筛选）
  • 第二步：列出所有"在家"的衣物 + 鞋类
  • 第三步：按温度匹配季节标签（春夏/秋冬/四季）、材质（薄/厚）、场景（运动/职场/居家）
底层工作流

list --category-id 138 --status "在家"（衣物顶级）+ list --category-id 148 --status "在家"（鞋类二级）→ 按 tags 筛选 → 推荐组合。

注意：AI 不会自动改状态，你决定穿哪件。

会检查什么

  • 标签完整性：缺标签、标签太少、重要属性遗漏
  • 表利用率：物品是否都有分类/位置/状态；位置路径是否重复（如 "卧室/东南角/..." vs "卧室东南角/..."）

常见问题

Q1：AI 能不能帮我自动录物品，不用每次都确认？

不能。所有写操作必须你点头。这是防误操作的核心机制。

Q2：录错了能删除吗？

不能物理删除物品，但可以改状态为"已废弃"。历史记录会保留，方便追溯。

Q3：同名物品怎么区分？

AI 会列出所有同名物品让你选，不默认选任何一个。你也可以用位置/标签/分类组合精确指定。

Q4：补充库存时遇到"已用完"的同名物品？

AI 会问你："补充到现有记录"还是"新建记录"。不会自动决定。

Q5：分类找不到想要的怎么办？

告诉 AI 想要什么分类，AI 会先查 categories 表确认；如不存在，会先建议创建（需你确认）。

Q6：照片存哪里？

环境变量 HOME_PHOTOS_DIR 配置，默认为技能目录下 photos/。文件名格式 YYYYMMDD_{物品ID}_{描述}.jpg。

Q7：怎么备份数据？

复制整个技能目录即可。核心数据在 .db/home.db（SQLite 单文件），照片在 photos/。

Q8：手机能用吗？

可以。用 SkillBoard 网页版，选 home.db 文件即可。
