# 录音机 - 推荐定时任务

> 本文档说明录音机技能搭配怎样的定时任务效果最好

---

## 核心原理

录音机通过增量扫描 session 文件入库，每次扫描基于 `scan_checkpoint` 记录的上次位置，只扫描新消息。

**频率越高，实时性越强；但频率太高会增加系统开销。**

---

## 推荐配置

### 🏆 最佳推荐：每10分钟

```
频率：every 10m
任务名：录音机-每10分记录语录
session：isolated（静默执行）
```

**理由**：
- 覆盖日常使用场景（QQ/微信/电脑端）
- 10分钟内的新消息会被及时记录
- 不会对系统造成明显负担
- 用户体验和资源消耗平衡

---

## 其他可选配置

### 选项1：每5分钟（高实时性）

```
频率：every 5m
适合：需要更高实时性的用户
缺点：系统开销增加约2倍
```

### 选项2：每小时（低开销）

```
频率：every 1h
适合：低频使用或节省资源
缺点：消息最多延迟1小时入库
```

### 选项3：每30分钟（折中）

```
频率：every 30m
适合：不想太频繁但也不想太久
缺点：折中方案，无明显优势
```

## 创建推荐任务

如果需要重新创建，推荐的命令：

```bash
openclaw cron add \
  --name "录音机-每10分记录语录" \
  --every 10m \
  --session isolated \
  --message "静默执行记录今日语录：python3 scripts/record.py"
```

---

## 注意事项

1. **session 必须为 isolated**：录音机扫描是静默后台任务，不需要回复用户
2. **不需要 delivery**：扫描结果不需要推送，纯后台运行
3. **不要设置 announce**：录音机不需要打扰用户，只有当用户主动查询时才需要看到结果
4. **增量扫描是自动的**：每次运行自动从上次的 checkpoint 继续，不需要手动管理
