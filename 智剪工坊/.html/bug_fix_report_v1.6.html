🐛 12 个 bug 修复详情

audioenhance.py（2 bug）

enhance_dialog (line 30)

af = f"dialoguenhance=level={level}" # ffmpeg 7.1 不认 level
af = f"dialoguenhance=enhance={level * 3}" # 新参数 enhance (0-3)

ffmpeg 7.1 把 dialoguenhance 的 level 改为 enhance（范围 0-3 而非 0-1）。 映射：用户传 0.5 → ffmpeg 收到 1.5（中等强度）。

⭐ 这是 voice_extract.py 100% 失败的根因，已修。

dynamic_equalizer (line 114)

threshold=-20 # 用户 dB；ffmpeg 范围 [0-100]，报错
threshold=20 # ffmpeg 7.1 范围 [0-100]，不再是 dB

ffmpeg 7.1 adynamicequalizer 的 threshold 范围 [0-100]（不是 dB）。 函数签名更新：默认 threshold=20，docstring 标注「不再是 dB」。

audionormalize.py（1 bug）

normalize_dynamic (line 46)

af = f"dynaudnorm=f={target_rms}:c={compress}" # f 实际是 frame length (10-8000 msec)
target_rms = 10 ** (target_rms_db / 20) af = f"dynaudnorm=targetrms={target_rms}:compress={compress}" # targetrms 是线性 (0-1)

用户传 dB，内部转 linear：-23 dB → 0.0708 (linear)。 函数签名更新：target_rms → target_rms_db（语义明示）。

audiotransform.py（1 bug）

excite (line 114)

af = f"aexciter=frequency={frequency}:amount={amount}" # ffmpeg 7.1 不认 frequency
af = f"aexciter=freq={frequency}:amount={amount}" # ffmpeg 7.1 用 freq (2000-12000 Hz)

audiochannel.py（3 bug）

widen_stereo (line 28)

af = f"stereowiden=...:dry={1.0}" # ffmpeg 7.1 不认 dry
af = f"stereowiden=...:drymix={drymix_val}" # drymix 范围 [0-1]

ffmpeg 7.1 stereowiden 把 dry 改成 drymix（范围 [0-1]）。

pan_audio (line 69)

def pan_audio(..., pan="0.5|c0=c1"): # Invalid channel layout '0.5'
def pan_audio(..., pan="stereo|c0=c1"): # 合法 layout 开头

ffmpeg 7.1 pan filter 要求第一个字符必须是合法 channel layout（如 stereo、mono），不是数字。

upmix_to_surround (line 86)

af = f"surround=chl={surround_type}" # ffmpeg 7.1 不认 chl
af = f"surround=chl_out={surround_type}:chl_in=stereo" # chl_out 是输出 layout

audiovisualize.py（3 bug + 1 关键架构变更）

waveform_image / spectrum_image / histogram_image

vf = f"showwavespic=s=..." run_ffmpeg([..., "-vf", vf, "-frames:v", "1", "-y", ...]) # ffmpeg 报 "Output file does not contain any stream"
fc = f"[0:a]showwavespic=s=...[v]" run_ffmpeg([..., "-filter_complex", fc, "-map", "[v]", "-frames:v", "1", "-update", "1", "-c:v", "png", "-y", ...]) # filter_complex 强制 audio → video 转换链

架构变更：从 -vf 切到 -filter_complex。 原因：-vf 在纯音频输入上无法产生 video output stream； -filter_complex 强制链式 audio → video 转换。 同时加 -update 1（image2 muxer 持续模式）和 -c:v png（强制 png 编码）。

audioeffect.py（1 bug）

spectral_tilt (line 73)

af = f"atilt=tilt={tilt}" # ffmpeg 7.1 不认 tilt
af = f"atilt=slope={slope}" # ffmpeg 7.1 用 slope (-1 到 1)

函数签名更新：tilt → slope。

videowatermark.py（1 bug + 1 表达式修复）

add_text_watermark

两处真错：

错 1：shadow 参数不存在

drawtext=...:shadow=1 # ffmpeg 7.1 drawtext 不认 shadow
shadow_opts = ":shadowx=2:shadowy=2:shadowcolor=black@0.5" drawtext=...{shadow_opts} # 用 shadowx/y/color 替代

错 2：overlay_w 表达式错误

x=main_w-overlay_w-10 # drawtext 不认 overlay_w（它属于 overlay filter）
x=main_w-tw-10 # tw 是 drawtext 自己的 text width

drawtext 的 x/y 表达式只能用 main_w/H/tw/th 等常量，不能引用 overlay_w。 POSITIONS 字典全局替换 overlay_w/h → tw/th。
