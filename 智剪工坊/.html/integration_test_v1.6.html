🐛 12 个 lib bug 总览

原因：v1.5 阶段 1 + v1.6 写的 ffmpeg 命令基于旧版本，ffmpeg 7.1 API 已变更。所有 bug 集中在「参数名 / 参数范围 / 输出流格式」三类错误。

#lib 模块函数问题
1audio.enhancedynamic_equalizerthreshold 范围错（dB vs [0-100]）
2audio.enhanceenhance_dialogdialoguenhance 不接受 level（ffmpeg 7.1 改为 enhance）
3audio.normalizenormalize_dynamicdynaudnorm f 范围错（[10-8000] 不是 dB）
4audio.transformexciteaexciter 不接受 frequency（API 变）
5audio.channelwiden_stereostereowiden 不接受 dry
6audio.channelupmix_to_surroundsurround 不接受 chl
7audio.channelpan_audiopan filter layout 错（0.5 不是合法 layout）
8audio.visualizewaveform_imageshowwavespic 输出流格式错
9audio.visualizespectrum_imageshowspectrumpic 同上
10audio.visualizehistogram_imageahistogram 同上
11audio.effectspectral_tiltatilt 不接受 tilt 参数
12video.watermarkadd_text_watermarkdrawtext 不接受 shadow 参数

📂 按模块分类

audioenhance 模块（2 bug）

enhance_dialog()
代码：af = f"dialoguenhance=level={level}"
ffmpeg 7.1 真参数：dialoguenhance=enhance=N (0-3, default 1)
⚠️ 影响：scripts/audio/voice_extract.py 第 82/92 行调用失败
dynamic_equalizer()
代码：adynamicequalizer=threshold=-20
ffmpeg 要求 threshold 范围 [0-100]（不是负 dB）

audionormalize 模块（1 bug）

normalize_dynamic()
代码：dynaudnorm=f=-23
ffmpeg 要求 f 范围 [10-8000]（线性值，不是 dB）

audiotransform 模块（1 bug）

excite()
代码：aexciter=frequency=N:amount=N
ffmpeg 7.1 aexciter 没有 frequency 参数（API 变）

audiochannel 模块（3 bug）

widen_stereo()
stereowiden 不接受 dry 参数
upmix_to_surround()
surround 不接受 chl 参数
pan_audio()
代码：pan=0.5\|c0=c1
ffmpeg 报 "Invalid channel layout '0.5'"（应该是 stereo\|c0=c1 等）

audiovisualize 模块（3 bug）

waveform_image() / spectrum_image() / histogram_image()
共同问题：-vf showwavespic=... 输出格式错（"Output file does not contain any stream"）
可能原因：缺 -c:v png 或输出格式参数

audioeffect 模块（1 bug）

spectral_tilt()
atilt 不接受 tilt 参数（ffmpeg 7.1 API 变）

videowatermark 模块（1 bug）

add_text_watermark()
代码：drawtext=...:shadow=1
ffmpeg 7.1 drawtext 没有 shadow 参数（用 box/bordercolor 替代）
