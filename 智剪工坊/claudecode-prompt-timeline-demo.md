## 窗口布局（核心概念）

传统 NLE（Premiere/剪映/DaVinci）的核心设计哲学：**Timeline 就是预览，预览就是 Timeline。**
不是"一个播放器 + 一个独立的 timeline"上下排列，而是一个统一的区域：

```
┌──────────────────────────────────────────────────────────┐
│  ▶  00:05  /  01:23                                      │  ← 时间指示器
├──────────────────────────────────────────────────────────┤
│                                                          │
│   [ 帧缩略图序列 ────────────── 拖动即预览 ]              │  ← 帧序列（timeline主体）
│   [                             ]                        │
│   [                             ]                        │
│        ▲  播放头（红色竖线）                              │
├──────────────────────────────────────────────────────────┤
│  ─────────────────────────────────────────── 00:00 01:00│  ← 时间刻度
└──────────────────────────────────────────────────────────┘
```

视频加载后，区域内有画面（第一帧或当前帧），用户拖动/点击任意位置，视频跳转到该帧并显示在帧条上。这样用户不需要额外的"播放器窗口"，timeline 本身就是最大的预览器。
同时支持外接独立的视频播放窗口（点击右上角按钮在新窗口打开），方便看细节。

## 帧提取策略

**第一阶段 — 即时加载（< 200ms 内完成）**
- 视频元数据加载后，只提取第一帧作为 poster
- 显示"加载帧序列中…"进度提示（如"35/120 帧"）
- 用户在第一帧出现后就可以开始操作

**第二阶段 — 后台渐进提取**
- 用 requestIdleCallback 或 setTimeout 分片提取剩余帧
- 每次提取 5-10 帧，渲染到 timeline
- 不阻塞用户交互（用户可以在提取过程中点击、缩放）

**帧提取密度（分 4 档）：**

| 缩放级别 | 密度 | 60s 视频帧数 | 视觉体验 |
|---|---|---|---|
| L0 最小（全景） | 每 2 秒 1 帧 | ~30 | 看全局节奏 |
| L1 默认 | 每秒 1 帧 | ~60 | 正常浏览 |
| L2 放大 | 每秒 2-4 帧 | ~120-240 | 看细节 |
| L3 最大（帧级） | 原生帧率（30fps） | ~1800 | 帧级精确 |

- 每一档预先提取一次，缓存，不重复提取
- 用户缩放触发档位切换时，如果该档位帧已经提取，直接渲染；如果没提取，后台渐进提取并实时填充
- **关键技术点**：每帧用 canvas 提取为 thumbnail blob URL（image/jpeg, quality 0.6），缓存为数组，不重复 drawImage

## 缩放交互（Ctrl+滚轮）

- Ctrl + wheel up → 放大一级（L(n) → L(n+1)）
- Ctrl + wheel down → 缩小一级（L(n) → L(n-1)）
- 每级对应固定的像素密度（px/帧）：L0=40px, L1=80px, L2=160px, L3=320px
- **最小缩放**：当所有帧 × L0 密度 ≤ 容器宽度时，不再缩小（即 L0 已是最小）
- **最大缩放**：L3（帧级），不再放大
- 缩放点在鼠标位置（zoom to cursor，即以鼠标锚点缩放，不是以中心缩放）
- 切换档位时：当前帧的数据若已缓存则立即渲染，否则显示"提取中"提示（同渐进逻辑）
- 缩放过渡使用 CSS transition: transform 0.15s ease-out

## 点击/拖动跳转

- 在 Timeline 上鼠标左键单击 → 视频跳转到对应时刻
- 计算方法：clickX / timelineScrollWidth × 视频总时长 = 目标 currentTime
- 跳转后立即更新帧缓冲（显示该时刻对应图片），200ms 内重新 drawImage
- 拖动播放头（scrubbing）→ 下落暂停播放 → 中间实时切换显示当前帧 → 释放恢复播放

## intent.json 数据结构

在现有源文件格式基础上，video 对象新增 `regions` 字段：

```json
{
  "_meta": {
    "tool": "智剪工坊",
    "version": "0.8",
    "revision": 1
  },
  "project": {},
  "videos": [
    {
      "file": "xxx.mp4",
      "index": 1,
      "summary": "...",
      "intent": "...",
      "ops": {
        "speed-up": { "on": true, "factor": 2 },
        "mute": { "on": true }
      },
      "regions": [
        {
          "id": "r1",
          "startSec": 5.0,
          "endSec": 15.0,
          "ops": {
            "speed-up": { "on": true, "factor": 4 },
            "mute": { "on": true }
          }
        },
        {
          "id": "r2",
          "startSec": 30.0,
          "endSec": 45.0,
          "ops": {
            "reverse": { "on": true }
          }
        }
      ]
    }
  ]
}
```

- `id` = 唯一标识（格式 "r" + 递增数字，便于调试和对比）
- `startSec` / `endSec` = 秒数（浮点数），不是 "0:30" 格式字符串
- `ops` 结构与整视频 ops 完全一致
- 没有 regions 时，不输出该字段（保持向后兼容）
- 新老版本交互：读取旧 intent.json（无 regions）应正常加载不报错

## 关键代码结构建议

1. **状态管理**：每个视频卡片维护一个对象，包含：
   ```
   {
     entry, frameCache, regions, thumbnailBlobs, zoomLevel, selectedRegionId, isExpanded, videoEl
   }
   ```
2. **帧提取器**：抽帧函数接收 (videoFile, intervalSec, maxWidth, maxHeight)，返回 Promise<Blob[]>
3. **timeline 渲染器**：renderTimeline() 函数，按当前 zoomLevel 和 frameCache 绘制
4. **区域管理器**：addRegion / removeRegion / mergeOverlappingRegions / getOpsForTime(t)
5. **事件委托**：用事件委托（video-card 级别监听）替代逐元素绑定，减少内存开销
6. **缩放档位常量**：ZOOM_LVL = [{level:0, pxPerFrame:40}, {level:1, pxPerFrame:80}, ...]
