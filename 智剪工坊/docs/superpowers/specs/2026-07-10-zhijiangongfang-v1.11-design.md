Step 5  用户把 json 路径给 AI

Step 10 AI 让用户过目 → 不满意回 Step 7 / 满意进 Step 11
Step 11 进入精加工 — 两条路：
        ├ 路径 A：纯语言交互（AI 问答深挖 → 用户确认 → AI 对应底层能力 → 出成片 → 人工确认 → 不满意回退）

## 2. 用户隐藏的产品决策（6 个 insight）

> 这些是用户回答里**显式说出但没意识到自己已经做了产品决策**的关键判断。

### 2.1 日志 = 流程合规性审计 + 错误诊断**双用途**
- 目的 1：判断 AI 是否按流程在走（**预防走偏**）
- 目的 2：出错时定位是 AI 流程问题还是底层脚本 bug（**事后诊断**）

### 2.2 模板 = 给新用户的捷径，**不是炫技**
- 核心用户群：**不懂视频剪辑的小白**（不是剪映玩家）
- 模板的作用：让小白跳过"学理论 → 学操作 → 学参数"直接出片
- 模板 YAML 应：**强默认 + 弱可调** + **自动应用炫酷参数**

### 2.3 红线 = 触发 AI 审查，**不要自动化工具**
- 明确拒绝 `_check_consistency.py` 类的自动化保护
- 红线是「AI 触发时主动读契约清单」，不是「工具自动跑」

### 2.4 粗加工产物 = 通用素材集（**独立可用**）
- 可以新开工作区把这些素材作为原始素材再走一遍智剪工坊
- 可以交给剪映等其他软件用
- **意味着粗加工产物 = 自包含、可独立使用、可二次加工的素材集**

### 2.5 Step 7 意图对齐必须有"风险提示"
- 必须告诉用户：「当前意图会对哪些文件执行什么操作」
- 必须强调："一旦错误，返工会消耗很多时间"

### 2.6 模板只是 demo，未开发多个
- 当前 `模板/健身vlog.yaml` 是 demo，**不是终态**
- 真实产品价值 = AI 自动从用户素材推断用哪个模板 + 自动应用

## 3. 已对齐的关键决策（4 个核心选择）

### 3.1 红线实现 = **纯文字契约**（不要自动化工具）

| 维度 | 决策 |
|---|---|
| 自动化保护 | ❌ 不开发 `_check_consistency.py` |
| AI 触发 | ✅ 改完主动读 SKILL.md §能力链路完整性 + 5 步自检清单 |
| 报告形式 | AI 自报报告（chat + 写入 logs/） |
| 演进路径 | 文字契约先稳定，未来可选择性加加速器 |

### 3.2 AI 行为日志粒度 = **C（流程 + 决策 + 思考链）**

| 维度 | 决策 |
|---|---|
| 粒度 | **流程节点 + 决策 + 思考链**（最强诊断） |
| 风险 | ⚠️ LLM self-reference（AI 看到自己思考会强化早期决策） |
| 缓解方案 | **XML 标签隔离** + **阶段 checkpoint 强制重决策**（待详细设计） |

### 3.3 AI 行为日志存储 = **双格式 Markdown + JSONL**

| 文件 | 用途 | 写入时机 | 内容 |
|---|---|---|---|
| `<task_id>.md` | 人类可读总结 | AI 阶段结束 + 异常时 | 阶段名 + 关键决策总结 + 异常 |
| `<task_id>.jsonl` | 机器可读详细 | 每次 CLI 调用前 append | 每行结构化 entry |

**路径约定**：
```
<workspace>/00_智剪/中间产物/logs/
├── <task_id>_<timestamp>.md        # 人类总结
└── <task_id>_<timestamp>.jsonl     # 机器详细
```

**JSONL 字段 schema**（草案）：
```json
{
  "time": "2026-07-10T10:30:15",
  "stage": "1",
  "step": "2",
  "action": "trim",
  "decision": "trim 0-5s",
  "thinking": "视频抖动 + 用户提到...",
  "result": "exit 0",
  "error": null
}
```

### 3.4 异常处理 = **B（重试 2-3 次后停下报告）**

| 维度 | 决策 |
|---|---|
| 重试次数 | 2-3 次同类问题 |
| 临时性失败 | 自动恢复（网络抖动等） |
| 持续性失败 | 停下报告（CUDA OOM 等） |
| 报告内容 | 异常类型 + 重试次数 + 失败原因 + 影响下游 stage |

### 3.5 日志输出位置 = **文件**（不是聊天）⚠️ 用户关键澄清

> 用户在第二轮澄清：日志输出到**文件**，不是聊天回复。
> 
> **原因**：聊天会污染 LLM context，文件零成本 + 持久化 + 可审计。

## 5. 设计细节（6 大改造点的设计草案）

### 5.1 Logging 系统

**新建文件**：`references/AI行为日志协议.md`

**核心设计**：
- 路径：`<workspace>/00_智剪/中间产物/logs/<task_id>_<timestamp>.{md,jsonl}`
- 文件命名：`<task_id>` 来自 intent.json.project.title 或时间戳
- 写入触发：
  - `.jsonl`：**每次 CLI 调用前**（AI 用 Python `open('a').write(json.dumps(...)+'\n')`）
  - `.md`：**每个 stage 结束时** + **异常时**（AI 重写整个文件或 append stage section）
- 文件生命周期：跟随任务，剪映导出时可选择归档到 `.archive/logs/`

**待细化**：
- 文件命名约定（task_id 怎么生成？时间戳格式？）
- 写入触发是 AI 自动还是 CLI wrapper 自动？（**已确认 AI 自己写文件**，不需要 wrapper）
- .md 的章节结构（每 stage 一节？含哪些子节？）

### 5.2 AI 行为日志协议

**新建文件**：`references/AI行为日志协议.md`

**核心设计**：
- **写入格式**（每次 AI 决策前）：
  ```
  <thinking>
  [阶段 X.Y] 决策：...
  思考：考虑了 A/B/C，B 最优因为...
  </thinking>
  
  <action>
  [JSONL append]
  </action>
  ```
- **XML 隔离**：`thinking` 放 `<thinking>` 标签，AI 在重新读自己 log 时不主动回看 thinking 内容
- **阶段 checkpoint**：每 stage 结束时 AI 必须 re-decide "我现在还坚持前面的决策吗？"
- **审计时机**：
  - 每 stage 结束
  - 异常时
  - 决策犹豫时（用户在 chat 追问）

**待细化**：
- thinking 标签的 SKILL.md 提示模板
- checkpoint 重决策的触发格式
- 用户如何在 .md 里快速浏览 thinking

### 5.3 异常处理协议

**新建文件**：`references/异常处理协议.md`

**核心设计**：
- **重试规则**：
  - 同类问题（同错误信息/同 exit code）最多重试 2-3 次
  - 重试间隔：指数退避（1s, 2s, 4s）
  - 临时性失败（网络/CUDA OOM 偶发）自动恢复
  - 持续性失败停下报告
- **报告格式**（写入 logs/.md 异常节 + chat 通知用户）：
  ```
  [异常报告]
  类型：CUDA OutOfMemory
  位置：scripts/audio/separate.py (demucs)
  重试：3/3 全部失败
  原因：GPU 显存不足（已用 7.8GB / 总 8GB）
  影响：Stage 2.3 声源分离失败，下游 Stage 2.4 ASR 无法进行
  建议：换 CPU 模式 / 释放 GPU 显存 / 用更小模型
  ```
- **降级选项**（如有）：CUDA OOM → 改 CPU；whisper large-v3 失败 → 改 base

**待细化**：
- 哪些异常触发"自动降级" vs "直接报告"？
- 重试间隔的精确规则
- 报告是否需要用户确认才能继续？

### 5.4 红线契约强化

**新建文件**：`references/红线契约-AI触发审查.md`

**核心设计**（基于现行 §能力链路完整性）：
- **触发条件**：
  - 用户说"修改 SKILL.md" / "改 references/" / "改 scripts/ 或 lib/"
  - AI 主动判断"这个改动会影响链路一致性"
- **5 步自检清单**（细化现行）：
  ```
  □ 1. 我改了什么？→ 列出改动清单（文件 + 段落 + 原因）
  □ 2. 链路里其他位置是否需要同步？
        - 改 scripts/ → 同步 SKILL.md 触发词 + references/*.md
        - 改 SKILL.md 触发词 → 同步 references/*.md + scripts/
        - 改 references/*.md → 同步 SKILL.md 触发词
  □ 3. 主动检查（AI 自己读文件）：
        - 读 SKILL.md 触发词 → 新能力是否已声明？
        - 读 references/ 对应文档 → 是否覆盖新能力？
        - ls scripts/{audio,asr,video,ai,batch}/ → 新能力是否实际存在？
  □ 4. 全部一致 → 标记完成 + 写入 logs/
  □ 5. 发现不一致：
        - 严重不一致 → 立即停下报告
        - 轻微不一致 → 自动修 + 在最终回复里报告
  ```
- **报告模板**：写入 logs/<task_id>.md 红线节 + chat 输出

**待细化**：
- 哪些改动触发"严重不一致"自动停下？
- 轻微不一致"自动修"的边界

### 5.5 Step 7 操作影响告知

**新建文件**：`references/意图对齐-操作影响告知.md`

**核心设计**：
- **触发时机**：Step 7 意图对齐完成 + 用户即将确认前
- **告知内容**：
  ```
  [操作影响]
  阶段：Stage 2 粗加工（视频1-3）
  影响文件：
    - 输入：video_1.mp4, video_2.mp4, video_3.mp4
    - 输出：粗加工/单视频/video_1_trimmed.mp4, ...
  操作类型：trim + fade-in + color preset
  风险等级：低（不可逆操作仅 trim，原始 video_*.mp4 不动）
  返工成本：低（粗加工产物可重新生成，原始素材安全）
  ```
- **风险等级分类**：
  - **低**：可重新生成（粗加工产物）
  - **中**：覆盖原文件（导出剪映前的最终成片）
  - **高**：原始素材被修改（违反"原始素材不动"原则）

**待细化**：
- 风险等级判定规则（自动 vs 用户标？）
- 告知时机（每次 Stage 结束？只在 Step 7？）

### 5.7 Progressive Disclosure 原则（升级点 1：明确定义规则）

**核心问题**：SKILL.md 不是越短越好——太短 AI 失去核心 context，太长 context 爆炸。

**判定规则**：

```
进 SKILL.md 的（AI 必须每次加载）：
  - 触发词 YAML（AI 检索入口，~115 行）
  - 子技能索引 + 路由表（AI 调度决策，~30 行）
  - 红线 / 能力链路完整性（宪法，~80 行）
  - 工作区结构约定（路径契约，~30 行）
  - 6 象限操作清单 schema（执行契约，~20 行）

进 references/ 的（按需加载）：
  - 任何 > 20 行的详细协议 / 参考表 / 示例
  - 子技能详细文档（音频/视频/ASR 等）

```

**应用结果**：当前 SKILL.md 估算 800 行 → 进一步压缩到 **600 行**（激进精简 47%）

### 5.9 Loading 触发器（升级点 3：AI 何时加载 references/）

**核心问题**：当前 SKILL.md 是被动引用（"详见 references/XX.md"），AI 不会主动加载。

**解决方案**：在 SKILL.md 加**显式触发器**，告诉 AI 何时加载哪个 references/。

**触发器清单**（草案，写入 SKILL.md）：

```
Loading 触发器（AI 必读）：

- 路由命中"美颜/磨皮/瘦脸/大眼" → 加载 references/美颜-四种人脸美化.md
- 路由命中"调色/LUT/视频调色/调色预设" → 加载 references/调色预设-18种预设LUT风格迁移.md
- 路由命中"ASR/Whisper/语音转文字/说话人分离/声源分离" → 加载 references/ASR链路-声源分离说话人分离Whisper烧字幕.md
- 阶段 4 产物审查 → 加载 references/产物审查-用户过目指南.md
- AI 撞异常（ffmpeg 失败/CUDA OOM/whisper 失败）→ 加载 references/异常处理协议.md
- AI 修改 SKILL/md/脚本 → 加载 references/红线契约-AI触发审查.md
- 阶段 0 用户选 ①（从零开始） → 加载 references/主流程-阶段编排.md
- AI 路由"音频配乐/BGM/混音/降噪/声源分离/说话人分离" → 加载 references/音频配乐-BGM循环淡入淡出节拍.md
```

**额外规则**：
- AI 不确定时，先 grep `references/` 目录，按文件名匹配
- 加载 references/ 后，把关键摘要写回 logs/<task_id>.md（避免重复加载）
