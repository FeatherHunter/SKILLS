## 2. 状态机图（带回退的状态机，不是线性管道）

状态机（v1.12 明确回退粒度）：

```
                         ┌─→ [回退到 Step 7：重新对齐意图]
                         │     重做范围：Step 7+（粗加工可能保留）
                         │
状态:    意图 → 粗加工 → 精加工 → 出片
              │        │        ↑
              │        │   ┌────[回退到 Step 11 整体回退]──┘
              │        │   │     清空精加工产物，从 Step 11 重做
              │        │   │
              │        │   └─→ [回退到 Step 11 局部回退]
              │        │         保留精加工产物，只重跑某个 stage
              │        │
              │        └─→ [复用：备份 + 全新会话]
              │
              └─→ [Step 10 审查不通过 → 回 Step 7]
```

**回退粒度由用户在审查时决定**：
- 整体回退："全部重做"
- 局部回退："只改 X stage"

AI 必须问清楚，不替用户决定。

**每个状态的进入条件**：

| 进入状态 | 触发条件 | 输出物 |
|---|---|---|
| 意图 | Step 7 用户确认 | `intent.json` + 操作清单 |
| 粗加工产物 | Step 9 跑完 | `00_智剪/粗加工/` 5 类 |
| 精加工产物 | Step 11 跑完 | `00_智剪/精加工/` 终稿 |
| 出片 | Step 13 跑完 | `00_智剪/出片/<title>_final.mp4` |

**每个状态的退出条件**：

| 退出状态 | 必须满足 |
|---|---|
| 意图 → 粗加工 | 用户明确同意（Step 8）|
| 粗加工 → 精加工 | Step 10 用户审查 OK |
| 精加工 → 出片 | Step 12 用户审查 OK |
| 出片 | 文件生成 + 命名 + 编码 |

## 4. 详细步骤契约

### Step 1-4 初始化（用户 + AI 共担）

```
Step 1  用户加载技能（主动动作）
Step 2  用户说"这次是剪辑完整 vlog"（或类似触发意图）
Step 3  AI 用 shell 命令主动打开 智剪工坊-意图编辑.html
        命令：**优先 Chrome**（File System Access API 唯一完整支持）:
          - Windows:  Start-Process "C:\Program Files\Google\Chrome\Application\chrome.exe" "<skill_root>/智剪工坊-意图编辑.html"（未装 Chrome 则 fallback 默认浏览器）
          - macOS:    open -a "Google Chrome" "<skill_root>/智剪工坊-意图编辑.html"（未装则 fallback `open`）
          - Linux:    google-chrome "<skill_root>/智剪工坊-意图编辑.html"（未装则 fallback `xdg-open`）

### 4.5 片头/片尾模板（v1.20 新增 · DAY17 教训）

**核心规则**：想在主视频**首/尾**追加独立的"黑屏 + 文字"段
（如片头标题、收尾致谢、下期预告），**必须**拆成 3 步：

1. **生成独立段**：用 ffmpeg `color=black:s=WxH:d=N` 生成 N 秒黑屏源
2. **加字**：在黑屏源上用 `opening.py add --text "..." --region middle-center
   --duration N` 叠加文字
3. **拼接**：把这个独立段 prepend/append 到主视频（`trim.py concat`）

**为什么不能直接用 `opening.py add --start N`？**

`opening.py add` 的 `--start / --duration` 是**相对输入视频内部时间轴**的偏移量，
**不是相对于成片末尾**。所以：
- `--start 60` = "在输入视频的第 60 秒开始显示文字" → **叠字**，不是末尾追加
- 没有任何参数能实现"在视频**末尾**追加 2 秒黑屏 + 文字"

**5 个场景适用表**：

| 场景 | 是否走本模板 | 替代方案 |
|---|---|---|
| A. 视频**末尾**加 2s 黑屏 + "晚安" | ✅ 走 | 独立段 + opening.py + concat |
| B. 视频**开头**加 2s 黑屏 + "DAY 17" 片头 | ✅ 走 | 独立段 + opening.py + concat（prepend） |
| C. 视频**中间**某段加 caption（"训练开始"） | ❌ 不走 | opening.py add 直接用 |
| D. 视频**末尾** freeze 帧 + 文字覆盖（不要黑屏） | ❌ 不走 | ffmpeg freeze 滤镜 |
| E. 视频**末尾**直接 BGM 淡出 + 黑场（无文字） | ❌ 不走 | mix.py `--bgm-fade-out` |

**反模式（踩坑记录）**：
- ❌ 想"在主视频末尾加 ending 文字" → 试图用
  `opening.py add --start {视频总时长-2}` → 错：文字被截在最后一帧，无黑屏过渡
- ❌ 试图在原视频上用 ffmpeg drawtext 末尾追加 → 复杂、不能复用 ending 模板
- ✅ 正确做法：**先做 2s 黑屏段（独立段）→ opening.py 加字 → trim.py concat 拼接**

**DAY17 实战**（沉淀 27.1s 成片的 ending 段）：
- 段 1：`ffmpeg -f lavfi -i "color=black:s=1080x1920:d=2:r=30"` → `_ending_black_2s_raw.mp4`
- 段 2：`opening.py add --text "明天第十八天，继续推！" --region middle-center
  --duration 2 --font-size 60 --no-shadow` → `_ending_with_text.mp4`
- 段 3：`trim.py concat` 把 `_ending_with_text.mp4` append 到 6 段主体后
- 总时长：6 段 25.1s + ending 2s = 27.1s

## 5. 二次加工工作流（粗加工完成后）

> **你的原话**："如果满意 但担心出错可以将满意的输出粗加工产品放到某个目录 然后全新对话加载该技能 可以进一步二次加工"

### 5.1 粗加工完成时 AI 必告知协议

**Step 9 完成后，AI 必须主动告知**：

```
AI 输出：
"粗加工已完成。产物在以下目录：
  - 单视频/   （X 个文件）
  - 组合/     （Y 个文件）
  - 文字稿/   （Z 个文件）
  - cover/    （W 个文件）
  - 决策.md

【下一步选项】
  A) 满意 + 想继续：进入 Step 10 审查 → Step 11 精加工
  B) 满意 + 担心出错：可执行'复用备份'（见 §5.2）
  C) 不满意：回 Step 7 重新对齐意图"
```

**announce fallback（v1.12 强制 P0-4）**：

AI 必须在粗加工完成时写日志 `logs/<task_id>.jsonl`：
```json
{"stage":"粗加工","action":"announce_complete","decision":"粗加工完成，X 产物","result":"success"}
```

**自我检测协议**：
- AI 写完上述日志后**必须主动**读自己的日志
- 如果日志中没有 `announce_complete` → AI 必须立即告诉用户"粗加工完成"
- ❌ 禁止 AI 跳过 announce 直接进入 Step 10

### 5.2 复用备份（用户主动操作）

> **完整规范见 `references/二次加工-复用工作流.md`**（v1.12 重写）

**用户**：
1. 创建目录 `<workspace>/00_智剪/粗加工_备份_<YYYYMMDD_HHMMSS>_<task_id_slug>/`（命名格式见 §5.2.1）
2. 复制 `00_智剪/粗加工/` 的全部内容到该目录
3. AI 自动生成 `BACKUP_NOTE.md`（见二次加工 references §2.3）
4. **结束当前对话**，开新对话

**全新对话加载技能后，AI 必检测**（完整性校验见 §5.4）：

```
AI 检测：
  - 工作目录存在 00_智剪/粗加工_备份_*/ 目录？
  - 校验完整性：单视频/ 决策.md BACKUP_NOTE.md 必须存在

如果是 → 提示用户：
"检测到粗加工备份（00_智剪/粗加工_备份_20260710_140000_fitness-vlog/）。
  任务：健身 vlog 训练
  时间：2026-07-10 14:00
  产物：单视频 3 个，组合 2 个，文字稿 3 个，封面 1 个
  决策摘要：trim 0-5s、color 电影感、BGM 添加
  BACKUP_NOTE.md：✓ 完整

  - A) 复用现有粗加工 → 直接进 Step 11 精加工（跳过 Step 8-10）
  - B) 重新开始 → 删除备份目录，按正常流程走"
```

### 5.3 复用流程

```
新 Step 7   AI 重新对齐意图（用户可能想换路径）
新 Step 11  AI 直接进精加工（基于复用粗加工）
新 Step 12-13  审查 + 出片
```

**约束**：
- 复用模式下**禁止重新跑粗加工**（避免覆盖用户备份）
- AI 复用时**只读**备份目录，不修改
- 如果备份有多个（不同时期），AI 让用户选哪个

### 5.4 intent.json 备份（v1.12 新增 P1-4 修复）

**触发**：Step 7 完成（用户确认最终意图后）

**AI 必做**：
1. 复制 `intent.json` 到 `00_智剪/中间产物/intent_backup_<时间戳>.json`
2. 写 `logs/<task_id>.jsonl`：stage="1", action=backup_intent

**用途**：
- 二次加工复用时如 intent.json 损坏/丢失，可从备份恢复
- 调试时可对比原始意图与粗加工结果

**用户决策**：
- ✅ 默认：每次 Step 7 完成都备份
- ⚠️ 可选：用户说"不备份"则跳过

## 7. 关键概念定义

### 7.1 粗加工 vs 精加工

| 概念 | 范围 | YAML 参与 |
|---|---|---|
| **粗加工** | 基础处理（trim/color/normal/ASR/拼接/封面草稿）| ❌ 不涉及 |
| **精加工** | 创意编排（节奏/转场/数据叠加/AI 创意）| ⚠️ 仅模板路径涉及 |

### 7.2 二次加工 vs 二次使用

| 概念 | 含义 |
|---|---|
| **二次加工** | 用户复制粗加工备份 + 新会话加载技能 → 直接进精加工 |
| **二次使用** | 粗加工产物导出剪映/Premiere 等其他工具继续编辑 |
