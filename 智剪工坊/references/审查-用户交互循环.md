# 审查-用户交互循环（Step 10 + Step 12 详细）

> **何时加载**：AI 进入 Step 10（粗加工审查）或 Step 12（精加工审查）时必读
> **目的**：定义两次审查的统一协议——列出产物、用户逐项 OK、讨论队列、收敛异常
> **范围**：主流程 Step 10 + Step 12，对应 SKILL.md 老版本"阶段 4 产物审查"

---

## 1. 概述

**两次审查的共同模式**：
```
S#.1 AI 列出全量产物清单（路径 + 摘要 + 异常）
S#.2 用户逐项检查（OK / 有问题 / 跳过）
S#.3 讨论队列每项 → AI 重跑/调整
S#.4 全部 OK 或用户说"全部 OK, 出成片" → 进入下一阶段
```

**两次审查的区别**：

| 维度 | Step 10 粗加工审查 | Step 12 精加工审查 |
|---|---|---|
| **审查对象** | 粗加工产物（单视频/组合/文字稿/cover/决策.md）| 精加工产物（路径 A 或 B 终稿）|
| **不满意的回退** | 回 Step 7（重新对齐意图）| 回 Step 11（重做精加工）|
| **问题诊断** | 基础处理错（trim 错、ASR 错、拼接错）| 创意编排错（节奏错、转场错）|

## 2. Step 10 粗加工审查详细

### S10.1 列出粗加工产物清单

**输入**：`00_智剪/粗加工/` 全量产物
**输出**：AI 输出 markdown 清单 + 写入 `00_智剪/粗加工/中间产物/审查清单_粗加工.md`

**清单格式**：

```markdown
# 粗加工产物审查清单

## 单视频（X 段）
- [ ] video_1.mp4（duration: Xs, applied_ops: [cut-middle, color], error: null）
- [ ] video_2.mp4（duration: Xs, applied_ops: [trim], error: null）
- [ ] video_3.mp4（duration: Xs, applied_ops: [cut-middle], error: ⚠️ profile.error 非空）

## 组合（Y 段）
- [ ] seq_开场.mp4（duration: Xs, xfade 链: [fade]）
- [ ] seq_高潮.mp4（duration: Xs, xfade 链: [wipe]）

## 文字稿（Z 份）
- [ ] 视频_1.md（X 段，Y 字）
- [ ] 全部.md（汇总）

## 封面（W 张）
- [ ] cover_draft.jpg（来源：AI 生成 / 用户提供）

## 决策报告
- [ ] 决策.md（异常情况 + 用户额外要求 + 模板建议）
```

### S10.2 用户逐项检查

```
(a) 标注「OK」→ 该项通过
(b) 标注「X 有问题: <具体描述>」→ 该项进讨论队列
(c) 标注「跳过」(可选) → 该项按意图保留但不审查
```

### S10.3 讨论队列处理

```
对每项问题：
  - 单视频问题 → 调 lib/video_processing.py 重跑该视频 / 调 atomic CLI 改 ops
  - sequence 问题 → 调 scripts/video/xfade.py 重拼 / 改 sequences
  - 文字稿问题 → 重新跑 ASR（换模型/换参数）
  - 封面问题 → 重新跑 scripts/ai/cover.py

每修完一项 → 回到 S10.1 重新摘要
```

### S10.4 全部 OK 后

```
写 00_智剪/粗加工/中间产物/审查签字.md：
  project: <项目名>
  时间戳: <ISO 时间>
  通过项: [列表]
  跳过项(可选): [列表]
  用户签字: <确认信息>

进入 Step 11 精加工
```

### 粗加工完成后 AI 必告知协议（关键）

> 你的原话："粗加工后 告诉用户 粗加工完成 素材 在 哪些目录 然后提醒用户..."

```
AI 输出（粗加工完成后）：
"粗加工已完成。产物在以下目录：
  - 单视频/   （X 个文件）
  - 组合/     （Y 个文件）
  - 文字稿/   （Z 个文件）
  - cover/    （W 个文件）
  - 决策.md

【下一步选项】
  A) 满意 + 想继续：进入 Step 10 审查 → Step 11 精加工
  B) 满意 + 担心出错：可执行'复用备份'（详见 references/二次加工-复用工作流.md）
  C) 不满意：回 Step 7 重新对齐意图"
```

## 3. Step 12 精加工审查详细

### S12.1 列出精加工产物清单

**输入**：`00_智剪/精加工/终稿/精加工_final.mp4` + 路径 A/B 中间产物
**输出**：AI 输出 markdown 清单 + 写入 `00_智剪/精加工/审查清单_精加工.md`

**清单格式**：

```markdown
# 精加工产物审查清单

## 终稿
- [ ] 精加工_final.mp4（duration: Xs, 来源: 路径A v1 / 路径B stage 4）

## 路径 A 候选（如果是纯语言）
- [ ] AI编排_v1.mp4
- [ ] AI编排_v2.mp4
- [ ] AI编排_v3.mp4（如有）

## 路径 B 中间产物（如果是模板）
- [ ] stage_1_节奏决策_产物.mp4
- [ ] stage_2_时间排序_产物.mp4
- [ ] stage_3_转场_产物.mp4
- [ ] stage_4_数据叠加_产物.mp4
```

### S12.2 用户逐项检查

```
(a) 标注「OK」→ 该项通过
(b) 标注「X 有问题: <具体描述>」→ 该项进讨论队列
(c) 标注「跳过」→ 按意图保留
```

### S12.3 讨论队列处理

```
对每项问题：
  - 路径 A 候选问题 → 用户选其他候选版本，或要求 AI 出新版本
  - 路径 B 某 stage 问题 → 回到 yaml 重新跑该 stage
  - 终稿问题 → 回 Step 11 重做精加工

每修完一项 → 回到 S12.1 重新摘要
```

### S12.4 全部 OK 后

```
写 00_智剪/精加工/审查记录.md：
  project: <项目名>
  时间戳: <ISO 时间>
  通过项: [列表]
  跳过项(可选): [列表]
  用户签字: <确认信息>

进入 Step 13 出片
```

## 4. 异常处理（两次审查通用）

### 4.1 反复修改不收敛（v1.12 阈值可配置）

**默认阈值**（不同问题不同）：
- 基础问题（trim/color/ASR）：3 次
- 创意问题（路径 A 编排/路径 B stage）：5 次
- 用户主动设的阈值：覆盖默认

**触发暂停条件**（任一）：
- 达到该问题类型的阈值
- 用户说"先停一下" / "我看看"
- AI 跑出相同错误 3 次（说明不是参数问题，是逻辑问题）

**AI 暂停行为**：
1. 写 logs/<task_id>.md `## Review Issue` 记录每次重跑的差异
2. 明确告知用户：「此问题 X 次未通过。你想：
   (a) 接受现状
   (b) 继续重跑（增加阈值）
   (c) 跳过此项
   (d) 换种方式（如改参数/换工具）」
3. 等用户明确决策

**用户决策后**：
- (a) 标 OK → 继续
- (b) 增加阈值 → 继续重跑
- (c) 跳过 → 标"跳过"，继续
- (d) 改参数 → 重跑（重置次数）

### 4.2 用户始终不确认

```
用户不主动说 OK → 不进入下一阶段
不假装出成片 → 不假装粗加工/精加工完成
```

### 4.3 AI 强行"通过"

```
❌ 禁止 AI 自己说"看起来 OK"代替用户决策
❌ 禁止 AI 跳过审查直接进下一步
✅ 必须等用户明确输入"OK"或"通过"
```

## 5. 抗用户疲劳协议（v1.12 新增）

### 5.1 批量汇报（粗加工）

**触发**：Step 9.3 单视频处理 ≥ 10 个视频

**AI 行为**：
- ❌ 禁止：每处理完一个就立即汇报（"已处理 video_1.mp4"）
- ✅ **必须**：每 10 个汇报一次
- ✅ **必须**：汇报格式：

```
[批量汇报 1/5]
  已完成：video_1 至 video_10
  成功：8 个
  异常：2 个（profile.error 非空）
  下次汇报：video_11 至 video_20
  继续处理中...
```

### 5.2 分组审查（Step 10 + Step 12）

**AI 行为**：
- ❌ 禁止：列出所有 100+ 产物让用户逐项标 OK
- ✅ **必须**：按类别分组（单视频 / 组合 / 文字稿 / 封面），用户可只审一类
- ✅ **必须**：提供"一键全部 OK" 选项（前提是用户**明确说**"我信任你，全部 OK"）

```
AI 输出格式：
"粗加工产物分 4 类：
  - 单视频：30 个
  - 组合：5 个
  - 文字稿：10 个
  - 封面：3 个

【审查选项】
  A) 全部 OK（基于产物统计 + 决策.md 推荐）
  B) 只审单视频
  C) 只审组合
  D) 全部详细审查"
```

### 5.3 精加工路径 A 选问（v1.12 优化）

**AI 行为**：
- ❌ 禁止：5 个核心问题**全部**问（用户答 5 轮）
- ✅ **必须**：AI 评估用户意图清晰度
  - 意图清晰（如"做一个热血健身 vlog"） → AI 自动推荐编排方案 → 用户只确认
  - 意图模糊（如"做个视频"） → AI 选 3 个核心问题问（其他 2 个默认）

## 6. 日志集成（v1.12 强制）

**审查时 AI 必写日志**：

每次审查（Step 10 + Step 12）AI 必须写：
- `logs/<task_id>.jsonl`：`stage=step_10_review` 或 `step_12_review`
- `logs/<task_id>.md`：`## Step 10 粗加工审查` 或 `## Step 12 精加工审查` section

**每个 OK/有问题决定都写一行 JSONL**：
```json
{"time":"2026-07-10T11:00:00","stage":"step_10_review","step":"video_3","action":"review","decision":"有问题: ASR 文字错误","thinking":"用户描述'ASR 把'的'识别成'得''","result":"flagged_for_fix","error":null}
```

## 7. 关键原则（原 §5）

### 7.1 零隐藏产物

- AI **必须**把粗加工/精加工全部产物列给用户
- ❌ 不得省略 / 跳过展示
- ❌ 不得"我觉得用户不在乎就不展示"

### 7.2 只调已实现的工具

- 讨论环节不让 AI 写新脚本
- 只用现有的 atomic CLI / lib
- ❌ 禁止 fork CLI 加功能（违反红线契约）

### 7.3 AI 不准自行判断"用户可能没注意"

- 任何 AI 觉得用户"应该发现"的问题
- 必须显式列在清单里
- 让用户主动决策，不是 AI 替用户做
