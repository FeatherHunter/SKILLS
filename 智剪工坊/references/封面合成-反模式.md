## 1️⃣ RGB 模式旋转填充黑色

**症状**: 旋转后图片角是纯黑色实色
**根因**: PIL `Image.rotate()` 在 RGB 模式下用 (0,0,0) 填充空白角
**修复**: 旋转前转 RGBA,临时大画布 (0,0,0,0) 透明填充

```python
# ❌ 错
im = Image.open(path).rotate(45)

# ✅ 对 (layers.rotate_hard 内部已封装)
im = im.convert("RGBA")
temp = Image.new("RGBA", (diag, diag), (0, 0, 0, 0))
temp.paste(rgba, ((diag - w) // 2, (diag - h) // 2), rgba)
rotated = temp.rotate(angle, resample=Image.BICUBIC, expand=False)
```

**验证**: `diagnose cover.jpg`,看 dark_areas.pct 应 < 60%。

## 2️⃣ alpha 羽化导致半透明黑

**症状**: 旋转后边缘有黑色半透明残影
**根因**: PIL BICUBIC 抗锯齿产生 0-255 中间值 alpha
**修复**: 旋转后 `binarize_alpha()` 二值化

```python
# ❌ 错
rotated = temp.rotate(angle)  # 抗锯齿 alpha ∈ [0, 255]
canvas.paste(rotated, ...)    # 中间值 alpha 跟画布混合 → 半透明黑

# ✅ 对
a_arr = np.array(rotated.split()[3])
a_bin = (a_arr > 128).astype(np.uint8) * 255
binarized = Image.merge("RGBA", (r, g, b, Image.fromarray(a_bin, "L")))
```

**验证**: `diagnose cover.jpg --check transparency`,看 semi_transparent.pct 应 < 1%。

---

## 3️⃣ JPG 不支持透明

**症状**: `.jpg` 输出后黑色背景,期望透明背景的图片"丢失"
**根因**: JPG 标准不支持 alpha 通道
**修复**: 输出 `.jpg` 时 `canvas.convert("RGB")`,丢掉 alpha

```python
# ❌ 错
canvas.save("cover.jpg")  # RGBA → JPG 会自动转 RGB,但可能报错

# ✅ 对 (canvas.safe_save 已封装)
if ext in (".jpg", ".jpeg"):
    if canvas.mode == "RGBA":
        canvas = canvas.convert("RGB")
    canvas.save(p, quality=92)
```

**何时保留透明**: 输出用 `.png` 格式(`.png` 支持 RGBA)。

---

## 4️⃣ PIL paste 没传 mask

**症状**: 粘贴 RGBA 图层后,看到的是黑底而非透明
**根因**: `canvas.paste(img, (x, y))` 没传 mask,默认用整个图层(包括 alpha=0 区)
**修复**: paste 第 3 参数传 img 自己(自动用 alpha 作为 mask)

```python
# ❌ 错
canvas.paste(img, (x, y))  # alpha=0 的像素覆盖到画布,看到黑/旧色

# ✅ 对 (layers.place 已封装)
canvas.paste(img, (x, y), img)  # img 自己是 RGBA,自动用 alpha mask
```

**本质**: paste 的 mask 参数告诉 PIL "哪些像素要贴"。

## 5️⃣ 旋转后角是黑色实色

**症状**: 旋转后图片的 4 个角是黑色矩形(不是透明)
**根因**: `Image.rotate(expand=True)` 把图放大,4 角是 RGB (0,0,0)
**修复**: 用**临时大画布** + bbox 裁切(rotate_hard 7 步流程)

```python
# ❌ 错
rotated = img.rotate(angle, expand=True)  # expand=True 会让角是黑

# ✅ 对
diag = int(sqrt(w**2 + h**2)) + 20
temp = Image.new("RGBA", (diag, diag), (0, 0, 0, 0))  # 临时画布透明
temp.paste(rgba, ((diag-w)//2, (diag-h)//2), rgba)
rotated = temp.rotate(angle, expand=False)
bbox = rotated.getbbox()  # 找到非透明 bbox
cropped = rotated.crop(bbox)  # 裁掉透明角
```

**核心**: "bbox 裁切"是消除旋转后填充色的关键。

## 6️⃣ 文字无描边

**症状**: 亮色照片上的浅色文字读不清
**根因**: 文字颜色和背景色对比度不足
**修复**: 描边宽度 ≥ 4,描边色用黑色

```python
# ❌ 错
draw.text((x, y), "标题", font=font, fill=(255, 215, 0))  # 亮黄无描边,亮背景上糊

# ✅ 对
draw.text((x, y), "标题", font=font, fill=(255, 215, 0),
          stroke_width=4, stroke_fill=(0, 0, 0))
```

**自动描边宽度**: `text.draw_text` 默认按字号比例 `font_size // 25`,确保大字号也有粗描边。

## 7️⃣ 字号过大超出画布

**症状**: 长标题(比如 "30 天减肥挑战")字号 200,文字宽度超过画布
**根因**: 字号没适配画布宽度
**修复**: `text.fit_text_size(content, max_w, max_h, start=200)` 二分查找

```python
# ❌ 错
font_size = 200  # 不管多长都用 200

# ✅ 对
size = 200
while size > 20:
    font = ImageFont.truetype(font_path, size)
    bbox = font.getbbox(content)
    if bbox[2] - bbox[0] <= max_width and bbox[3] - bbox[1] <= max_height:
        return size
    size -= 5
```

**默认最小字号**: 20(再小读不清)。

## 8️⃣ 重复缩小主图

**症状**: symmetric-cascade 布局下,主图(第 1 张)反而比左右副图小
**根因**: 主图按 canvas 0.65 计算,但左右副图按 canvas 0.5 计算,逻辑看似对,但当照片比例不一致时,主图实际渲染尺寸反而小
**修复**: 主图固定为 `0.65 * min(W, H)`,副图固定为 `0.5 * min(W, H)`,**主图绝对不缩小**

```python
# ❌ 错
main_size = canvas_w * 0.65  # 横屏 16:9 时,canvas_w >> canvas_h,主图过大
left_size = canvas_h * 0.5    # 但 canvas_h 小,副图小

# ✅ 对
main_size = min(canvas_w, canvas_h) * 0.65  # 以短边为基准
left_size = min(canvas_w, canvas_h) * 0.5
```

## 9️⃣ 多图层 z-order 错乱

**症状**: symmetric-cascade 下,主图反而盖住左右副图
**根因**: `LayerSpec.z` 没严格按"主图在底,副图在上"递增
**修复**: z 强制约定:
- 主图: z=1
- 左右副图: z=2
- 第 4+ 张: z=3+i(从 4 开始递增)

```python
# ❌ 错
layers = [
    LayerSpec(path=main, z=0),   # 主图在最下... 但 z=0 可能被 grid 当作"无 z"
    LayerSpec(path=left, z=1),
    LayerSpec(path=right, z=1),
]

# ✅ 对
layers = [
    LayerSpec(path=main, z=1),  # 主图明确在 z=1
    LayerSpec(path=left, z=2),  # 副图 z=2 在主图之上
    LayerSpec(path=right, z=2),
]
# pipeline.compose() 用 sorted(layers, key=lambda l: l.z) 保证顺序
```

## 🔟 颜色 hex 解析错误

**症状**: `bg="#FFF"` 报错或输出异常
**根因**: PIL/Python 标准库不直接接受 `#FFF`,需要先解析
**修复**: `canvas.hex_to_rgb(hex)` 统一接受 `#FFF` / `#FFFFFF` / `FFF` / `FFFFFF`

```python
# ❌ 错
Image.new("RGB", (W, H), "#FFF")  # PIL 不接受带 # 的字符串

# ✅ 对
def hex_to_rgb(hex_color: str):
    s = hex_color.lstrip("#")
    if len(s) == 3:
        s = "".join(c * 2 for c in s)
    return (int(s[0:2], 16), int(s[2:4], 16), int(s[4:6], 16))
```

## 1️⃣1️⃣ 画布尺寸硬编码

**症状**: 改 aspect 比例要改 5 个地方
**根因**: 1920x1080 散落在 layout 计算里
**修复**: `pipeline.ASPECT_RATIOS` 字典单一真理

```python
# ❌ 错
if aspect == "16:9":
    W, H = 1920, 1080
elif aspect == "9:16":
    W, H = 1080, 1920
# ... 散落

# ✅ 对
ASPECT_RATIOS = {
    "16:9": (1920, 1080),
    "9:16": (1080, 1920),
    "4:3":  (1440, 1080),
    "3:4":  (1080, 1440),
    "1:1":  (1080, 1080),
    "4:5":  (1080, 1350),
    "5:4":  (1350, 1080),
}
W, H = ASPECT_RATIOS[aspect]
```

## 1️⃣2️⃣ 字号固定不缩放

**症状**: `text.size=64` 在大画布上偏小,在小画布上偏大
**根因**: 字号是绝对值,没按画布尺寸缩放
**修复**: text_layer / draw_text 的 size 是参数,但用户用 fit_text_size 自动算

```python
# ❌ 错
text_layer(canvas, "标题", size=64)  # 在 1920x1080 偏小,在 720x480 偏大

# ✅ 对
size = fit_text_size("标题", max_width=W - 160, max_height=H // 3, start=200)
text_layer(canvas, "标题", size=size)
```

**何时用绝对字号**: 标题固定(比如 "系列名")+ 副标题适配(主标题用 200 起步,副标题用 80 起步)。

## 1️⃣3️⃣ 旋转后角是 fill 色

**症状**: 旋转后角不是黑色,而是**底色**(比如底色是黄色,角是黄色)
**根因**: 临时画布用了 RGB 模式或非 (0,0,0,0) 填充
**修复**: 临时画布必须 RGBA + (0,0,0,0) 透明

```python
# ❌ 错
temp = Image.new("RGB", (diag, diag), (255, 255, 255))  # 白色填充 → 角是白
temp = Image.new("RGBA", (diag, diag), (0, 0, 0, 255))  # 不透明黑填充 → 角是黑

# ✅ 对
temp = Image.new("RGBA", (diag, diag), (0, 0, 0, 0))  # 完全透明填充
```

**本质**: rotate 不管原图,会把 rotate 范围外的像素设成"画布原色",所以画布原色必须透明。

## 1️⃣4️⃣ 文字位置硬编码

**症状**: 改画布尺寸,文字位置错位
**根因**: 文字位置是绝对坐标 (x=100, y=100)
**修复**: `text.POSITIONS_9_GRID` 用 lambda 按画布尺寸算

```python
# ❌ 错
x, y = 100, 100  # 在 1920x1080 是左上角,在 720x480 接近中心

# ✅ 对
positions = {
    "top-center": lambda tw, th, cw, ch, margin: ((cw - tw) // 2, margin),
    "middle-center": lambda tw, th, cw, ch, margin: ((cw - tw) // 2, (ch - th) // 2),
    "bottom-center": lambda tw, th, cw, ch, margin: ((cw - tw) // 2, ch - th - margin),
    # ... 9 宫格
}
x, y = positions["top-center"](text_w, text_h, W, H, margin=80)
```

**margin**: 文字距画布边缘的最小距离(默认 80 像素)。

## 1️⃣5️⃣ shadow 扩散全图变暗

**症状**: 整张封面偏暗,主角颜色被压
**根因**: layers.place 用了 `ImageFilter.GaussianBlur` shadow 扩散
**修复**: **place 不用 shadow**,保持硬贴

```python
# ❌ 错
shadow = img.filter(ImageFilter.GaussianBlur(15))
canvas.paste(shadow, (x+10, y+10))  # shadow 叠加,全图变暗
canvas.paste(img, (x, y), img)

# ✅ 对 (layers.place 实现)
def place(canvas, img, x, y):
    if canvas.mode != "RGBA": canvas = canvas.convert("RGBA")
    if img.mode != "RGBA": img = img.convert("RGBA")
    canvas.paste(img, (x, y), img)  # 纯硬贴,无 shadow
    return canvas
```

**设计取舍**: 牺牲"立体感"换"清晰度",符合 B 站/抖音封面的扁平风格。

## 1️⃣6️⃣ 0 字节 cover 输出

**症状**: `cover.jpg` 是 0 字节或不存在
**根因**: 校验失败后 silent fail,或 save 路径无目录
**修复**: `validators.validate_all()` 早失败 + `canvas.safe_save()` 自动建目录

```python
# ❌ 错
result = compose(photos=[], output="out.jpg")  # 不校验,silent fail
canvas.save("out.jpg")  # out 目录不存在 → FileNotFoundError

# ✅ 对
ok, errors = validate_all({"photos": [], ...})
if not ok:
    return {"status": "error", "errors": errors, ...}

# canvas.safe_save 自动建目录
p = Path(output_path)
p.parent.mkdir(parents=True, exist_ok=True)  # 关键
canvas.save(p, ...)
```

**校验清单**(在 validators.py):
1. photos 至少 1 张
2. 每张照片路径存在且可读
3. output 路径的目录可写
4. aspect / layout 在白名单
5. text JSON 解析成功
6. bg 颜色合法

## 验收清单(出片前自查)

```
□ 1. diagnose cover.jpg --check transparency
   → semi_transparent.pct < 1% ✅
□ 2. diagnose cover.jpg --check darkness
   → dark_areas.pct < 60% ✅
□ 3. diagnose cover.jpg --check symmetry
   → diff_pct < 8% ✅(对称布局时)
□ 4. 文件大小 > 30KB(说明确实有内容)
□ 5. 打开 cover.jpg 肉眼检查:
   □ 主图清晰,主角不糊
   □ 文字读得清(有描边)
   □ 文字在画布内(没溢出)
   □ 布局合理(主图 + 副图层级对)
   □ 没有"莫名黑角"或"半透明黑带"
```
