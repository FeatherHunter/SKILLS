## 关键 bug 故事(每个对应 1 个反模式)

### Bug #1 · 旋转后角是黑色实色 (反模式 #5)

**v15 现象**: 旋转 14° 后,图的 4 个角是**正方形黑块**

**根因**: PIL `Image.rotate(expand=True)` 用 (0,0,0) 填充扩展区

**修复 (rotate_hard 7 步)**:
```python
# Step 1: 转 RGBA
rgba = im.convert("RGBA")
# Step 2: 临时大画布(对角线 + 20),完全透明
diag = int(sqrt(w**2 + h**2)) + 20
temp = Image.new("RGBA", (diag, diag), (0, 0, 0, 0))
# Step 3: paste 原图到中心
temp.paste(rgba, ((diag-w)//2, (diag-h)//2), rgba)
# Step 4: 旋转(中心旋转,不 expand)
rotated = temp.rotate(angle, resample=Image.BICUBIC, expand=False)
# Step 5: bbox 裁切
bbox = rotated.getbbox()
cropped = rotated.crop(bbox) if bbox else rotated
# Step 6: alpha 二值化
return binarize_alpha(cropped)
```

**验证**: 旋转 14° 后,4 角是 alpha=0 透明(诊断半透明像素=0%)。

---

### Bug #2 · 边缘半透明黑带 (反模式 #2)

**v15 现象**: 旋转后图边缘有一圈"灰黑过渡带"

**根因**: PIL BICUBIC 抗锯齿产生 alpha ∈ [0, 255] 中间值,中间值跟画布混合 → 半透明黑

**修复 (binarize_alpha)**:
```python
def binarize_alpha(rgba):
    a_arr = np.array(rgba.split()[3])
    a_bin = (a_arr > 128).astype(np.uint8) * 255
    return Image.merge("RGBA", (r, g, b, Image.fromarray(a_bin, "L")))
```

**关键细节**:
- 阈值 128 是经验值,< 50 会丢失细边,> 200 会保留过多中间值
- 整个 alpha 通道二值化,不是局部处理

**验证**: `diagnose cover.jpg --check transparency`,semi_transparent.pct = 0.0% ✅

---

### Bug #3 · 主图被副图压住 (反模式 #8)

**v15 现象**: symmetric-cascade 下,主图(第 1 张)反而比左右副图小

**根因**: 主图按 canvas 宽度 0.65 算,但 16:9 时 canvas_w >> canvas_h,实际主图被压扁

**修复 (layout 主图固定短边)**:
```python
# ❌ 错
main_size = canvas_w * 0.65
# ✅ 对
main_size = min(canvas_w, canvas_h) * 0.65
```

**本质**: 9:16 画布 canvas_w=1080 canvas_h=1920,主图按 min(W,H)=1080 算 = 702x702,左右副图按 1080*0.5=540 算,主图 702 > 副图 540,层级对了。

---

### Bug #4 · 文字溢出画布 (反模式 #7)

**v15 现象**: 长标题"30 天减肥挑战"字号 200,文字宽度 2500 像素 > 画布 1920

**根因**: 字号固定不缩放

**修复 (fit_text_size)**:
```python
def fit_text_size(content, max_width, max_height, start=200, min=20):
    size = start
    while size > min:
        font = ImageFont.truetype(font_path, size)
        bbox = font.getbbox(content)
        if (bbox[2] - bbox[0]) <= max_width and (bbox[3] - bbox[1]) <= max_height:
            return size
        size -= 5
    return min
```

**关键细节**:
- 步长 5(精细,但不至于漏)
- 最小 20(再小读不清)
- 同时检查宽和高(光看宽会高度溢出)

**验收**: 任何长标题都能在指定区域内,字号自动缩。

## 调试技巧(下次遇到类似问题)

1. **半透明黑**: 第一时间检查 `rotate_hard` 是否被调用,alpha 是否二值化
2. **主图变小**: 第一时间检查 layout 主图按 `min(W, H)` 还是 `canvas_w`
3. **文字溢出**: 第一时间检查 `fit_text_size` 是否调用,start 字号是否过大
4. **JPG 黑底**: 第一时间检查 `safe_save` 是否转 RGB
5. **shadow 扩散**: 第一时间检查 `place` 是否加了 GaussianBlur
6. **z-order 错乱**: 第一时间检查 `LayerSpec.z` 是否按"主图最小,副图递增"
