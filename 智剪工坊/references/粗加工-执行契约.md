# 粗加工-执行契约（Step 8-9 详细）

> **何时加载**：AI 进入 Step 8（粗加工）时必读
> **目的**：定义粗加工 5 步流程的详细契约（输入/输出/跳过/异常/强制）
> **范围**：主流程 Step 8-9，对应 SKILL.md 老版本"阶段 2"

---

## 1. 概述

**输入**：
- `intent.json`（用户填写）
- 已确认的 6 象限操作清单（Step 7 产物）
- 源素材（`video_*.mp4` / `audio_*.mp3` / `image_*.jpg`）

**输出**：
- `00_智剪/粗加工/` 下 5 类产物
- 产物是 Step 11 精加工的输入

**约束**：
- AI 是编排者，按本文件 5 步顺序执行
- 调原子 CLI（`scripts/{audio,asr,video,ai,batch}/*.py`）
- ❌ 禁止调 `scripts/_internal/`（AI 不可调内部工具）
- ❌ 禁止自写 ffmpeg 命令（违反红线）

---

## 2. 5 步流程

### Step 9.1 解析 + 自检

**输入**：`intent.json` + 已确认操作清单
**输出**：`00_智剪/粗加工/中间产物/自检报告.json`
**跳过**：无（必须）

**行为**：
1. 解析 JSON（用 `lib/cli_args.py` 辅助）
2. 检查源文件存在性、必填字段、`exclude` 列表
3. 写状态到 `自检报告.json`
4. 异常不退出整体（仅记录）

**日志要求（v1.12 强制）**：
- 写 `logs/<task_id>.jsonl`：stage=粗加工, step=9.1, action=parse_intent
- 异常字段：缺失字段列表（即使整体不退出也要写）
- schema：`{"time":"...","stage":"粗加工","step":"9.1","action":"parse_intent","decision":"<意图路径>","thinking":"<AI 推断>","result":"<ok/error>","error":null|"<error msg>"}`

**自检报告 schema**：

```json
{
  "intent_path": "C:\\...\\intent.json",
  "video_count": 3,
  "missing_videos": [],
  "missing_required_fields": [],
  "excluded_videos": [],
  "ready_for_step_9_2": true
}
```

### Step 9.2 ASR（语音转文字）

**输入**：操作清单 A 象限中 `voice != mute / bgm-only` 的视频
**处理**：`scripts/asr/transcribe.py` 调 `lib/asr/whisper.py`（faster-whisper GPU）
**输出**：
- `00_智剪/粗加工/文字稿/视频_{idx}.md`（每个视频一份）
- `00_智剪/粗加工/文字稿/全部.md`（汇总）

**跳过**：`voice == "mute"` 或 `"bgm-only"` 的视频
**强制**：每完成一个 ASR → 立即向用户汇报

**日志要求（v1.12 强制）**：
- 每个 ASR 完成写一行：stage=粗加工, step=9.2, action=asr, decision="<model> for <video_name>"
- schema：`{"time":"...","stage":"粗加工","step":"9.2","action":"asr","decision":"whisper-large-v3 for video_3","thinking":"用户意图需要文字稿","result":"exit 0","error":null}`

**目的**：让 Step 9.3 处理视频时 AI 已有文字稿可优化 ops（D2 片头叠加文字、D5 水词判定）。

### Step 9.3 单视频处理（基于 ASR 优化）

**输入**：源 mp4 + ops（来自 intent）+ 文字稿（Step 9.2 产出）
**处理**：`lib/video_processing.py process_video()` 自动处理所有 ops + 自动 `video_normalize`
**输出**：
- `00_智剪/粗加工/单视频/video_{idx}.mp4`
- `00_智剪/粗加工/单视频/profile_{idx}.json`

**应用 ops 顺序**：
```
counter-rotate → 用户 op → scale/pad (aspect-fill 或 aspect-fit)
```

**强制（v1.12 抗疲劳版）**：
- ❌ **禁止**：每处理完 1 个就立即向用户汇报（用户疲劳）
- ✅ **必须**：每 10 个汇报一次（批量）
- ✅ **必须**：写 `logs/<task_id>.jsonl` 每个 CLI 调用：
  ```json
  {"time":"2026-07-10T11:00:00","stage":"粗加工","step":"9.3","action":"trim","decision":"trim 0-5s for video_3","thinking":"用户意图说要剪开头","result":"exit 0","error":null}
  ```
- ✅ **必须**：卡死 / 超时立即向用户汇报（这是例外）

**profile_{idx}.json schema**：

```json
{
  "index": 3,
  "source_file": "video_xxx.mp4",
  "source_resolution": "1080x1920",
  "has_rotation_metadata": true,
  "rotation_applied": -90,
  "applied_ops": ["cut-middle"],
  "output_resolution": "1920x1080",
  "output_duration": 6.4,
  "voice_mode": "keep",
  "output_path": "D:\\\\...\\\\粗加工\\\\单视频\\\\video_03.mp4",
  "error": null
}
```

**异常处理**（分两种，不退出整体）：
- 命令失败：ffmpeg return code ≠ 0 → 写入 `profile.error`，继续下一个
- 产物无效：ffmpeg 成功但输出 < 1KB → 写入 `profile.error`，继续下一个

**音频同步关键点**：
- trim/cut/pin-range 后视频流 `setpts=PTS-STARTPTS` 归零
- 但音频流原始 PTS 范围未归零，**必须同步 trim 音频范围 + 重置 PTS**
- 使用 `atrim + asetpts=PTS-STARTPTS`

**mute 视频特殊处理**：
- 源视频无音频轨时 `[0:a]` 不存在
- 用 `anullsrc` 生成等长空白音轨：`anullsrc=r=44100:cl=stereo[a];[a]apad=whole_dur=${输出时长}`
- 否则 filter graph 引用失败

**rotation 处理**：
- 0° → 空 filter
- 90° → `transpose=1`
- -90° / 270° → `transpose=2`
- ±180° → `transpose=1,transpose=1`
- 旋转后**必须**清 metadata（用 `lib/video/patch_mp4_rotation.py` 二进制 patch）

### Step 9.4 sequence 拼接（仅 sequence 内视频）

**输入**：`单视频/` 里的成品 + `intent.json.sequences`
**输出**：
- `00_智剪/粗加工/组合/seq_{name}.mp4`（每个 sequence 一段）
- `00_智剪/粗加工/中间产物/拼接日志.log`
- `00_智剪/粗加工/中间产物/自由素材清单.md`（不在任何 sequence 内的视频）

**跳过**：不在任何 sequence 内的视频
**行为**：每个 sequence 按内部顺序 xfade 链拼；跨 sequence 最后再拼一次
**异常处理**：ffmpeg 失败 → 写入拼接日志，不退出整体
**强制**：跨 sequence 最后再拼一次（一次性拼接全片）

**日志要求（v1.12 强制）**：
- 每个 sequence 完成写一行：stage=粗加工, step=9.4, action=xfade_chain, decision="<seq_name> 用 <xfade_type>"

### Step 9.5 整体复核 + 模糊项兜底

**输入**：操作清单 D 象限 + Step 9.1-9.4 实际产物
**输出**：
- `00_智剪/粗加工/中间产物/模糊项处理记录.md`
- (可能的) `单视频/video_xx.mp4` 更新
- (可能的) `文字稿/video_xx.md` 更新

**行为**：
1. 重新审视所有"模糊项"（意图对齐阶段没明确 + Step 9.1-9.4 没处理的）
2. 跟用户**逐条澄清** → 应用原子操作 → 更新产物
3. 每处理完一个模糊项 → 写入处理记录

**强制**：
- 模糊项**不闷头猜**：D 象限中标记"必须问"的项目必须等用户回答
- 已"建议问"的项目，AI 可提供默认假设 + 让用户否决

**日志要求（v1.12 强制）**：
- 每个模糊项处理写一行：stage=粗加工, step=9.5, action=clarify_with_user, decision="<用户回答>"

### Step 9.6 决策报告

**输入**：全量产物
**输出**：`00_智剪/粗加工/决策.md`
**行为**：写决策报告 + 进入 Step 10 审查

**决策.md 必须包含**：
- intent.json 整体要求摘要
- 异常情况报告（哪些源是竖屏、哪些有 rotation、哪些 op 可能出问题）
- 用户在粗加工过程中提的额外要求
- 模板加载建议（基于操作清单 + 关键帧 + ASR 文字稿）
- 精加工路径建议（基于用户偏好的路径 A 纯语言 / B 模板）

---

## 3. AI 编排职责

```
1. 加载 intent.json → 解析
2. 按 9.1 → 9.6 顺序执行（不可跳步）
3. 每步完成后立即向用户汇报
4. 异常不退出整体，仅记录到 profile.error
5. 全部完成后 → Step 10 触发（粗加工审查）
```
