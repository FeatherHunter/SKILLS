# 精加工-两路径（Step 11 详细）

> **何时加载**：AI 进入 Step 11（精加工）时必读
> **目的**：定义精加工路径 A（纯语言）+ 路径 B（模板）的详细契约
> **范围**：主流程 Step 11，对应 SKILL.md 老版本"阶段 3 模板 + 阶段 5 出片前"

---

## 1. 概述

**输入**：
- `00_智剪/粗加工/`（Step 9 产物）
- `决策.md`（Step 9.6 产物）
- 已确认的精加工路径（A 或 B）

**输出**：
- `00_智剪/精加工/终稿/精加工_final.mp4`（用户选定的精加工版本）
- `00_智剪/精加工/审查记录.md`（Step 12 审查日志）

**关键约束**：
- **用户必选路径**（A 或 B），AI 不可替选
- **默认输出 1 个版本**（避免选择疲劳）
- **用户可追问**"再来一版"或"加入 X 思想后再来一版" → AI 出 v2/v3
- ❌ 禁止 AI 一次出 5 个版本让用户挑

## 2. 路径 A：纯语言（AI 创意编排）

### 2.1 流程

```
A.1 AI 按"专业剪辑师问题清单"问 5 个核心问题
A.2 用户回答 + AI 复述理解 + 用户确认
A.3 AI 设计编排方案（自由发挥）
A.4 AI 出"精加工草稿"v1（默认 1 个版本）
A.5 用户可追问"再来一版"或"加入 X 思想后再来一版"
A.6 用户确认 OK → Step 12 审查
```

### 2.2 专业剪辑师问题清单（AI 必问）

**核心 5 问**（必问）：

```
1. 【目标】
   "这个视频的目标观众是谁？
    选项示例：自己记录 / 朋友分享 / B站 / 抖音 / 商业投放"

2. 【情感基调】
   "你想传达的核心情绪是什么？
    选项示例：热血 / 治愈 / 搞笑 / 安静记录 / 震撼 / 紧张 / 怀旧"

3. 【节奏偏好】
   "你希望视频的节奏感？
    选项示例：
      - 快剪密集（每 1-2 秒切换，能量爆棚）
      - 适中（每 3-5 秒切换，舒适观看）
      - 慢节奏（每 5+ 秒切换，沉静氛围）
      - 张弛有度（开头慢 / 中间快 / 结尾慢）"

4. 【重点素材】
   "哪段素材最重要，想让观众最难忘？
    （AI 列单视频/组合让用户选 1-3 个核心片段）"

5. 【结尾期待】
   "你希望观众看完后记住什么？
    选项示例：一个金句 / 一个画面 / 一个动作 / 一个情绪 / 一段BGM"
```

**补充 3 问**（AI 判断是否问）：

```
6. 【视觉参考】
   "有没有参考视频或风格倾向？（如 B站某 UP主 / 抖音某账号）"
   → 仅在 AI 检测到意图不清晰或用户犹豫时问

7. 【情绪曲线】
   "想引导观众什么情绪变化？
    示例：困惑 → 惊喜 → 满足 / 平静 → 紧张 → 释放"
   → 仅在用户提到"故事性"或"起伏"时问

8. 【特殊要求】
   "有没有具体的转场/特效/BGM 卡点要求？"
   → 仅在用户主动提到具体技术细节时追问
```

### 2.3 AI 行为约束

- ✅ **必须**：核心 5 问全部问完才能进 A.3
- ✅ **必须**：用户回答后 AI 复述理解 + 用户确认
- ⚠️ **可选**：补充 3 问 AI 判断后选择性问
- ❌ **禁止**：一次问 10 个问题（用户疲劳）
- ❌ **禁止**：AI 替用户回答"大概是这样吧"
- ❌ **禁止**：跳过问题直接出 v1（"反正用户没耐心"是 AI 揣测）

### 2.4 v1 出片后

```
AI 输出 v1 路径：
  00_智剪/精加工/纯语言编排/AI编排_<timestamp>_v1.mp4

AI 报告：
  "精加工 v1 已生成。路径：...
   - 你可以追问'再来一版'或'加入 X 思想后再来一版'
   - 满意就确认进 Step 12 审查"
```

### 2.5 "再来一版" 协议

用户可说：
- "再来一版" → AI 基于已有回答出 v2
- "加入 X 思想后再来一版" → AI 必须先理解 X，再出 v2（不能跳过 X 直接出）

AI 出 v2 时**必须**说明：
- 与 v1 的差异（哪些 op 改了、为什么）
- 输出路径：`精加工/纯语言编排/AI编排_<timestamp>_v2.mp4`
- 之前的 v1/v2 保留（用户可回看）

## 3. 路径 B：模板（YAML 驱动）

### 3.1 流程

```
B.1 AI 列出可用模板（当前：模板/健身vlog.yaml 等）
B.2 用户选模板
B.3 AI 加载 YAML → 按 yaml stages 自动编排
B.4 AI 出"精加工草稿"v1
B.5 用户可要求"换模板"或"改 YAML 参数"
B.6 用户确认 OK → Step 12 审查
```

### 3.2 可用模板（v1.10 起）

| 模板 | 适用场景 | stages 数 |
|---|---|---|
| `模板/健身vlog.yaml` | 健身/训练视频 | 4 |
| （更多待开发）| — | — |

### 3.3 YAML 模板契约

```yaml
# 模板/<name>.yaml
stages:
  - id: <stage_id>
    name: <stage 名称>
    description: <该 stage 是干什么>
    ai_prompt: <AI 引导用户的话，可用 {video_count} {suggested_order} 等占位>
    user_input: <用户怎么回答>
    ai_action: <AI 调哪些原子 CLI，顺序如何>
    output: <该 stage 的产物路径>
completion:
  trigger: <所有 stage 完成触发条件>
  ai_action: <AI 调 atomic CLI 拼成片>
  output: 00_智剪/精加工/终稿/<sanitized_title>.mp4
failure:
  user_signal: <用户 skip / cancel 怎么算>
  ai_action: <跳过该 stage 怎么记录>
```

### 3.4 AI 编排职责

```
1. 加载 模板/<name>.yaml
2. 逐 stage 执行：给用户看 ai_prompt → 等用户回答 → 调 ai_action 里的 CLI
3. 拿 stage 产物作为下一个 stage 的输入
4. 全部 stage 完成后跑 completion.ai_action → 出精加工草稿
5. 询问用户是否满意（进 Step 12）或调整
```

### 3.5 AI 行为约束

- ✅ **必须**：完整执行所有 yaml stages（不可跳）
- ✅ **必须**：每个 stage 完成后向用户汇报
- ❌ **禁止**：跳过 yaml stage 直接出成片（"看着差不多"）
- ❌ **禁止**：自创不在 yaml 里的 op（必须先改 yaml）

### 3.6 "换模板" 或 "改 YAML 参数" 协议

```
用户：
  - "换 X 模板" → AI 重新加载 X.yaml，重新跑
  - "改 stage 3 的 fade 时长 0.5s → 1s" → AI 修改 in-memory yaml 后重跑该 stage

AI 必须说明：
  - 改了哪几个参数
  - 输出路径
```

## 4. 路径 A vs 路径 B 对比

| 维度 | 路径 A 纯语言 | 路径 B 模板 |
|---|---|---|
| **AI 角色** | 创意编排师 | 模板应用者 |
| **用户参与** | 答 5+ 问题（多） | 选模板 + 看 stages（少）|
| **YAML** | ❌ 不涉及 | ✅ 核心 |
| **适用用户** | 有想法但不会技术 | 没想法 + 想"超过水平"效果 |
| **炫酷程度** | 高（AI 自由发挥）| 中（预设参数）|
| **可重复性** | 低（每次不同）| 高（同模板同结果）|
