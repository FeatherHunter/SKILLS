# 智剪工坊 — 架构设计（v1.0）

## 1. 一句话目标

用户写一个 intent.json，AI 按 **阶段 0-4 端到端流程**，把一堆原始视频拼成一段可用的 vlog。**阶段 1 必须输出「操作清单」**作为阶段 2 的执行契约。

## 2. 工具端（智剪工坊/）

```
智剪工坊/
├── SKILL.md           ← ⚠️ AI 必读 + 完整流程 + 操作清单 schema
├── 架构.md            ← 本文件（设计决策 + 决策落地位置）
├── HANDOFF.md         ← Session 交接
├── CHANGELOG.md       ← 版本历史
├── 模板/              ← 类别化工作流
│   ├── 健身vlog.yaml
│   ├── 教程vlog.yaml
│   └── ...
├── 脚本/              ← 30 个原子
├── lib/
│   ├── common.py      ← 公共（ffmpeg + 错误 + 日志）
│   ├── asr.py         ← ASR 包装（whisper）— v1.0 用于 Step 2
│   └── modify.py      ← 改素材操作菜单
├── 文档/              ← references/
├── intent.html
└── executor.py        ← 粗加工执行器（5 个原子函数）
```

**约定**：
- `模板/` 是**工作流脚本**（AI 引导用户做决策的步骤），不是配置
- `脚本/` 是原子工具
- `executor.py` 处理"粗加工"阶段的可机械化部分
- `SKILL.md` 是**任何 AI 必读**（含 ⚠️ AI 必读章节）

## 3. 工作区（<workspace>/）

```
<workspace>/
├── video_*.mp4                ← 源视频（用户已放，AI 不动）
├── intent.json                ← 唯一跟源混居的 AI 文件
├── intent_v1.json             ← 版本快照
└── 00_智剪/                   ← AI 自管区（00_ 排序第一）
    ├── 粗加工/
    │   ├── 单视频/            ← 每个视频处理后的标准片段
    │   ├── 组合/              ← sequence + 转场拼好的组
    │   ├── 文字稿/            ← ASR 结果（v1.0 在 Step 2 输出）
    │   ├── 中间产物/
    │   │   ├── 自检报告.json          ← Step 1
    │   │   ├── 单视频汇总.md          ← Step 2 末尾
    │   │   ├── 拼接日志.log           ← Step 3
    │   │   ├── 自由素材清单.md        ← Step 3 新增（未约束视频）
    │   │   ├── 操作清单.md            ← 阶段 1 产物（执行契约）
    │   │   └── 模糊项处理记录.md      ← Step 4
    │   └── 决策.md            ← Step 5 + 模板衔接
    └── 成片/
        └── vlog_final.mp4     ← 阶段 4 收尾输出
```

**约定**：
- 源视频**不挪动**
- `00_智剪/` 是 AI 命名空间
- `intent.json` 跟源混居
- `操作清单.md`（阶段 1 产物）是阶段 2 执行契约，必须先确认再动

## 4. 完整流程（v1.0：阶段 0-4）

```
阶段 0 ▸ 项目初始化
├─ 0.1  AI 提示用户用 intent.html 填表
├─ 0.2  用户填表 → intent.json
├─ 0.3  用户把 intent.json 给 AI
└─ 0.4  [可选] intent.json 缺字段 → AI 触发交互式采访补全

阶段 1 ▸ 意图对齐（必走，输出操作清单）
├─ 1.1  AI 解析 intent.json → 生成「操作清单.md」（草稿）
├─ 1.2  AI 向用户呈现清单（六象限）
├─ 1.3  用户逐条确认/修正/补充
└─ 1.4  AI 写「操作清单.md (已确认)」+「自检报告.json」
       → 阶段 1 产物是阶段 2 的执行契约

阶段 2 ▸ 粗加工（5 步，详见 §5）
├─ Step 1: 解析 + 自检
├─ Step 2: 单视频处理 + ASR（ASR 前置）
├─ Step 3: sequence 拼接（仅 sequence 内视频）
├─ Step 4: 整体复核 + 模糊项兜底
└─ Step 5: 决策报告 + 模板衔接

阶段 3 ▸ 模板工作流（按 模板/<name>.yaml）

阶段 4 ▸ 收尾成片（烧字幕 + BGM + 封面）
└─ 输出: 00_智剪/成片/vlog_final.mp4
```

## 5. 粗加工 5 步（v1.0）

**输入**：`intent.json` + 已确认操作清单
**输出**：`00_智剪/粗加工/` 下 5 类文件

| Step | 输入 | 处理 | 输出 | 行为约束 |
|---|---|---|---|---|
| 1 | intent.json | 解析 + 自检 | `中间产物/自检报告.json` | 异常不退出 |
| 2 | 操作清单 A 象限 + 源 mp4 | **v1.1 子步骤**：<br>2.1 ASR 优先（批量）<br>2.2 单视频处理（基于 ASR 优化） | 2.1: `文字稿/视频_{idx}.md`<br>2.2: `单视频/video_{idx}.mp4` + `profile_{idx}.json` | **每完成一步立即向用户汇报** |
| 3 | 操作清单 C 象限 + 单视频 | xfade 链按 sequence 顺序拼 | `组合/seq_<name>.mp4` + `自由素材清单.md` | 未约束视频**不处理** |
| 4 | 操作清单 D 象限 | 模糊项兜底（澄清 → 应用原子操作） | `模糊项处理记录.md` + (可能的) 单视频/文字稿更新 | **AI 不闷头猜** |
| 5 | 全量产物 | 写决策报告 + 推荐模板 | `决策.md` + 模板加载建议 | 衔接阶段 3 |

## 6. 模板工作流（健身vlog.yaml 示例）

**模板 = 工作流脚本**。每个 stage = AI 一来一回对话：

| Stage | AI 问 | 用户答 |
|---|---|---|
| 内容分类 | 自动分类结果 + 确认 | 确认/手动调整个别 |
| 节奏决策 | 节奏建议 | 确认/调整 |
| 顺序与转场 | 顺序与转场建议（**含自由素材**） | 确认/调整 |
| 收尾风格 | 收尾选项 | 选 A/B/C |

**约定**：
- 每步**直接改** `00_智剪/粗加工/` 里的素材
- 对话历史即决策记录
- v1.0 强制：模板 Stage 顺序阶段必须处理「自由素材」（Step 3 没拼的视频）

## 7. 约定（合约）

| # | 约定 |
|---|---|
| 1 | 粗加工是实质工作，每步生成文件 |
| 2 | 模板是工作流脚本，不是 config |
| 3 | 工作区结构：源视频 + `00_智剪/粗加工/` + `00_智剪/成片/` |
| 4 | 目录命名：粗加工中文，文件名/JSON 字段英文 |
| 5 | `intent.json` 跟源混居 |
| 6 | 不写 `decisions.json` / `state.json` / `review.html` |
| 7 | 模板命名：按类别（健身vlog/教程vlog/VLOG），不带人名 |
| 8 | HTML 文件名保持英文 |
| 9 | 不写 `schemas/` |
| 10 | 用户主导决策，每 stage 用户点头 |
| 11 | 粗加工失败不退出主体，列在决策.md；子流程修复后重跑 |
| 12 | **v1.0**：阶段 1 必走，输出「操作清单」作为阶段 2 执行契约 |
| 13 | **v1.0**：ASR 前置到 Step 2，与单视频处理同步产出 |
| 14 | **v1.0**：Step 4 是模糊项兜底环节，AI 不闷头猜 |
| 15 | **v1.0**：未约束视频不在 Step 3 处理，留给模板工作流 Stage 顺序阶段 |
| 16 | **v1.0**：Step 2 每处理完一个视频立即向用户汇报 |
| 17 | **v1.0**：不硬编码项目内容；所有流程描述用通用 schema |
| 18 | **v1.0**：AI 推断必须显式标记（操作清单 E 象限），不混淆用户明确项 |
