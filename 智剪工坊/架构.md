# 智剪工坊 — 架构设计

## 1. 一句话目标

用户写一个 intent.json，AI 按 `粗加工 → 模板加载 → 工作流 → 成片` 流程，把一堆原始视频拼成一段可用的 vlog。

## 2. 工具端（智剪工坊/）

```
智剪工坊/
├── SKILL.md
├── 架构.md
├── 模板/                ← 类别化工作流
│   ├── 健身vlog.yaml
│   ├── 教程vlog.yaml
│   └── ...
├── 脚本/                ← 30 个原子
├── lib/
├── 文档/                ← references/
├── intent.html
└── executor.py          ← 粗加工执行器
```

**约定**：
- `模板/` 是**工作流脚本**（AI 引导用户做决策的步骤），不是配置
- `脚本/` 是原子工具
- `executor.py` 处理"粗加工"阶段的可机械化部分

## 3. 工作区（<workspace>/）

```
<workspace>/
├── video_*.mp4                ← 源视频（用户已放，AI 不动）
├── intent.json                ← 唯一跟源混居的 AI 文件
├── intent_v1.json             ← 版本快照
└── 00_智剪/                   ← AI 自管区（00_ 排序第一）
    ├── 粗加工/
    │   ├── 单视频/            ← 每个视频处理后的标准片段
    │   ├── 组合/              ← sequence + 转场拼好的组
    │   ├── 文字稿/            ← ASR 结果
    │   ├── 中间产物/          ← log / profile
    │   └── 决策.md            ← 整体要求 + 用户新增
    └── 成片/
        └── vlog_final.mp4     ← 模板工作流深度加工后
```

**约定**：
- 源视频**不挪动**
- `00_智剪/` 是 AI 命名空间
- `intent.json` 跟源混居

## 4. 完整流程

```
1. 用户：intent.html 填表 → intent.json
2. 用户：把 intent.json 给 AI
3. AI：素材粗加工（5 步）
4. AI：扫描 决策.md + 视频 + 文字稿
   "你这是 [项目] [类型]，看到匹配 [模板] 模板，加载吗？"
5. AI：加载 模板/<name>.yaml
6. AI：按工作流引导（每 stage 一来一回）
7. AI：拼成 vlog_final.mp4
8. 用户：直接播放看
```

## 5. 粗加工 5 步

**输入**：`intent.json`
**输出**：`00_智剪/粗加工/` 下的 5 类文件

| Step | 输入 | 处理 | 输出 |
|---|---|---|---|
| 1 | intent.json | 解析 + 自检（源文件、必填字段、exclude 列表） | （无文件，仅状态） |
| 2 | 源 mp4 + ops | trim/cut-middle/speed/fade/调色 等 | `单视频/video_{idx}.mp4` + `profile_{idx}.json` |
| 3 | 单视频 + sequence | xfade 链按 sequence 顺序拼 | `组合/seq_{name}.mp4` |
| 4 | 单视频 mp4 | ASR | `文字稿/视频_{idx}.md` + `全部.md` |
| 5 | intent.json + 异常报告 | 写整体要求 + 异常 + 用户补充 | `决策.md` |

## 6. 模板工作流（健身vlog.yaml 示例）

**模板 = 工作流脚本**。每个 stage = AI 一来一回对话：

| Stage | AI 问 | 用户答 |
|---|---|---|
| 内容分类 | 自动分类结果 + 确认 | 确认/手动调整个别 |
| 节奏决策 | 节奏建议 | 确认/调整 |
| 顺序与转场 | 顺序与转场建议 | 确认/调整 |
| 收尾风格 | 收尾选项 | 选 A/B/C |

**约定**：
- 每步**直接改** `00_智剪/粗加工/` 里的素材
- 对话历史即决策记录

## 7. 约定（合约）

| # | 约定 |
|---|---|
| 1 | 粗加工是实质工作，每步生成文件 |
| 2 | 模板是工作流脚本，不是 config |
| 3 | 工作区结构：源视频 + `00_智剪/粗加工/` + `00_智剪/成片/` |
| 4 | 目录命名：粗加工中文，文件名/JSON 字段英文 |
| 5 | `intent.json` 跟源混居 |
| 6 | 不写 `decisions.json` / `state.json` / `review.html` |
| 7 | 模板命名：按类别（健身vlog/教程vlog/VLOG），不带人名 |
| 8 | HTML 文件名保持英文 |
| 9 | 不写 `schemas/` |
| 10 | 用户主导决策，每 stage 用户点头 |
