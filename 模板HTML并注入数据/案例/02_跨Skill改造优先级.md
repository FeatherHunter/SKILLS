# 案例 02 · 跨 Skill HTML 模板改造优先级

> 写于 2026-07-23 · 基于卡路里改造经验 · 给后续 50+ Skill 改造排序
> 配套：[01_卡路里Skill改造案例.md](./01_卡路里Skill改造案例.md)

## 一、第一性原理：什么 Skill 最值得改？

**判定标准（按重要性排序）**：

| 标准 | 说明 | 权重 |
|---|---|---|
| **复用频率** | 用户每周/每天调用次数 | ⭐⭐⭐⭐⭐ |
| **视觉密度** | 是否多字段 + 多条目 + 需要扫读 | ⭐⭐⭐⭐ |
| **改造阻力** | CLI 现状是否容易支持 JSON 化 | ⭐⭐⭐ |
| **业务价值** | 是否需要做决策/排行/对比 | ⭐⭐⭐ |

**核心结论**：**复用频率高 + 视觉密度高 + 改造阻力低** = 优先做

## 二、优先级分层

### 🔴 S 级 · 最优先（1-2 周内动手）

满足"复用频率高 + 视觉密度极高 + 改造阻力低"。

| Skill | 唤醒词数 | 改造理由 | 模板数 | 估算工作量 |
|---|---|---|---|---|
| **居家管家** | 30+ | 已有 8 个 HTML 模板但**风格不统一**（R9 DRY 已开始重构 `_shared.py`） | 8 现有 + 5 新增 | 3-5 天 |
| **备忘录** | 20+ | wish/打卡/回执数据丰富，最适合"清单 + 详情页"模式 | 3-5 | 2-3 天 |

### 🟡 A 级 · 第二批（2-4 周动手）

视觉密度高但调用频率适中。

| Skill | 唤醒词数 | 改造理由 | 模板数 | 估算 |
|---|---|---|---|---|
| **学习系统** | 10+ | 番茄钟/卡片/统计适合"进度可视化"模板 | 2-3 | 2 天 |
| **瑞幸咖啡** | 10+ | 订单详情/积分/排行适合"详情页 + 排行" | 2-3 | 1-2 天 |

### 🟢 B 级 · 第三批（按需）

复用频率低或视觉密度小，单条 CRUD 居多。

| Skill | 备注 |
|---|---|
| **大厨** | 食谱详情可做，但不紧急 |
| **健身/卡路里**（已改） | 已 6 个模板，剩小功能 |
| **百科采集** | 数据采集类，HTML 价值低 |
| **美团/微信运动** | 外部系统对接，价值有限 |
| **冥想/相册** | 已有部分内容，优先级低 |

## 三、改造 SOP（标准操作流程）

### Step 1 · 调研 30 分钟

```bash
# 1. 看 Skill 目录
ls -la ../<skill>/

# 2. 检查 CLI 是否支持 JSON
python3 ../<skill>/scripts/<cli>.py --help | grep -E "format|json"

# 3. 跑一下主要 CLI 看输出
python3 ../<skill>/scripts/<cli>.py 2>&1 | head -30
```

### Step 2 · BUG 清单 + 第一性原理回答 60 分钟

**必填 4 个问题**（参见 [01 案例 §三](./01_卡路里Skill改造案例.md#三17-个-bug-汇总按严重度)）：

1. 哪些数据需要"可视化"（多字段 + 多条目）？
2. CLI 能返回结构化 JSON 吗？不支持就需要先重构。
3. 视觉密度高的唤醒词有几个？优先级排序。
4. 现有 HTML 模板（如有）符合《手册》规范吗？

### Step 3 · 最小阻力 Phase 1（1 天）

按卡路里 P0 #5 模式：
- **改 1 个 CLI 加 `--format json`** → 1 小时
- **写 1 个简单模板（按钮 + 数据展示）** → 2-3 小时
- **写 1 个渲染器** → 1 小时
- **Chrome 验证 + commit** → 1 小时

### Step 4 · 渐进到完整 5 Phase

按卡路里 4 Phase 经验：
- **Phase A**：最小阻力（已合规 CLI 的功能 → 1 模板）
- **Phase B**：数据已合规的功能 → 直接做
- **Phase C**：规范化已有 HTML
- **Phase D**：CLI 重构 + 阻塞功能解锁

## 四、跨 Skill 共享资源（一次沉淀，多次受益）

### 共享 1：模板色板（`_shared.css`）

```css
/* 跨 Skill 统一色板（可抽出做 _shared.css）*/
:root{
  --bg:#fafafa;--card:#fff;
  --fg:#1d1d1f;--fg2:#6e6e73;--fg3:#86868b;
  --accent:#0071e3;--accent-soft:#f5f5f7;
  --green:#34c759;--orange:#ff9500;--red:#ff3b30;
  --shadow:0 1px 3px rgba(0,0,0,.04),0 4px 16px rgba(0,0,0,.06);
}
body{
  font-family:-apple-system,BlinkMacSystemFont,"PingFang SC","Microsoft YaHei",sans-serif;
}
```

**复用时机**：下次任何 Skill 写模板时，直接 `@import "_shared.css"`

### 共享 2：占位符 + 注入器（`_injector.py`）

```python
# scripts/_injector.py（建议沉淀到 templates/_shared/）
def inject(template: str, data: dict) -> str:
    if template.count('<!--INJECT-DATA-->') != 1:
        raise ValueError("占位符必须唯一")
    payload = json.dumps(data, ensure_ascii=False).replace('</', '<\\/')
    inject = f'<script>window.__DATA__ = {payload};</script>'
    return template.replace('<!--INJECT-DATA-->', inject, 1)
```

**复用时机**：下次任何 Skill 写 `render_*.py` 时，直接 `from _injector import inject`

## 五、量化收益预估

| 维度 | 卡路里（已改造） | 改造后预期 |
|---|---|---|
| **模板数** | 0 → 6 | 唤醒词覆盖 8/60+ → 13% |
| **复盘类功能 HTML 率** | 0% | 100% |
| **CLI JSON 化率** | ~10% | 100%（all analysis 函数） |
| **用户对每个 Skill 体验** | 文本 | 文本 + 可视化 |

**预估给 50+ Skill 整体改造后**：
- 总模板数 **~80-120 个**
- 总 commit **~200-300 个**
- 总耗时 **~8-12 周**（单人） / **~3-4 周**（多 agent 并行）
