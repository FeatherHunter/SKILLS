## 二、5 个共通设计原则

### 原则 1: 模板四段式（首屏 / 主体 / 交互 / 尾部）

每个模板都按这个结构组织，与《手册》第 7 节对齐：

```
┌────────────────────────────────┐
│  首屏 (HERO)                     │
│  - 标题 / 时间范围 / 状态徽章     │
│  - 关键 KPI 卡 4-6 个            │
├────────────────────────────────┤
│  主体 (BODY)                     │
│  - 按维度分组 (折叠 details)     │
│  - 表格 + 列表 + mini chart       │
├────────────────────────────────┤
│  尾部 (TAIL)                     │
│  - 复制回 AI 按钮                 │
│  - "回到顶部" 锚点                │
└────────────────────────────────┘
```

| 段落 | 元素 | 用途 |
|---|---|---|
| 首屏 | 状态徽章 + KPI 卡 | 5 秒内让用户知道"现状如何" |
| 主体 | 折叠/分组/排序 | 让用户扫读而非滚动 |
| 尾部 | 复制按钮 | 让 AI 接力分析 |

### 原则 2: 5 种必有的状态

| 状态 | 何时出现 | 处理 |
|---|---|---|
| **正常态** | 数据完整 | 完整渲染 |
| **空态** | 列表为空（ex: "无禁忌记录"） | empty-state 卡片 |
| **缺数据态** | 关键字段缺失 | 友好提示 |
| **错误态** | 没注入数据 / JSON 解析失败 | ❌ 标记 + 错误消息 |
| **离线态** | `window.addEventListener('offline')` | offline banner |

**共同组件**：5 个模板都用 `[OK / WARN / FAIL]` 状态徽章 + 颜色映射（绿/橙/红）。

### 原则 3: 一致的设计系统

| 维度 | 统一规则 | 案例 |
|---|---|---|
| **主色** | Apple 蓝 `#0071e3` | 6 个模板都用 |
| **背景** | `#fafafa` | 浅色 + 留白 |
| **阴影** | `0 1px 3px rgba(0,0,0,.04), 0 4px 16px rgba(0,0,0,.06)` | 卡片统一 |
| **圆角** | 12-16px | 按钮/卡片 |
| **字体** | `-apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei"` | 系统优先 |
| **KPI 卡布局** | 4 列网格 + 渐变背景 | 首屏统一 |

### 原则 4: 占位符 + 注入模式（5 个必须遵守）

```html
<!-- 1. 唯一占位符 -->
<!--INJECT-DATA-->

<script>
window.__DATA__ = { /* 注入的 JSON */ };
</script>
```

**5 个必须**：

1. 唯一占位符 `<!--INJECT-DATA-->`（恰好 1 次）
2. `</` 转义防断标签（`.replace('</', '<\\/')`）
3. JS 守卫 `typeof window.__DATA__ === 'object'`（防未注入）
4. 5 种状态 fallback（空/错误/缺数据/离线）
5. `escapeHTML()` 工具函数防 XSS

### 原则 5: 复制回 AI 是核心交互

所有 6 个模板都有"📤 复制回 AI"按钮：

```js
function copyToAI() {
  const text = document.getElementById('copyText').textContent;
  navigator.clipboard.writeText(text).then(() => {
    const btn = document.getElementById('copyBtn');
    btn.textContent = '✅ 已复制';
    setTimeout(() => { btn.textContent = '复制摘要'; }, 2000);
  }).catch(() => { /* fallback for non-HTTPS */ });
}
```

**优点**：
- 用户主动从页面切换到 AI 对话（不强制）
- 复制内容是结构化文本（AI 可直接解析）
- 失败有 fallback（非 HTTPS 也能复制）

## 三、3 种实现模式（按数据复杂度）

### 模式 A：Jinja2 渲染器（生成后基本静态）

- **适合**：详情页 / 报告页 / 采购清单（如 workout_plan_view 的早期版本）
- **流程**：CLI JSON → render.py → Jinja2 template.render(data) → output.html
- **优点**：生成后完全静态，无 JS 依赖
- **缺点**：修改样式要重新生成

### 模式 B：静态 HTML + window 注入（推荐）⭐

- **适合**：做菜模式 / 步骤向导 / 离线交互页（如 6 个模板都用这种）
- **流程**：模板预渲染结构 → 数据通过 `window.__DATA__` 注入 → JS 客户端渲染 DOM
- **优点**：模板稳定，数据变动不重新生成
- **缺点**：完全依赖 JS（但现代浏览器都支持）

### 模式 C：HTML 内嵌 JSON Script

- **适合**：数据很大、想避免全局变量污染
- **流程**：`<script type="application/json">{...}</script>` + JS 解析
- **优点**：无全局污染，可被其他 JS 框架消费
- **缺点**：稍复杂，需要 JSON.parse

**卡路里 6 个模板全部用模式 B**（最简洁）。

## 五、共享 Injector 提案

```python
# D:\2Study\StudyNotes\SKILLS\模板HTML并注入数据\_shared\injector.py
"""共享注入器 — 所有 render_*.py 复用"""
import json


def inject(template: str, data: dict, placeholder: str = '<!--INJECT-DATA-->') -> str:
    """读模板 + 注入 window.__DATA__ + 校验占位符唯一"""
    if template.count(placeholder) != 1:
        raise ValueError(f"占位符数量异常: 期望 1, 实际 {template.count(placeholder)}")
    payload = json.dumps(data, ensure_ascii=False).replace('</', '<\\/')
    inject_str = f'<script>window.__DATA__ = {payload};</script>'
    return template.replace(placeholder, inject_str, 1)
```

**未来用法**：
```python
# render_xxx.py
import sys
sys.path.insert(0, str(Path(__file__).parent.parent.parent / '模板HTML并注入数据' / '_shared'))
from injector import inject

template = read('templates/xxx.html')
output = inject(template, data)  # 1 行调用，5 个原则都满足
```

```js

renderCategory(t.dataset.cat)  // tab.onclick
```

**适用条件**：X 个唤醒词的输出字段一致。

### 复用 2：`review_template_v2.html`（5 复盘类型）

```js
renderHeader(e.range.type)  // day/week/month/year/custom
```

**适用条件**：X 个唤醒词只是数据范围不同。

```js
renderHeatmap(startDate, endDate)  // 默认 7 天
```

**适用条件**：X 个唤醒词只是时间窗口不同。
