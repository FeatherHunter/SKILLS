# 每日检查

## 核心设计

**不再使用数据库，不再使用issues表，不再使用脚本**。

纯自然语言问答：
1. AI查询各技能数据
2. 全面展示查询结果
3. 一次抛出6个问题，用户按需回答
4. AI根据回答给简短评价/建议

---

## 工作流程

### Step 1：查询各技能数据

按顺序查询以下技能和对应数据：

| 顺序 | 问题 | 技能 | 查什么 |
|------|------|------|--------|
| 1 | 备忘录待办 | 备忘录 | 所有待提醒事项 |
| 2 | 体重 | 卡路里 | 最近体重记录 + 7天趋势 |
| 3 | 热量 | 卡路里 | 昨日摄入/消耗数据 |
| 4 | 睡眠 | 作息管家 | 昨日睡眠时长 |
| 5 | 运动 | 卡路里 | 昨日运动记录 |
| 6 | 作息 | 作息管家 | 昨日作息执行情况 |

### Step 2：展示查询结果

AI将查询到的数据格式化展示，示例：

```
📋 每日检查 · 05-25

【备忘录】📝
  • 14:00 会议
  • 20:00 学习提醒
  • 暂无其他待办

【体重】⚖️
  最近：95.2kg（05-25早上）
  趋势：近7天基本平稳，轻微下降0.3kg

【热量】🔥
  昨日摄入：1800卡 / 目标2000卡
  昨日消耗：400卡

【睡眠】😴
  昨日：6.5小时（01:00 - 07:30）

【运动】🏃
  昨日：骑行45分钟

【作息】📅
  昨日执行：基本正常，01:00略晚
```

### Step 3：一次抛出6个问题

展示完数据后，一次性问完6个问题：

```
--- 今天的情况 ---

1. 今天体重有变化吗？报个数
2. 今天吃了啥/热量大概多少？
3. 今天睡了多久？
4. 今天运动了吗？做了啥/多久？
5. 今天作息正常吗？几点睡几点起的？
6. 备忘录有没有要新增的待办/提醒？
```

### Step 4：用户回答，AI简短评价

- 用户回答哪些就哪些，不强制全答
- 每有一条有效回答 → AI给一句评价/建议
- 没回答 → 跳过

**示例**：
```
用户：今天体重94.8kg了
AI：👍 掉了0.4kg，趋势不错，继续保持

用户：晚上吃了烧烤超标了
AI：哈哈偶尔放纵一下没问题，明天记得回来就行😂

用户：今天睡眠还没统计
AI：没关系，等你睡了再告诉我，或者明早一起报都行～
```

## 触发方式

### 手动触发

用户说以下任意一句即触发：
- "每日检查"
- "健康检查"
- "检查一下"
- "lint"

### 定时触发

Cron：每天 22:00

---

## 注意事项

- 查询各技能时，如果某技能没有数据，返回"暂无数据"而不是报错
- 如果某技能查询出错，跳过该技能，不影响其他
- 不存任何数据到数据库，每次检查都是独立对话
- 六个问题顺序固定为：备忘录 → 体重 → 热量 → 睡眠 → 运动 → 作息
- 展示数据时用实际日期替换示例中的日期
