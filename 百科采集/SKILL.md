## 唤醒词清单（12个）

> 全部为动词+名词格式，指令化，不追求自然。

### A. 采集模式（5个）

| 唤醒词 | 功能 | 说明 |
|--------|------|------|
| 采集百科 | 全流程(随机) | 随机选领域→搜索→生成文档 |
| 指定领域 | 全流程(指定) | 后接领域名，如"指定领域 医学" |
| 搜索百科 | 仅搜索 | 只执行搜索，不生成文档 |
| 迭代采集 | 换领域重试 | 搜索失败时换领域循环 |
| 批量采集 | 多领域采集 | 一次采集多个领域 |

### B. 搜索引擎（3个）

| 唤醒词 | 功能 | 说明 |
|--------|------|------|
| 深度搜索 | 双引擎搜索 | Tavily + MiniMax 联合搜索 |
| 备用搜索 | MiniMax搜索 | 直接调用 MiniMax |
| 切换引擎 | 手动切换 | 指定使用哪个搜索引擎 |

### C. 输出控制（4个）

| 唤醒词 | 功能 | 说明 |
|--------|------|------|
| 生成百科 | 生成文档 | 已有搜索结果时，生成完整文档 |
| 精简百科 | 精简输出 | 只输出标题+介绍，省略延伸/评语 |
| 扩展知识 | 延伸板块 | 只输出延伸知识部分 |
| 百科评语 | 评语板块 | 只输出悟空评语 |

```

    ↓

    无唤醒词命中 → 默认走"采集百科"全流程
```

## 工作流

### Step 1：随机选行业领域

从以下领域中随机挑选一个：

医学、历史、地理、科技、生物、物理、化学、天文、民俗、艺术、哲学、文学、经济、法律、教育、心理、宗教、军事、农业、气象、材料、环境

### Step 2：Tavily 深度搜索

使用 mcp_tavily_tavily_search 对该领域进行百科知识搜索。
- 查询格式：如 "医学 百科知识" / "历史 百科 趣味知识"
- 获取有深度的科普内容

### Step 3：失败时切换 MiniMax 搜索

如果 Tavily 搜索失败或结果不理想，使用 mcp_minimax_web_search 补充搜索。

### Step 4：迭代直到成功

如果上述两步都未获得有效内容，换一个相关但不重复的领域重新搜索，循环迭代直到成功采集到内容。

### Step 5：生成文档

整合搜索结果，输出 Markdown 文件，内容包括：

1. **词条标题** + 领域标签
2. **百科词条介绍**（300-500字，图文并茂的自然段落）
3. **延伸知识**（2-3个趣味知识点）
4. **悟空评语**（20字以内，一句话点评）
