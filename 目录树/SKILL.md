# 目录树 · 技能文档

## 触发词

- `目录树`
- `目录导航`

---

## 根目录规则

```
Step 1：读取环境变量 DIR_NAV_ROOT
        → 有值 → 验证路径存在 → 直接使用
        → 无值 → 进入 Step 2

Step 2：检查会话状态 current_root
        → 有 → 直接使用
        → 无 → 询问用户输入路径

Step 3：用户输入路径 → 验证存在 → 存入会话状态
```

---

## HTML 生成规则

### 输出路径

```
{根目录}/directory-navigator.html
```

### 扫描规则

- `os.walk()` 递归遍历所有子目录和文件
- 每项记录：name / path / is_dir / size / children[]

### HTML 要求

| 要求 | 实现 |
|------|------|
| 默认全部折叠 | 只展开根目录 + 第一层子目录 |
| 点击展开/折叠 | JS `classList.toggle('open')` |
| 复制路径按钮 | 每项右侧按钮，Clipboard API |
| 路径格式判断 | `path.startswith('/mnt/')` → WSL，否则 Windows |
| 纯静态 | 无后端，纯 JS |
| 无需服务器 | 双击 HTML 即可在浏览器打开 |

---

## 缓存策略

```
检测到已有 HTML → 询问用户：
  「检测到已有缓存，是否更新？（输入「更新」重新生成，或直接打开现有文件）」
用户输入「更新」→ 重新扫描生成
用户输入其他 → 打开现有 HTML
```

---

## 状态管理

```python
current_root: str     # 当前根目录（已确认的）
root_confirmed: bool  # 是否已确认根目录
html_path: str        # 生成的 HTML 完整路径
```

---

## 工作流程

```
用户：目录树
    ↓
检查 DIR_NAV_ROOT 环境变量
    ↓
有值 → 验证路径存在 → 确认根目录
无值 → 检查会话状态 current_root
            有 → 直接使用
            无 → 询问用户输入路径
    ↓
确认根目录后 → 检测 HTML 是否已存在
    ↓
有缓存 → 询问「更新？」→ 用户输入「更新」→ 重新生成
         → 用户输入其他 → 打开现有文件
无缓存 → 直接生成
    ↓
生成完毕 → 告诉用户 HTML 路径，提示可双击打开
```

---

## 错误处理

| 情况 | 处理 |
|------|------|
| 根目录路径无效 | 报错并重新询问 |
| 目录无法访问 | 提示「无权限访问该目录」+ 询问其他路径 |
| 扫描中断（空目录） | 正常生成，空目录显示「（空）」 |
| HTML 生成失败 | 提示错误，不创建空文件 |
