# AI技能执行者工作流审计设计文档

> 日期：2026-05-23
> 状态：已批准
> 目标：以AI执行者视角走查全部7个功能，发现阻塞性故障和逻辑矛盾

---

## 审计范围

覆盖7个功能模块，每个走2-3条典型用户输入路径，共16条功能测试路径 + 10条路由测试用例：

| 功能 | 优先级 | 测试路径数 | 重点场景 |
|------|--------|-----------|---------|
| 做菜模式 | P1 | 2 | 完整做菜流程 + 中断恢复 |
| 查看食谱 | P2 | 2 | 基础查看 + HTML生成 |
| 搜索筛选 | P3 | 3 | 按名称/按食材/组合筛选 |
| 修改食谱 | P4 | 3 | 改步骤/加食材/废弃食谱 |
| 烹饪历史 | P5 | 2 | 记录烹饪/查看统计 |
| 采购清单 | P6 | 2 | 单菜/多菜合并 |
| 录入食谱 | P7 | 2 | CLI方式 + JSON导入 |

---

## 审计方法

对每条路径，以AI执行者视角逐步推演：

```
用户输入
    ↓
第1步：路由匹配
  - SKILL.md中的触发词能否匹配？
  - 菜名提取是否正确？
  - 优先级是否正确？
    ↓
第2步：功能文件加载
  - 对应的features/*.md是否可读？
  - 文件中的指令是否完整可执行？
    ↓
第3步：依赖检查
  - 外部技能引用是否存在？
  - 文件路径是否有效？
  - CLI命令是否正确？
    ↓
第4步：输出验证
  - 预期输出格式是否合理？
  - 边界情况是否处理？
```

**输入**：SKILL.md路由表

**产出**：路由匹配正确性报告

**测试用例**：
| 用户输入 | 预期路由 | 预期菜名 |

### 任务2-8：各功能走查

**方法**：对每个功能，模拟AI执行者完整执行流程：

1. 读取features/*.md，理解功能要求
2. 按用户输入构造CLI命令
3. 检查命令参数是否正确
4. 检查输出格式是否符合预期
5. 记录所有发现的问题

**重点检查项**：
- 外部技能引用（Taste Skill、UI/UX Pro Max）路径是否存在
- 文件存储路径（/home/feather/.openclaw/media/qqbot/）是否有效
- CLI命令参数格式是否与commands.md一致
- 数据库表名和字段名是否正确
- HTML生成要求是否完整可执行

### 任务9：依赖链审计

**输入**：所有功能文件中的外部引用
**方法**：提取所有外部依赖，逐一验证
**产出**：依赖风险报告

**依赖清单**：
| 依赖类型 | 引用位置 | 风险点 |
|---------|---------|--------|
| Taste Skill | view.md | 外部技能路径 ~/.openclaw/skills/taste-skill/ |
| UI/UX Pro Max | view.md | 外部技能路径 ~/.openclaw/skills/ui-ux-pro-max/ |
| QQBot文件路径 | view.md, shopping.md | /home/feather/.openclaw/media/qqbot/ |
| 数据库文件 | 所有功能 | scripts/db_config.py 中的路径 |
| Manager脚本 | 所有功能 | scripts/manager_*.py 是否存在 |
| recipe_import.py | P7 录入 | JSON导入脚本 |

### 任务10：交叉引用一致性检查

**输入**：references/commands.md + 各feature文件
**方法**：对比feature文件中引用的命令与commands.md中的定义
**产出**：不一致清单

**检查项**：
- feature文件中的CLI命令格式是否与commands.md一致
- 参数名是否拼写正确
- 必填/可选标记是否一致
- 字段值范围是否一致

### 任务11：汇总与优先级排序

**输入**：任务1-10的所有发现
**方法**：按严重程度分类，去重，排序
**产出**：最终审计报告

## 故障记录格式

```markdown
### [严重程度] 故障标题

- **位置**：文件名:行号 或 功能模块
- **现象**：AI执行者会遇到什么问题
- **复现路径**：用户输入 → 期望行为 → 实际行为
- **影响范围**：影响哪些功能/用户场景
- **修复建议**：具体怎么改
```

---

## 严重程度分级

| 级别 | 定义 | 示例 |
|------|------|------|
| P0-阻塞 | 功能完全无法执行 | 外部技能路径不存在、CLI命令参数错误 |
| P1-严重 | 主流程有明显缺陷 | 路由匹配错误、必填字段遗漏导致崩溃 |
| P2-中等 | 边界情况处理不当 | 菜名提取在特殊输入下失败 |
| P3-轻微 | 体验问题，不影响功能 | 输出格式不一致、提示信息不清晰 |

---

## 执行方式

**建议使用Subagent-Driven Development**：
- 任务1-8可并行执行（各自独立）
- 任务9-10依赖前序结果
- 任务11为汇总任务

**每个任务的执行者**：
- 读取相关文件
- 按审计方法逐步推演
- 记录所有发现的故障
- 输出结构化的故障清单
