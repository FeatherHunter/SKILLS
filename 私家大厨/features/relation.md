# 派生关系

> 路由：SKILL.md 用例 H → features/relation.md

---

## ⚠️ 操作规范（强制）

本技能所有数据操作必须通过 CLI，禁止直连数据库。

---

## 功能说明

派生关系用于标记一道菜基于另一道菜的变体（衍生、改良、变体），形成"食谱族谱"。

支持的场景：
- **派生**：改良版宫保虾球 → 宫保虾球（父本）
- **变体**：川式麻婆豆腐 vs 粤式麻婆豆腐
- **改良**：低脂版红烧肉 → 经典红烧肉

**遵循规范**：SKILL.md 中的"AI使用规范"和"字段推测规则"。

---

## 内部分流逻辑

```
用户说"添加派生关系"
    → 询问父本菜名（必填）
    → 询问子本菜名（必填）
    → relation_type 推测（派生/变体/改良，根据上下文）
    → change_summary 可选（描述改了什么）
    → 执行 relation_manager.py add

用户说"查看派生关系"
    → relation_manager.py list-parent <菜名>（谁基于我变）
    → relation_manager.py list-child <菜名>（我基于谁）
    → 合并展示
```

---

## 工作流

### 添加派生关系

```
用户说"添加派生关系"
    ↓
【问父本】菜名或 ID
    ↓
【问子本】菜名或 ID
    ↓
【问类型】派生 / 变体 / 改良（默认派生）
    ↓
【问改动说明】可空
    ↓
【执行】relation_manager.py add \
    --parent_id <ID> --child_id <ID> \
    [--relation_type 派生] [--change_summary "..."]
```

### 查看派生关系

```
用户说"查看派生关系 宫保虾球"
    ↓
【查父本】relation_manager.py list-parent <ID>
    → "基于我的变体"：列出所有 child_id = 自己的记录
【查子本】relation_manager.py list-child <ID>
    → "我的父本"：列出所有 parent_id = 自己的记录
    ↓
【合并展示】按方向分组显示
```

---

## 字段映射

| 字段 | 必填 | 推断 | 说明 |
|------|------|------|------|
| `parent_id` | ✅ | — | 父本食谱 UUID（必须存在） |
| `child_id` | ✅ | — | 子本食谱 UUID（必须存在） |
| `relation_type` | ❌ | 派生（默认）/ 变体 / 改良 | 关系类型枚举 |
| `change_summary` | ❌ | — | 自由文本，说明改了什么 |

---

## 错误处理

| 场景 | 行为 |
|------|------|
| 父本或子本不存在 | 报错"未找到食谱"，要求重输 |
| 父本 == 子本 | 报错"不能派生自身" |
| 同一对已存在 | 报错"已存在关系"，可选择更新 change_summary |
| 双方均已废弃 | 警告"双方都已废弃，是否仍要登记？" |

---

## 边界

- 仅支持 1 对 1（不直接支持网状关系，需多次 add）
- 不级联：删一道菜时不会自动删它的关系（待评估是否需要 ON DELETE CASCADE）
- 跨技能：不联动到 居家管家（菜谱实体不共享）
