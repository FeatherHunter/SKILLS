?? JSON文件结构详解

一个完整的食谱JSON文件包含以下部分。AI只需要按照这个结构填充数据即可：

recipe.json - 宫保虾球 
{
  // ?? 基本信息
  "name": "宫保虾球",
  "description": "川菜经典，虾球Q弹，酸甜微辣",
  "difficulty": "中等",
  "servings": 2,
  "total_time": 25,
  "status": "未做",

  // ?? 分类信息
  "category": {
    "cuisine": "川菜",
    "region": "中国-四川",
    "country": "中国"
  },

  // ?? 食材清单
  "ingredients": [
    {
      "name": "虾",
      "quantity": 300,
      "unit": "g",
      "category": "海鲜",
      "sequence": 1
    },
    {
      "name": "花生",
      "quantity": 50,
      "unit": "g",
      "category": "干货",
      "is_optional": true
    }
    // ... 更多食材
  ],

  // ?? 烹饪步骤
  "steps": [
    {
      "sequence": 1,
      "action": "虾去壳开背，用料酒和盐腌制10分钟",
      "duration": 10,
      "heat_level": "小火",
      "expected_result": "虾肉变红，去腥",
      "ingredients_used": [
        {
          "name": "虾",
          "quantity_used": 300
        }
      ]
    }
    // ... 更多步骤
  ],

  // ?? 可选信息
  "tips": [...],
  "cookware": [...],
  "nutrition": {...}
}

JSON结构的优势

1. 自描述性：字段名就是说明（name, difficulty, ingredients）
2. 层次清晰：食材、步骤、分类各自独立，不会混淆
3. 关联自然：步骤中直接引用食材名称，无需传递ID
4. 可选字段明确：没有的字段直接省略，不影响其他部分

?? 学习成本对比

传统方式 需要记忆的内容

10个不同的命令：
  • recipe_manager.py add - 位置参数
  • category_manager.py add - ID + 命名参数
  • ingredient_manager.py add - ID + 命名参数
  • step_manager.py add - ID + 命名参数
  • step_ingredient_manager.py add - 双ID + 命名参数
  • technique_manager.py add - 双ID + 命名参数
  • tip_manager.py add - ID + 可选ID + 命名参数
  • background_manager.py add - ID + 命名参数
  • cookware_manager.py add - ID + 命名参数
  • nutrition_manager.py add - ID + 命名参数
还要记住：
  • 每个命令的参数顺序
  • 哪些是位置参数，哪些是命名参数
  • 如何获取和传递ID
  • 步骤的正确执行顺序

JSON导入 需要记忆的内容

1个命令：
python recipe_import.py import <json文件>
1个JSON结构：
  • name - 菜名
  • difficulty - 难度
  • ingredients[] - 食材数组
  • steps[] - 步骤数组
  • category - 分类对象
  • ... 其他可选字段
只需知道：
  • JSON是键值对结构
  • 数组用 [] 包裹
  • 没有的字段可以省略
