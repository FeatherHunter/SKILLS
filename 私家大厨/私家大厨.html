私家大厨 🍳

用户的私家菜谱本。录菜、查菜、做菜、采购清单、烹饪历史评分,一站管理。

同步规范: 本手册内容必须与 SKILL.md 完全一致;任何唤醒词、CLI 命令、功能的修改都要同步更新本页。详见 SKILL.md 顶部 「HTML 同步规范(最高优先级)」。

A. 做菜模式 (2 个)

进入交互式做菜界面。系统调取食谱全量数据,渲染做菜 HTML(带计时/份数切换/剩余时间),输出到 $CHEF_OUTPUT_DIR/cooking/。

做菜模式进入做菜模式,需配合菜名 ▼
触发示例

「做菜模式 宫保虾球」「开始做红酒慢炖牛腩」

cooking_render.py render <菜名或ID>

AI 收到此唤醒词必须自动调用此命令。脚本内部自动执行 recipe_manager.py show --json 拿 17 表全量数据,然后注入 templates/cooking_mode.html 渲染。

输出位置$CHEF_OUTPUT_DIR/cooking/做菜模式_<recipe_slug>_<YYYYMMDD_HHMMSS>.html
读取的表recipes, ingredients, cooking_steps, step_ingredients, tips, step_techniques, nutrition_info, background_knowledge, cookware, recipe_history, recipe_relations 等 17 张
副作用无(纯渲染)
必发规则渲染成功后必须用 <media> 标签把 HTML 推给用户,不能只给路径
开始做菜同上,口语化入口 ▼
触发示例

「开始做宫保虾球」

cooking_render.py render <菜名或ID>

与「做菜模式」同命令。R2 最长匹配保证两词不冲突。

D. 修改食谱 (9 个)

修改粒度细到字段级。原则:只增不删,整道菜废弃用 discard(标记为已废弃),不物理删除。所有操作必走 CLI,禁直连数据库。

修改食谱通用修改入口,需配合菜名+修改内容 ▼
前置步骤

先 recipe_manager.py show <菜名> 拿 recipe_id。

recipe_manager.py update <recipe_id> [--name|--description|--difficulty|--servings|--total_time|--status|--photo_url|--source|--source_url]
写入字段recipes 表对应列
校验枚举值(difficulty/status)过 validators.py;缺必填字段返回友好报错
事务db.execute() 自动包裹事务
副作用无(只改 recipes 单行)
修改步骤改步骤内容/顺序 ▼
前置步骤

recipe_manager.py show <菜名> 拿 recipe_id;step_manager.py list <recipe_id> 拿 step_id。

step_manager.py update <step_id>
写入字段cooking_steps.action / sequence / duration / heat_level / temperature / expected_result
校验validate_heat_level(5 值枚举) / validate_positive_int(duration)
副作用无
step_manager.py reorder <recipe_id> --from <N> --to <M>
写入字段cooking_steps.sequence
副作用调整序列后,step_ingredients 关联不动
修改食材改食材用量/添加食材 ▼
ingredient_manager.py update <ingredient_id>
写入字段ingredients.name / quantity / unit / quantity_text / sequence / category / substitute
校验validate_array_enum(category 枚举) / L1 NOT NULL 兜底
副作用step_ingredients 关联需手工重新关联
ingredient_manager.py add <recipe_id> --name --quantity --unit ...
写入字段ingredients 全字段
副作用新增食材行,自动 sequence+1
修改难度改难度等级 ▼
recipe_manager.py update <recipe_id> --difficulty <新值>
写入字段recipes.difficulty
合法值快手菜/简单/中等/困难/大师
校验枚举校验,非法值返回"合法值清单"
副作用无
修改份量改份数 ▼
recipe_manager.py update <recipe_id> --servings <人数>
写入字段recipes.servings
校验validate_positive_int(正整数)
副作用无(注意:食材 quantity 不自动按比例缩放,需手工改)
废弃食谱标记为已废弃(只增不删) ▼
recipe_manager.py discard <recipe_id>
写入字段recipes.status = '已废弃'
物理删除无(保留所有历史记录,只标记)
副作用该菜不再出现在常规列表;历史记录/派生关系保留

等价唤醒词:「不想要」「删掉」「废弃」都路由到此命令。

不想要口语化入口,等价「废弃食谱」 ▼
recipe_manager.py discard <recipe_id>

同「废弃食谱」CLI。R2 最长匹配:这些短词不会拦截更长唤醒词(如「废弃食谱」)。

删掉口语化入口,等价「废弃食谱」 ▼
recipe_manager.py discard <recipe_id>

同「废弃食谱」CLI。R2 最长匹配:这些短词不会拦截更长唤醒词。

E. 烹饪历史 (3 个)

记录/查看/统计做菜历史。菜名或 recipe_id 都支持,脚本自动识别。

记录做菜记录一次烹饪(评分+反馈) ▼
触发示例

「今天做了宫保虾球,评分 4.5,反馈虾很 Q 弹」

history_manager.py add <recipe_id或菜名> [--cook_date] [--rating] [--feedback]
写入字段recipe_history.recipe_id / cook_date / rating / feedback
校验validate_rating(1-5 范围) / validate_date_format(YYYY-MM-DD)
默认值cook_date 默认今天
副作用新增 history 行;recipes.avg_rating 由 stats 重算
python scripts/history_manager.py add "宫保虾球" \
  --cook_date 2026-05-15 \
  --rating 4.5 \
  --feedback "虾很Q弹,下次可以少放点盐"
查看历史查看某道菜的烹饪历史 ▼
history_manager.py list <recipe_id或菜名>
读取的表recipe_history JOIN recipes
返回烹饪记录列表(日期/评分/反馈)
副作用无
查看统计查看评分统计(均分/次数) ▼
history_manager.py stats <recipe_id或菜名>
读取的表recipe_history
聚合COUNT(*)、AVG(rating)、最近反馈摘要
副作用无

录入食谱录入新食谱(图片/文本/JSON) ▼

「录入这道菜」(附图片/文本)/「录入食谱 宫保虾球」(信息完整时走 JSON 导入)

import_orchestrator.py <json_file> [--dry-run] [--json]统一入口(推荐)

所有数据导入走单入口,事务包裹。AI 先用 recipe_json_validate.py 校验,再用 import_orchestrator.py 写入。

读取JSON 文件(templates/recipe_template.json schema)
写入17 张表(事务包裹,任一失败回滚)

状态字段success / error / warning / dry_run
recipe_import.py import <json_file>(备选路径)

同 import_orchestrator 行为;L3 后统一由 orchestrator 替代。

冲突处理--choice view|derive|update|cancel
派生--choice derive --new_name "新菜名"

添加派生关系标记父子关系(派生/变体/改良) ▼
触发示例

「添加派生关系 宫保虾球 基于 上海排骨年糕」「宫保虾球 是 上海排骨年糕 的变体」

relation_manager.py add --parent_id <父ID> --child_id <子ID> [--relation_type] [--change_summary]
写入字段recipe_relations.parent_id / child_id / relation_type / change_summary
合法值relation_type: 派生 / 变体 / 改良
前置父菜和子菜必须都已存在(recipe_manager show 拿 ID)
副作用新增 relations 行;子菜的 查看派生关系 会列出父本
python scripts/relation_manager.py add \
  --parent_id 8f3b435b-... \
  --child_id cde0db94-... \
  --relation_type 变体 \
  --change_summary "减少干辣椒,增加甜味,适合儿童"
查看派生关系查看一道菜的所有父本/子本关系 ▼
relation_manager.py list-parent <recipe_id或菜名>父本(谁基于我变)
