# 一面：技术基础面SOP

> 轮次：第一轮（技术面）  
> 考察重点：基础概念、编码能力、技术广度  
> 对应Level：L1-L4  
> 系统推荐角色Soul：严谨追问者、温和引导者

---

## 一、流程步骤

```
+------------------------------------------------------------------+
|                    一面：技术基础面 - 流程SOP                      |
+------------------------------------------------------------------+
|                                                                  |
|  【Step 0: 角色Soul选择】                                        |
|    - 系统推荐角色（2个）：严谨追问者、温和引导者                  |
|    - 用户确认或从6种角色中选择                                   |
|    - 加载对应的 persona-*.md 文件                                |
|    - 将角色性格注入AI面试官                                      |
|                                                                  |
|  [ ] Step 0 完成：角色已加载                                     |
|                                                                  |
|  【Step 1: 题库准备】                                            |
|    - 读取主流程Step 1生成的诊断报告结果                          |
|    - 优先从诊断报告中的薄弱Level获取题目                         |
|    - Level 1-2题目占比：40%（基础概念+使用）                    |
|    - Level 3题目占比：30%（原理理解）                           |
|    - Level 4题目占比：30%（实现理解）                           |
|    - 从 LLM-WIKI QUERY 获取题目                                  |
|    - 如无数据，从 wiki/ 目录动态生成                            |
|    - 如两者都无，必须报错                                        |
|    - 题目持久化到 data/questions/{knowledge_id}/               |
|                                                                  |
|  [ ] Step 1 完成：题库已准备                                     |
|                                                                  |
|  【Step 2: 模拟面试】                                            |
|    - 总题数：10题                                                |
|    - 优先从诊断报告的薄弱Level抽取题目                           |
|    - 每题最多追问2层                                            |
|    - 记录回答质量、关键词覆盖度                                  |
|    - 写入 data/mock-history/{knowledge_id}/                    |
|                                                                  |
|  [ ] Step 2 完成：模拟面试已完成                                 |
|                                                                  |
|  【Step 3: 反馈生成】                                            |
|    - 调用 cm-feedback-generator.md 分析卡点原因                |
|    - 结合诊断报告结果，生成精准补强建议                          |
|    - 补强指向：learning-system 阶段1-4                          |
|    - 写入 data/feedback/                                        |
|                                                                  |
|  [ ] Step 3 完成：反馈已生成                                     |
|                                                                  |
|  【输出】                                                        |
|    - 模拟记录：data/mock-history/round-1/                       |
|    - 反馈文件：data/feedback/                                   |
|                                                                  |
+------------------------------------------------------------------+
```

---

## 二、题目比例

| Level | 占比 | 题型 |
|-------|------|------|
| L1-L2 | 40% | 概念题、使用题 |
| L3 | 30% | 原理题 |
| L4 | 30% | 实现题 |

---

## 三、评分标准

| 维度 | 权重 | 说明 |
|------|------|------|
| 关键词覆盖度 | 50% | 对比标准答案的关键词 |
| 回答深度 | 30% | 追问层级 |
| 表达流畅度 | 20% | 逻辑连贯性 |

---

## 四、角色Soul注入

加载角色后，AI面试官的所有对话都应体现该角色的：
- 追问风格
- 反馈方式
- 情绪表现
- 压力程度
