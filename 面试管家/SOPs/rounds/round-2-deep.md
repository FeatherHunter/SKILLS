# 二面：技术深挖面SOP

> 轮次：第二轮（技术深挖面）  
> 考察重点：项目深度、方案选型、踩坑复盘  
> 对应Level：L4-L5  
> 系统推荐角色Soul：严谨追问者、压力测试者

---

## 一、流程步骤

```
+------------------------------------------------------------------+
|                    二面：技术深挖面 - 流程SOP                      |
+------------------------------------------------------------------+
|                                                                  |
|  【Step 0: 角色Soul选择】                                        |
|    - 系统推荐角色（2个）：严谨追问者、压力测试者                  |
|    - 用户确认或从6种角色中选择                                   |
|    - 加载对应的 persona-*.md 文件                                |
|    - 将角色性格注入AI面试官                                      |
|                                                                  |
|  [ ] Step 0 完成：角色已加载                                     |
|                                                                  |
|  【Step 1: 题库准备】                                            |
|    - 读取主流程Step 1生成的诊断报告结果                          |
|    - 优先从诊断报告中的薄弱Level获取题目                         |
|    - Level 4题目占比：50%（实现理解）                           |
|    - Level 5题目占比：50%（实战案例）                           |
|    - 从 LLM-WIKI QUERY 获取题目                                  |
|    - 如无数据，从 wiki/ 目录动态生成                            |
|    - 如两者都无，必须报错                                        |
|    - 题目持久化到 data/questions/{knowledge_id}/               |
|                                                                  |
|  [ ] Step 1 完成：题库已准备                                     |
|                                                                  |
|  【Step 2: 模拟面试】                                            |
|    - 总题数：8题                                                 |
|    - 优先从诊断报告的薄弱Level抽取题目                           |
|    - 每题最多追问3层                                            |
|    - 重点考察项目中的实际使用场景                                |
|    - 写入 data/mock-history/{knowledge_id}/                    |
|                                                                  |
|  [ ] Step 2 完成：模拟面试已完成                                 |
|                                                                  |
|  【Step 3: 反馈生成】                                            |
|    - 调用 cm-feedback-generator.md 分析卡点原因                |
|    - 结合诊断报告结果，生成精准补强建议                          |
|    - 补强指向：learning-system 阶段5（实战深化）                |
|    - 写入 data/feedback/                                        |
|                                                                  |
|  [ ] Step 3 完成：反馈已生成                                     |
|                                                                  |
|  【输出】                                                        |
|    - 模拟记录：data/mock-history/round-2/                       |
|    - 反馈文件：data/feedback/                                   |
|                                                                  |
+------------------------------------------------------------------+
```

---

## 二、题目比例

| Level | 占比 | 题型 |
|-------|------|------|
| L4 | 50% | 实现题、源码题 |
| L5 | 50% | 实战案例、踩坑复盘 |

---

## 三、评分标准

| 维度 | 权重 | 说明 |
|------|------|------|
| 项目关联度 | 40% | 是否能关联到实际项目 |
| 方案选型 | 30% | 是否能说清trade-off |
| 踩坑经验 | 30% | 是否有真实踩坑经历 |

---

## 四、追问设计

二面的核心是验证用户"真的干过"：
- 问项目细节
- 问方案选型原因
- 问踩过的坑和解决过程
- 问为什么不用其他方案
