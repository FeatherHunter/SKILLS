# 三面：架构/系统设计面SOP

> 轮次：第三轮（架构/系统设计面）
> 考察重点：系统设计、trade-off、技术视野
> 对应Level：L5-L6
> 系统推荐角色Soul：战略视野者、严谨追问者

---

## 一、流程步骤

```
+------------------------------------------------------------------+
|                 三面：架构/系统设计面 - 流程SOP                    |
+------------------------------------------------------------------+
|                                                                  |
|  【Step 0: 角色Soul选择】                                        |
|    - 系统推荐角色（2个）：战略视野者、严谨追问者                  |
|    - 用户确认或从6种角色中选择                                   |
|    - 加载对应的 persona-*.md 文件                                |
|    - 将角色性格注入AI面试官                                      |
|                                                                  |
|  [ ] Step 0 完成：角色已加载                                     |
|                                                                  |
|  【Step 1: 题库准备���                                            |
|    - 读取主流程Step 1生成的诊断报告结果                          |
|    - 优先从诊断报告中的薄弱Level获取题目                         |
|    - Level 5题目占比：30%（实战深化）                           |
|    - Level 6题目占比：40%（边界探索）                           |
|    - Level 7题目占比：30%（架构设计）                           |
|    - 从 LLM-WIKI QUERY 获取题目                                  |
|    - 如无数据，从 wiki/ 目录动态生成                            |
|    - 如两者都无，必须报错                                        |
|    - 题目持久化到 data/questions/{knowledge_id}/               |
|                                                                  |
|  [ ] Step 1 完成：题库已准备                                     |
|                                                                  |
|  【Step 2: 模拟面试】                                            |
|    - 总题数：6题（偏重系统设计题）                               |
|    - 优先从诊断报告的薄弱Level抽取题目                           |
|    - 每题最多追问3层                                            |
|    - 重点考察架构设计能力                                        |
|    - 写入 data/mock-history/{knowledge_id}/                    |
|                                                                  |
|  [ ] Step 2 完成：模拟面试已完成                                 |
|                                                                  |
|  【Step 3: 反馈生成】                                            |
|    - 调用 cm-feedback-generator.md 分析卡点原因                |
|    - 结合诊断报告结果，生成精准补强建议                          |
|    - 补强指向：learning-system 阶段6-7                          |
|    - 写入 data/feedback/                                        |
|                                                                  |
|  [ ] Step 3 完成：反馈已生成                                     |
|                                                                  |
|  【输出】                                                        |
|    - 模拟记录：data/mock-history/round-3/                       |
|    - 反馈文件：data/feedback/                                   |
|                                                                  |
+------------------------------------------------------------------+
```

## 二、题目比例

| Level | 占比 | 题型 |
|-------|------|------|
| L5 | 30% | 实战深化题 |
| L6 | 40% | 边界探索题、性能极限 |
| L7 | 30% | 架构设计题 |

---

## 三、评分标准

| 维度 | 权重 | 说明 |
|------|------|------|
| 架构设计 | 35% | 分层、模块划分 |
| trade-off分析 | 35% | 方案取舍原因 |
| 技术视野 | 30% | 是否关注行业趋势 |

## 四、系统设计题

三面必有1-2道系统设计题，如：
- 设计一个消息推送系统
- 设计一个图片加载框架
- 设计一个APM监控系统
