# 四面：交叉面/HR面SOP

> 轮次：第四轮（交叉面/HR面）
> 考察重点：团队协作、成长故事、价值观匹配
> 对应Level：L6-L7
> 系统推荐角色Soul：洞察人心者、温和引导者

---

## 一、流程步骤

```
+------------------------------------------------------------------+
|                    四面：交叉面/HR面 - 流程SOP                     |
+------------------------------------------------------------------+
|                                                                  |
|  【Step 0: 角色Soul选择】                                        |
|    - 系统推荐角色（2个）：洞察人心者、温和引导者                  |
|    - 用户确认或从6种角色中选择                                   |
|    - 加载对应的 persona-*.md 文件                                |
|    - 将角色性格注入AI面试官                                      |
|                                                                  |
|  [ ] Step 0 完成：角色已加载                                     |
|                                                                  |
|  【Step 1: 题库准备】                                            |
|    - 读取主流程Step 1生成的诊断报告结果                          |
|    - 优先从诊断报告中的薄弱Level获取题目                         |
|    - Level 6题目占比：50%（边界认知）                           |
|    - Level 7题目占比：50%（架构思考）                           |
|    - 从 LLM-WIKI QUERY 获取题目                                  |
|    - 如无数据，从 wiki/ 目录动态生成                            |
|    - 如两者都无，必须报错                                        |
|    - 题目持久化到 data/questions/{knowledge_id}/               |
|                                                                  |
|  [ ] Step 1 完成：题库已准备                                     |
|                                                                  |
|  【Step 2: 模拟面试】                                            |
|    - 总题数：6题（行为面试+软技能）                              |
|    - 优先从诊断报告的薄弱Level抽取题目                           |
|    - 每题最多追问3层                                            |
|    - 重点考察价值观、成长故事、团队协作                         |
|    - 写入 data/mock-history/{knowledge_id}/                    |
|                                                                  |
|  [ ] Step 2 完成：模拟面试已完成                                 |
|                                                                  |
|  【Step 3: 反馈生成】                                            |
|    - 调用 cm-feedback-generator.md 分析卡点原因                |
|    - 结合诊断报告结果，生成精准补强建议                          |
|    - 补强指向：learning-system 阶段6-7 + 面试话术练习            |
|    - 写入 data/feedback/                                        |
|                                                                  |
|  [ ] Step 3 完成：反馈已生成                                     |
|                                                                  |
|  【输出】                                                        |
|    - 模拟记录：data/mock-history/round-4/                       |
|    - 反馈文件：data/feedback/                                   |
|                                                                  |
+------------------------------------------------------------------+
```

---

## 二、题目比例

| Level | 占比 | 题型 |
|-------|------|------|
| L6 | 50% | 边界认知、踩坑复盘 |
| L7 | 50% | 架构思考、团队影响 |

---

## 三、评分标准

| 维度 | 权重 | 说明 |
|------|------|------|
| 价值观匹配 | 35% | 与公司文化的契合度 |
| 成长故事 | 35% | 是否有清晰的成长路径 |
| 团队协作 | 30% | 能否体现影响力 |

---

## 四、常见问题类型

- "你最有挑战的项目是什么？"
- "你如何与团队成员协作？"
- "你的技术成长路径是怎样的？"
- "你为什么离职/选择新公司？"
- "你的缺点是什么？如何改进？"
