# 五面：总裁面/高管面SOP

> 轮次：第五轮（总裁面/高管面）
> 考察重点：前瞻性、价值观、影响力
> 对应Level：L7
> 系统推荐角色Soul：战略视野者

---

## 一、流程步骤

```
+------------------------------------------------------------------+
|                  五面：总裁面/高管面 - 流程SOP                     |
+------------------------------------------------------------------+
|                                                                  |
|  【Step 0: 角色Soul选择】                                        |
|    - 系统推荐角色（1个）：战略视野者                              |
|    - 用户确认或从6种角色中选择                                   |
|    - 加载对应的 persona-*.md 文件                                |
|    - 将角色性格注入AI面试官                                      |
|                                                                  |
|  [ ] Step 0 完成：角色已加载                                     |
|                                                                  |
|  【Step 1: 题库准备】                                            |
|    - 读取主流程Step 1生成的诊断报告结果                          |
|    - 优先从诊断报告中的薄弱Level获取题目                         |
|    - Level 7题目占比：100%（架构设计+战略视野）                 |
|    - 从 LLM-WIKI QUERY 获取题目                                  |
|    - 如无数据，从 wiki/ 目录动态生成                            |
|    - 如两者都无，必须报错                                        |
|    - 题目持久化到 data/questions/{knowledge_id}/               |
|                                                                  |
|  [ ] Step 1 完成：题库已准备                                     |
|                                                                  |
|  【Step 2: 模拟面试】                                            |
|    - 总题数：4-5题（高视野开放题）                               |
|    - 优先从诊断报告的薄弱Level抽取题目                           |
|    - 每题最多追问3层                                            |
|    - 重点考察技术视野、前瞻性、价值观                            |
|    - 写入 data/mock-history/{knowledge_id}/                    |
|                                                                  |
|  [ ] Step 2 完成：模拟面试已完成                                 |
|                                                                  |
|  【Step 3: 反馈生成】                                            |
|    - 调用 cm-feedback-generator.md 分析卡点原因                |
|    - 结合诊断报告结果，生成精准补强建议                          |
|    - 补强指向：learning-system 阶段7 + 行业研究                 |
|    - 写入 data/feedback/                                        |
|                                                                  |
|  [ ] Step 3 完成：反馈已生成                                     |
|                                                                  |
|  【输出】                                                        |
|    - 模拟记录：data/mock-history/round-5/                       |
|    - 反馈文件：data/feedback/                                   |
|                                                                  |
+------------------------------------------------------------------+
```

## 二、题目比例

## 三、评分标准

| 维度 | 权重 | 说明 |
|------|------|------|
| 前瞻性 | 40% | 对技术趋势的判断 |
| 价值观 | 35% | 技术为业务服务的理念 |
| 影响力 | 25% | 能否带动团队技术进步 |

## 四、常见问题类型

- "未来3-5年，你认为技术发展的方向是什么？"
- "如果让你负责一个技术团队，你会怎么做？"
- "什么是好的技术架构？"
- "你如何影响团队的技术选型？"
- "技术为业务带来什么价值？"
