# 消费分类参考(多级分类体系 v3.1)

分类采用 **L1/L2/L3 三级** 结构,用 `/` 分隔层级(如 `餐饮/外卖/午餐`)。
- 支出侧:L1 + L2 + L3(必填 L1,可填 L2/L3)
- 收入侧:L1 + L2

**数据兼容:** 旧数据中不带 `/` 的分类(如 `餐饮`、`交通`)自动视为 L1 主分类,无需迁移。

---

## 〇、分类心法(AI 解析的总纲)

> **写在前面:不要把分类当成"关键词查表"。要理解"为什么这么分",然后按意图推场景。**

### 分类的根本目的

分类不是为了"看上去整齐",是为了回答一个核心问题:

> **"我这个月钱花在哪了?哪些是必要的?哪些是浪费的?哪些是值得的?"**

如果分类不能帮决策,就是噪声。

### 判断的五个维度

碰到任何消费场景,按顺序问自己这五个问题:

1. **主体是谁?**
   - 我自己消费 → 个人类(餐饮/穿着/居家)
   - 我付费但为了别人 → 社交类(请客/礼物/红包)
   - 多人共同参与 → 社交类(看 AA 还是请客)

2. **目的是什么?**
   - 必要生活(吃饭、住、行、工作)→ 基础类
   - 娱乐/解压/放松 → 玩乐类
   - 自我提升/技能/知识 → 学习类
   - 维护健康/治病/康复 → 健康类

3. **对象是谁?**
   - 给自己 → 个人消费
   - 给家庭/共同生活 → 居家类
   - 给第三方 → 社交类

4. **工具还是体验?**
   - 工具(电脑、软件、设备)→ 看用途,可能是居家或学习
   - 体验(电影、演出、旅游)→ 玩乐类

5. **多人场景谁付费?**
   - 我全付 → 社交/请客
   - AA / 各出各 → 社交/聚会
   - 别人请 → 按场景分(餐饮),不归社交

### 边界场景的思考示例

- **"明天兄弟来,我请客吃饭 500"** → 多人(维度1)+ 我付费招待(维度5)→ `社交/请客`,**不是**餐饮
- **"家庭聚餐"** → 多人(维度1)+ 自家生活(维度3)+ 看在哪吃 → 在家做 → `居家/食材`;外出吃 → `餐饮/堂食`
- **"买了本 Python 书 60"** → 单人(维度1)+ 工具书(维度2/4)→ `学习/书籍`,**不是**玩乐
- **"瑜伽课 3000"** → 看上下文:"减压放松" → `玩乐/运动健身`;"考瑜伽证" → `学习/课程培训`
- **"洗发水 80"** → 看上下文:"超市买的" → `居家/日用品/洗护`;"理发店用的" → `穿着/洗护`
- **"出差打车"** → 工具(维度4)+ 出行目的 → `出行/网约车`,不进"玩乐"
- **"健身房办卡 3000"** → 为运动 → `玩乐/运动健身`;为治病康复 → `健康/保健品`(取决于目的)
- **"买狗粮 200"** → 给第三方(宠物)(维度3)→ `宠物/食物`
- **"话费 99"** → 必要生活(维度2)→ `居家/通讯`

### 兜底

如果五个维度都走完仍然不能确定,**默认归"其他"**,在回复中标注:

> "这个场景我没把握归类,先放到「其他」,你能告诉我是哪一类吗?"

用户答复后,AI 记下这个判断,下次类似场景直接命中。
